# 작업 계속

이전 세션에서 진행 중이던 작업을 계속합니다.

## 수행 단계

1. **활성 작업 확인**: `dev/active/` 디렉토리 스캔
2. **dev docs 읽기**:
   - plan.md: 전체 계획 파악
   - context.md: 컨텍스트 및 다음 세션 힌트 확인
   - tasks.md: 완료/미완료 작업 확인
3. **현재 상태 파악**:
   - 마지막으로 완료된 작업
   - 현재 진행 중이던 작업
   - 다음 해야 할 작업
4. **작업 재개**: 미완료 작업부터 계속

## 출력 형식

```
## 작업 재개: [task-name]

### 계획 요약
[plan.md 요약]

### 현재 상태
- 완료: N개 / 전체: M개
- 마지막 완료: [작업명]
- 다음 작업: [작업명]

### 컨텍스트
[context.md의 다음 세션 힌트]

### 진행
[다음 작업 시작]
```

## 주의사항
- dev/active/에 여러 작업이 있으면 어떤 것을 계속할지 확인
- context.md의 힌트를 반드시 확인
