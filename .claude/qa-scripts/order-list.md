## API 엔드포인트

| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/api/orders/with-dispatch` | 목록 조회 |
| GET | `/api/orders/with-dispatch/:id` | 상세 조회 |
| PATCH | `/api/orders/:id/status` | 주문 상태 변경 |

| POST | `/api/orders/batch` | 일괄 취소/수락 |
| POST | `/api/orders/:id/settlement` | 운송 마감 (정산 생성) |
| GET | `/api/companies` | 거래처 검색 |

| POST | `/api/orders/:id/memos` | 메모 생성 |
| PATCH | `/api/orders/:id/memos/:memoId` | 메모 수정 |

## 테스트 시나리오

### 1. 페이지 초기 로드

```
1. http://localhost:3000/broker/order/list 접속
2. 확인사항:
   - [ ] 로딩 스피너 표시 후 목록 렌더링
   - [ ] 기본 탭: "요청" 선택됨
   - [ ] 마지막 업데이트 시간 표시
   - [ ] 목록 데이터 정상 출력
   - [ ] 페이지네이션 표시 (데이터 있을 때)
```

### 2. 탭 전환

```
각 탭 클릭:
- [ ] 요청 탭 → flowStatus 필터, 체크박스 컬럼 표시
- [ ] 진행중 탭 → 기사/정보망/담당자/운송비 컬럼 표시
- [ ] 완료 탭 → 완료된 화물만 조회
- [ ] 전체 탭 → 모든 화물 조회

엣지케이스:
- [ ] 빈 목록일 때 적절한 메시지 표시
- [ ] 탭 전환 시 페이지 1로 리셋되는지
- [ ] 탭 전환 시 필터 상태 유지 여부
```

### 3. 뷰 모드 전환

```
우측 상단 토글:
- [ ] 테이블 보기 ↔ 카드 보기 전환
- [ ] 전환 시 데이터 유지
- [ ] 현재 뷰 모드 표시 (하이라이트)
```

### 4. 자동 갱신 / 수동 새로고침

```

- [ ] 새로고침 중 아이콘 회전 애니메이션

```

- [ ] 키워드 입력 후 Enter → 검색 실행
