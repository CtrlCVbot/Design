# Claude Design 결과 검수 체크리스트

## 검수 목적

Claude Design 결과가 보기 좋게 바뀌었더라도, 기존 업무 흐름과 필드가 빠지면 실패입니다. 이 체크리스트는 `wireframe`과 `high fidelity` 결과가 현재 확정한 B안에 맞는지 확인하기 위한 기준입니다.

## 1단계 Wireframe 검수

| 항목 | 통과 기준 | 상태 |
| --- | --- | --- |
| 화면 구조 | 상단 입력/수정 영역 + 하단 목록 구조가 유지됨 | 미검수 |
| 2열 구조 | 좌측 오더 입력, 우측 배차·상하차·화주 조건이 유지됨 | 미검수 |
| 운송 구간 | 상차지와 하차지가 좌측 최상단에 있음 | 미검수 |
| 화물 운송정보 | 톤수, 차종, 도착, 대수, 품목, 운송료, 수수료, 합계가 함께 있음 | 미검수 |
| 화물 요약 | 화물정보 요약, 중량, 증빙 옵션이 별도 그룹으로 있음 | 미검수 |
| 차주 정보 | 차주명, 차량번호, 차주 연락처가 하나로 묶임 | 미검수 |
| 화주 정보 | 의뢰자명, 구분, 사업자번호, 도착 연락처가 화주 정보로 묶임 | 미검수 |
| 수수료율 정보 | 주선약관 수수료율이 수수료 금액과 분리됨 | 미검수 |
| 액션 바 | 신규, 등록, 수정, 복사, 취소, 배차취소가 입력 영역 아래에 있음 | 미검수 |
| 목록 컨트롤 | 날짜, 상태 탭, 검색, 페이징, 정렬, 표시 설정이 있음 | 미검수 |
| 목록 테이블 | 화물ID, 처리시간, 상태, 상하차지, 운임, 차주/차량, 정산 컬럼이 있음 | 미검수 |

## 2단계 High Fidelity 검수

| 항목 | 통과 기준 | 상태 |
| --- | --- | --- |
| 기존 사용자 적응 | 원본 화면과 큰 배치가 비슷함 | 미검수 |
| 정보 밀도 | 핵심 필드가 과하게 숨겨지지 않음 | 미검수 |
| 시각적 그룹 | 섹션 경계가 명확하지만 과한 카드 UI가 아님 | 미검수 |
| 상태 표현 | 접수/완료/취소/선택 상태가 색상과 텍스트로 구분됨 | 미검수 |
| 위험 액션 | 화물취소, 배차취소가 일반 액션과 분리됨 | 미검수 |
| 금액 흐름 | 운송료, 수수료, 합계가 함께 보임 | 미검수 |
| 수수료율 | 주선약관 수수료율이 보조 정보로 분리됨 | 미검수 |
| 경고 문구 | 시간 입력 금지, 사업자번호 필수 경고가 관련 필드 근처에 있음 | 미검수 |
| 테이블 가독성 | 핵심 컬럼이 먼저 보이고 선택 행이 명확함 | 미검수 |
| 반응형 | Desktop, Tablet, Mobile 대응안이 있음 | 미검수 |

## 필드 누락 금지 목록

| 그룹 | 누락 금지 항목 |
| --- | --- |
| 운송 구간 | 상차지, 하차지, 주소 검색 |
| 화물 운송정보 | 톤수, 광폭, 차종, 도착, 대수, 품목, 운송비구분, 운송료, 수수료, 합계 |
| 화물 요약·중량·증빙 | 화물정보 요약, 중량, 전자세금계산서, 인수증 우편발송, 이사화물, 안내문, 산재보험 주의사항 |
| 차주 정보 | 차주명, 복지, 차량번호, 차주 연락처 |
| 배차 유형 | 독차, 혼적, 긴급, 왕복, 예약, 경유 |
| 상차 조건 | 상차방법, 상차일시, 상차지 전화, 초기화 |
| 하차 조건 | 하차방법, 하차일시, 보험료, 당착/내착 |
| 화주 정보 | 의뢰자명, 의뢰자구분, 의뢰자 사업자번호, 도착 연락처, 조회, 점검 |
| 수수료율 정보 | 주선약관 수수료율 표 |
| 목록 | 화물ID, 처리시간, 상태, 상차지, 하차지, 화물정보, 운송료, 수수료, 차주, 차량, 사업자번호, 계산서 정보 |

## 실패로 보는 결과

| 실패 유형 | 이유 |
| --- | --- |
| 상단 입력 영역과 하단 목록이 분리된 별도 앱처럼 바뀜 | 기존 사용자 위치 기억을 깨뜨림 |
| 화면이 대시보드형 카드 레이아웃으로 크게 바뀜 | 원본 업무 흐름과 달라짐 |
| 핵심 입력 필드가 탭이나 접힘 영역 안으로 숨음 | 빠른 오더 입력이 어려워짐 |
| `수수료` 금액이 운송료/합계와 떨어짐 | 금액 흐름을 이해하기 어려움 |
| 화주 정보와 차주 정보가 섞임 | 업무 주체가 혼동됨 |
| 목록 테이블이 너무 단순화됨 | 기존 목록 기반 업무를 수행하기 어려움 |

## 결과 피드백 작성 양식

| 항목 | 작성 예 |
| --- | --- |
| 통과 | B안의 2열 구조와 하단 목록이 유지됨 |
| 수정 필요 | `수수료`가 우측 수수료율 정보로 이동되어 운송료 흐름이 끊김 |
| 보류 | `화물정보 요약` 자동 생성 여부는 실제 프로그램 확인 필요 |
| 재실행 필요 | 디자인이 카드형 대시보드처럼 바뀌어 기존 화면과 너무 다름 |
