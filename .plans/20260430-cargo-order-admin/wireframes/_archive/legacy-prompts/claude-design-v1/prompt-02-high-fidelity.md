# Prompt 02. Claude Design High Fidelity

Wireframe 결과가 B안 구조를 충분히 유지한다고 판단되면, 같은 Claude Design 세션에서 아래 내용을 붙여넣고 `High Fidelity` 모드로 전환하세요.

```md
방금 만든 Wireframe을 기준으로 High Fidelity 디자인으로 발전시켜주세요.

작업 모드:
- High Fidelity 모드로 전환해주세요.
- Wireframe에서 확정한 구조와 필드 위치를 유지해주세요.
- 원본 화면의 큰 골격을 유지하되, 현대적인 업무용 UX/UI로 개선해주세요.
- SaaS 랜딩페이지나 마케팅 화면처럼 만들지 말고, 실제 운영자가 하루 종일 사용하는 고밀도 데스크톱 업무 화면처럼 설계해주세요.

기준 화면:
- SCR-ID: SCR-CARGO-ORDER-001
- 화면명: 화물 오더 접수/수정 관리 화면
- 핵심 사용자: 화물 오더를 빠르게 접수, 수정, 배차, 정산 처리하는 운영자
- 사용자 특성: 기존 화면 위치에 익숙함. 큰 배치 변화에 불편함을 느낌.

High Fidelity 핵심 목표:
- 현재 화면의 상단 입력/수정 + 하단 목록 구조를 유지합니다.
- 좌측 오더 입력 + 우측 배차·상하차·화주 조건의 2열 구조를 유지합니다.
- 오래된 Windows 프로그램 느낌은 줄이고, 현대적인 업무 도구처럼 정돈합니다.
- 정보 밀도는 유지하되, 섹션 경계와 상태, 우선순위를 명확히 합니다.

확정 정보 구조:
- 좌측: 운송 구간, 화물 운송정보, 화물 요약·중량·증빙, 차주 정보
- 우측: 배차 유형, 상차 조건, 하차 조건, 화주 정보, 수수료율 정보
- 중간: 액션 바
- 하단: 목록 컨트롤, 화물 목록 테이블

시각 디자인 방향:
- 전체 톤은 차분하고 업무적인 UI로 해주세요.
- 색상은 상태 의미를 중심으로 사용해주세요.
- 접수는 파란색 계열, 완료는 중립 회색/검정 계열, 취소와 위험 액션은 빨간색 계열로 구분해주세요.
- 경고 문구는 빨간색만 남발하지 말고, 경고 아이콘과 작은 안내 문구로 정리해주세요.
- 배경은 밝은 회색/화이트 기반으로 하고, 입력 그룹은 얇은 경계와 충분한 간격으로 나눠주세요.
- 과한 그라데이션, 장식용 이미지, 큰 카드형 랜딩페이지 구성은 사용하지 마세요.

컴포넌트 방향:
- 상차지/하차지는 주소 검색 입력 그룹으로 보여주세요.
- 톤수, 차종, 도착 조건, 운송비구분, 화주 구분은 선택형 컨트롤로 보여주세요.
- 독차/혼적은 세그먼트 컨트롤처럼 보이게 해주세요.
- 긴급/왕복/예약/경유는 작은 체크 옵션 또는 토글 그룹으로 표현해주세요.
- 상차방법/하차방법은 여러 개 선택 가능한 체크 그룹으로 표현해주세요.
- 상차일시/하차일시는 빠른 선택 칩과 날짜/시간 입력을 함께 보여주세요.
- 운송료, 수수료, 합계는 금액 입력/계산 영역으로 묶어주세요.
- 수수료 금액은 좌측 `화물 운송정보`에 있어야 합니다.
- `주선약관 수수료율`은 우측 `수수료율 정보`로 분리해주세요.
- 차주 정보는 차주명, 차량번호, 연락처를 한 줄에서 빠르게 확인할 수 있게 해주세요.
- 화주 정보는 의뢰자명, 구분, 사업자번호, 도착 연락처, 조회, 점검을 포함해주세요.

액션 바:
- 주요 액션: 신규(F3), 화물등록, 화물수정
- 보조 액션: 화물복사, 정보변경, 화물위치, 위탁증
- 위험 액션: 화물취소, 배차취소
- 출력/정산 액션: 엑셀, 인쇄, 계산서, 인수증, 정산처리, 서명확인
- 위험 액션은 실수 클릭을 줄이도록 시각적으로 분리해주세요.
- 기존 사용자가 버튼을 찾기 어렵지 않도록 액션 바는 기존처럼 입력 영역 아래에 유지해주세요.

목록 영역:
- 날짜, 상태 탭, 검색, 페이징, 정렬, 컬럼 설정, 글씨 크기 조절을 목록 위에 배치해주세요.
- 상태 탭은 전체/접수/완료/취소를 명확히 보여주세요.
- 테이블은 고밀도 정보를 유지하되, 컬럼 헤더를 읽기 쉽게 정리해주세요.
- 상태, 화물ID, 처리시간, 상차지, 하차지, 운송료는 핵심 컬럼처럼 먼저 보이게 해주세요.
- 선택 행은 배경색과 왼쪽 표시로 명확히 구분해주세요.
- 취소 행은 빨간색만 과도하게 칠하지 말고, `취소` 배지와 텍스트 색으로 구분해주세요.

반응형:
- Desktop 1280px 이상이 1차 대상입니다.
- Desktop에서는 상단 2열 + 하단 테이블을 반드시 유지해주세요.
- Tablet에서는 입력 그룹을 세로 스택으로 바꿔도 되지만, B안의 정보 묶음은 유지해주세요.
- Mobile에서는 전체 입력을 억지로 한 화면에 넣지 말고, 목록 중심 + 상세 편집 화면 또는 하단 시트 구조로 제안해주세요.

접근성:
- 상태를 색상만으로 구분하지 말고 텍스트 배지를 함께 사용해주세요.
- 입력 필드 라벨은 항상 보이게 해주세요.
- 위험 액션에는 명확한 라벨과 확인 흐름을 암시해주세요.
- 작은 버튼과 아이콘에는 tooltip이 있다고 가정하고 설계해주세요.
- 폰트는 업무용 화면에 적합하게 명확하고 읽기 쉬운 크기로 해주세요.

반드시 피해야 할 것:
- 기존 화면과 완전히 다른 대시보드/카드형 앱으로 재구성하지 마세요.
- 랜딩페이지처럼 큰 히어로나 홍보 영역을 만들지 마세요.
- 핵심 필드를 탭 뒤에 숨기지 마세요.
- 운송료/수수료/합계 흐름을 분리해서 찾기 어렵게 만들지 마세요.
- `수수료` 금액과 `주선약관 수수료율`을 같은 의미처럼 섞지 마세요.
- 화주 정보와 차주 정보를 혼동하지 마세요.

최종 산출물:
- Desktop High Fidelity 화면
- Tablet 대응안
- Mobile 대응안
- 상태별 테이블 행 예시: 접수, 완료, 취소, 선택됨
- 버튼 상태 예시: 기본, hover, disabled, 위험 액션
- 간단한 디자인 시스템 요약: 색상, 타이포그래피, 입력 필드, 버튼, 배지, 테이블
```
