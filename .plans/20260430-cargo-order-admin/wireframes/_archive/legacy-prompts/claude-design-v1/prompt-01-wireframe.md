# Prompt 01. Claude Design Wireframe

아래 내용을 Claude Design에 붙여넣고 `Wireframe` 모드로 실행하세요.

```md
화물 오더 접수/수정 관리 화면의 Wireframe을 설계해주세요.

작업 모드:
- Wireframe 모드로 실행해주세요.
- 시각 디자인, 브랜드 색상, 장식 요소보다 레이아웃, 정보 구조, 컴포넌트 배치가 우선입니다.
- 기존 화면과 큰 배치가 달라지면 안 됩니다.

기준 화면:
- SCR-ID: SCR-CARGO-ORDER-001
- 화면명: 화물 오더 접수/수정 관리 화면
- 화면 성격: 상단에서 개별 화물 오더를 입력/수정/확인하고, 하단에서 등록된 화물 목록을 조회/처리하는 고밀도 업무 화면입니다.
- 원본 화면은 오래된 데스크톱 업무 프로그램 스타일이며, 기존 사용자가 위치를 기억하고 빠르게 입력하는 화면입니다.

- 현재 화면의 큰 골격을 유지해주세요.

- 좌측 오더 입력 + 우측 배차·상하차·화주 조건의 2열 구조를 유지해주세요.
- 개선은 필드 재배치가 아니라 업무 의미에 맞는 그룹화, 라벨 정리, 액션 우선순위 정리 중심으로 해주세요.

- Desktop 1280px 이상: 기존처럼 상단 2열 입력/조건 + 하단 목록을 유지
- Tablet 768px-1279px: 상단 입력 그룹은 세로 스택 가능, 목록은 핵심 컬럼 우선
- Mobile 767px 이하: 목록 중심, 상세 입력은 별도 화면 또는 하단 시트로 표현
```
