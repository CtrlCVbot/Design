## 데이터 있는 상태의 표시 규칙

### 기본 2행 구조

운송구간 섹션은 아래처럼 세로 2행으로 구성합니다.

1. 상차지 행
2. 하차지 행

각 행은 다음 정보를 포함합니다.

상차지 행:

- 상차지명
- 주소
- 상차일시
- 상차방법
- 특이사항 표시

하차지 행:

- 하차지명
- 주소
- 하차일시
- 하차방법
- 특이사항 표시

### 표시 예시

```txt
[상차지] 코덱트 후진입차

[하차지] 무갑리 현장
경기 광주시 초월읍 무갑리
당일 · 지게차 · 특이사항 1
```

## 주소 표시 규칙

- 기본 화면에서는 상세주소를 제외한 주소만 보여주세요.
- 상세주소는 펼치기 영역에서 보여주세요.
- 예: `경기 여주시 산북면 후리`
- 상세주소 예: `18-1 코덱트 후진입차`

## 상하차지명 표시 규칙

- 상차지명 또는 하차지명이 있으면 표시합니다.
- 값이 없으면 해당 줄은 아예 보여주지 않습니다.
- 빈 줄이나 `-` 같은 placeholder는 쓰지 않습니다.

## 일시 표시 규칙

상차일시/하차일시는 아래처럼 간단히 표현합니다.

- `지금`
- `당일`
- `내일`
- 그 외 날짜/시간이 있으면 날짜 또는 시간을 표시
- 가능한 한 짧게 표시

예:

```txt
지금
당일
내일
04-30 16:00
```

## 방법 표시 규칙

상차방법/하차방법은 일시와 같은 줄에서 짧게 보여주세요.

예:

```txt
지금 · 지게차
당일 · 수작업
04-30 16:00 · 크레인
```

## 라벨 최소화 규칙

컴팩트한 명세서 형태가 목표이므로, 반복 라벨을 과하게 붙이지 말아주세요.

좋은 예:

```txt
[상차지] 코덱트 후진입차
경기 여주시 산북면 후리
지금 · 지게차 · 특이사항 2
```

피해야 할 예:

```txt
상차지명: 코덱트 후진입차
주소: 경기 여주시 산북면 후리
상차일시: 지금
상차방법: 지게차
```

단, 구분을 위해 `상차지`, `하차지` 정도의 큰 제목은 유지합니다.

## 특이사항 표시

특이사항이 있으면 텍스트를 길게 노출하지 말고, 아이콘 또는 숫자 배지로 표시해주세요.

예:

```txt

주의 1
메모 3
```
