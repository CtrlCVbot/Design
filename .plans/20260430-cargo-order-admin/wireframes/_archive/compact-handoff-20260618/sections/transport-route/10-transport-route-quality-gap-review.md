# Quality Gap Review: 운송구간 추가 보강 필요점

## 목적

이 문서는 `화주 정보` 섹션의 1차 완료 수준과 비교해, `운송구간` 섹션에서 추가 보강이 필요한 지점을 정리한다.

운송구간은 이미 D `compact-hybrid` 기준과 B 통합본 반영이 완료되어 있다. 이번 review는 구조를 뒤집기 위한 것이 아니라, 완성도를 높이기 위해 빠진 상태 화면과 interaction 기준을 찾기 위한 것이다.

## UX/UI 보완 제안

### 1. 주소는 inline edit보다 검색/적용 flow

주소는 단순 텍스트가 아니라 좌표, 행정주소, 거리 계산과 연결된다. 따라서 적용된 주소를 바로 input으로 바꾸기보다, `상차지 변경` 또는 `하차지 변경` 액션으로 주소 검색 다이얼로그를 다시 여는 편이 안전하다.

| 수정 대상 | 권장 방식 | 이유 |
| --- | --- | --- |
| 지명 | 주소 검색/적용 다이얼로그 | 장소명과 주소가 함께 바뀔 가능성 |
| 행정주소 | 주소 검색/적용 다이얼로그 | 좌표/거리 계산과 연결 |
| 상세주소 | 펼침 inline edit | 좌표가 바뀌지 않는 보조 정보 |
| 담당자/연락처 | 펼침 inline edit | 현재 오더 적용값 중심으로 빠른 수정 가능 |
| 특이사항 요약 | 펼침 inline edit | 현장 메모 성격 |

### 2. CTA 흡수 전환은 상차/하차 단위로 분리

화주 정보는 하나의 row를 적용하면 끝나지만, 운송구간은 상차와 하차가 독립적으로 채워진다. CTA 흡수 전환도 섹션 전체 1개 버튼이 아니라 상차/하차 각각에 맞춰야 한다.

| 상태 | 표시 |
| --- | --- |
| 둘 다 없음 | 상차지 주소 입력 CTA, 하차지 주소 입력 CTA |
| 상차만 있음 | 상차 row + 하차지 주소 입력 CTA |
| 하차만 있음 | 상차지 주소 입력 CTA + 하차 row |
| 둘 다 있음 | D안 닫힘 요약 |

### 3. 계산 메타는 확정값과 대기값을 시각적으로 구분

거리와 기준 금액은 운영 판단에 영향을 줄 수 있다. `톤수·차종 선택 후 계산` 상태는 회색 대기 chip으로, `거리 15.8km · 기준 105,000원`은 확정값 chip으로 구분한다.

| 상태 | tone |
| --- | --- |
| 주소 입력 후 계산 | 대기 |
| 톤수·차종 선택 후 계산 | 대기 |
| 차량 조건 변경됨 · 다시 계산 | 경고 |
| 거리/기준 금액 표시 | 확정 |

### 4. 배차 유형 편집은 오른쪽 메타에 머물게 함

상차/하차 row 안에 `독차`, `예약` 같은 칩을 다시 넣으면 중복이 생긴다. 편집이 필요해도 오른쪽 메타 안에서 popover로 처리하는 편이 현재 D안과 충돌이 적다.

## B 통합본 반영 전 체크리스트

| 체크 | 기준 |
| --- | --- |
| `route-section-wide` 유지 | 운송구간 full-width 구조가 깨지지 않아야 함 |
| 상차/하차 2행 유지 | 좌우 2열로 되돌아가지 않아야 함 |
| CTA 전환 일관성 | 화주 정보/화물 운송정보의 CTA 흡수형과 같은 사용자 경험 |
| 주소 검색 분리 | 지명/행정주소는 검색 다이얼로그에서 변경 |
| 상세 edit 제한 | 상세주소, 담당자/연락처, 특이사항만 inline edit |
| 계산 의존성 표시 | 톤수/차종 미선택 상태에서 금액 확정 표현 금지 |
| 중복 칩 금지 | 상차/하차 row 안에 배차 유형을 반복하지 않음 |
| 검증 | desktop/mobile overflow, row height, 메타 높이 정렬 확인 |

## 결론

운송구간은 구조 자체는 1차 완료 상태다. 다만 화주 정보와 같은 완성도로 보려면 `주소 입력 전`, `주소 검색/적용`, `부분 입력`, `적용 후 변경`, `상세 inline edit`, `계산 메타 의존성` 상태를 추가로 잠가야 한다.

다음 HTML 작업은 기존 D안을 교체하는 작업이 아니라, `transport-route-d-workflow.html`에 추가 상태를 붙이고 B 통합본에는 확정된 상태만 흡수하는 순서가 적합하다.
