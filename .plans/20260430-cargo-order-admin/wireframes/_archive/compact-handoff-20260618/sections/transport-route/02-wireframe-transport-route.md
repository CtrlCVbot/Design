## SCR-TR-001 데이터 있음 요약

1차 기준안은 D `compact-hybrid`이다. A/B/C는 비교와 회귀 확인을 위한 보조안으로 유지한다.

```txt
┌──────────────────────────────────────────────┐
│ ① 운송 구간                         상세 펼침 │
├──────────────────────────────────────────────┤
│ [상차지] 코덱트 후진입차              특이사항 2 │
│ 경기 여주시 산북면 후리                         │
│ 지금 · 지게차                                  │
├──────────────────────────────────────────────┤
│ [하차지] 무갑리 현장                  특이사항 1 │
│ 경기 광주시 초월읍 무갑리                       │
│ 당일 · 지게차                                  │
├──────────────────────────────────────────────┤
│ 대표 선택  독차(기본) · 옵션 없음 · 경유 없음   │
└──────────────────────────────────────────────┘
```

### 규칙

- 상차지와 하차지는 세로 2행이다.
- 주소는 상세주소를 제외한 행정주소만 표시한다.
- 지명 값이 없으면 지명 줄은 표시하지 않는다.
- `지금 · 지게차`처럼 일시와 방법은 한 줄로 묶는다.
- 특이사항은 긴 텍스트 대신 작은 배지로 표시한다.
- 거리와 기준 금액은 이 요약 행 안에서 확정값처럼 강조하지 않는다. 주소가 적용되어도 화물 운송정보의 `톤수`와 `차종`이 선택되어야 더 정확하게 표시된다.
- 배차 유형은 상차/하차 행에 반복하지 않고 운송구간 전체에 걸리는 칩 묶음으로 표시한다.
- `독차`와 `혼적`은 동시에 켜지지 않는 대표 선택이며, 초기 기본값은 `독차`이다.
- `긴급`, `왕복`, `예약`은 각각 독립 멀티선택 옵션이며, 세 옵션이 모두 꺼져 있을 수 있다.

### 보조 메타 표시 규칙

```txt
보조 메타: 거리 계산 대기 · 기준 금액 계산 대기
조건: 상차/하차 주소 + 톤수 + 차종 선택 후 정밀 표시
배차 유형: 독차(기본) · 예약 · 경유 없음
```

| 상태 | 보조 메타 표시 |
| --- | --- |
| 주소 없음 | `주소 입력 후 계산` |
| 주소 있음, 톤수/차종 없음 | `톤수·차종 선택 후 계산` |
| 주소 있음, 톤수/차종 있음 | `거리 15.8km · 기준 금액 105,000원` |
| 재계산 필요 | `차량 조건 변경됨 · 다시 계산` |

### 배차 유형 표시 규칙

```txt
배차 유형: 독차(기본) · 긴급 · 왕복 · 예약 · 경유 1
```

| 상태 | 표시 |
| --- | --- |
| 기본 대표 선택 | `독차(기본)` |
| 대표 전환 | `혼적`을 대표 유형으로 표시. `독차`와 동시 선택하지 않음 |
| 긴급/왕복/예약 | 선택된 옵션만 칩으로 추가. 모두 미선택이면 옵션 칩을 숨기거나 `옵션 없음`으로 약하게 표시 |
| 경유 있음 | `경유 1`, `경유 2`처럼 수량 표시 |
| 초기 입력 전 | `배차 유형 선택 전` 문구는 유지 가능하되, 실제 대표 기본값은 `독차`로 초기화 |

### D안: B+C 병합형 `compact-hybrid`

```txt
┌ 상차 │ 코덱트 후진입차 │ 경기 여주시 산북면 후리 │ 지금/지게차 │ 특이사항 2 │ 펼침 ┐
│      │ 상세주소: 18-1 ... │ 김상차 / 010... │ 진입로 협소              │
└──────────────────────────────────────────────────────────────────────┘

┌ 하차 │ 무갑리 현장 │ 경기 광주시 초월읍 무갑리 │ 당일/지게차 │ 특이사항 1 │ 펼침 ┐
│      │ 상세주소: 554-7 │ 박하차 / 010... │ 지게차 대기                  │
└──────────────────────────────────────────────────────────────────────┘
```

D안은 B안의 가로 명세 구조를 유지하되 C안처럼 카드 높이와 기본 라벨 노출을 줄인다. 접힘 상태에서는 값만 보이고, 상세 펼침 상태에서만 `상세주소`, `담당자 / 연락처`, `특이사항 요약` 라벨을 표시한다.

### 비교 판단

| 안 | 장점 | 주의점 |
| --- | --- | --- |
| A `sheet-row` | 가장 컴팩트하고 `Wireframe Spec Form.html`과 일관됨 | 비교 근거로 유지 |
| B `sheet-card` | 카드 구분감과 명세표 가로 흐름을 함께 확보 | 비교 근거로 유지 |
| C `stacked-card` | 읽기 쉽고 모바일 친화적 | 운영자가 가로 비교하기에는 정보 밀도가 낮음 |
| D `compact-hybrid` | B안의 정보 밀도와 C안의 가벼운 카드감을 병합 | 1차 기준안. 펼침 인터랙션이 명확해야 상세 라벨을 놓치지 않음 |

D안을 포함한 모든 표시안은 상하차 행 아래에 동일한 전체 조건 메타를 둔다.

```txt
전체 조건: 독차(기본) · 예약 · 경유 없음 · 톤수·차종 후 계산
```

### B 통합본 full-width 적용

`Cargo Order Wireframe B Original Tone.html`의 실제 적용본에서는 D안 운송구간을 좌우 2컬럼 전체 폭으로 확장한다. 기존 우측 컬럼의 `5. 배차 유형`, `6. 상차 조건`, `7. 하차 조건`은 독립 섹션으로 두지 않고 아래 구조로 흡수한다.

```txt
┌ 1 운송 구간 ───────────────────────────────────────────────────────────────────────────────┐
│ 상차 │ 코덱트 후진입차 │ 경기 여주시 산북면 후리 │ 지금/지게차 │ 특이사항 2 │ 펼침 │ 배차 유형 │
│ 하차 │ 무갑리 현장     │ 경기 광주시 초월읍 무갑리 │ 당일/지게차 │ 특이사항 1 │ 펼침 │ 계산 메타 │
└─────────────────────────────────────────────────────────────────────────────────────────────┘
```

오른쪽 메타 영역은 상차/하차 2행 높이에 맞춰 `배차 유형`과 `계산 메타`만 표시한다. `상하차 조건` 메타는 summary의 `지금/지게차`, `당일/지게차`와 중복되므로 제거한다. 펼침 영역은 `상세주소`, `담당자 / 연락처`, `특이사항 요약`만 표시한다. 상차일시와 상차방법을 선택하는 화면은 별도 화면으로 분리한다.

## SCR-TR-002 빈 상태

```txt
┌──────────────────────────────────────────────┐
│ A 운송 구간                         입력 전    │
├──────────────────────────────────────────────┤
│ ① 상차                                      │
│ 주소 / 상세      ───────────────             │
│ 일시             ─────────                   │
│ 방법             ─────────                   │
│ 담당자           ─────────                   │
├──────────────────────────────────────────────┤
│ ② 하차                                      │
│ 주소 / 상세      ───────────────             │
│ 일시             ─────────                   │
│ 방법             ─────────                   │
│ 담당자           ─────────                   │
├──────────────────────────────────────────────┤
│ 상차지와 하차지를 입력하면 운송구간이 생성됩니다. │
└──────────────────────────────────────────────┘
```

### 규칙

- `04 Wireframe Spec Form.html`의 폼시트/빈칸 감각을 따른다.
- 주소가 없으면 거리와 운송료 요약은 계산 불가 상태로 둔다.
- 주소가 있어도 화물 운송정보의 `톤수`와 `차종`이 없으면 기준 금액은 확정 표시하지 않는다.
- 사용자의 다음 행동은 `주소 검색 또는 직접 입력`으로 안내한다.

## SCR-TR-003 펼침 상태

```txt
┌──────────────────────────────────────────────┐
│ ① 운송 구간                         접기       │
├──────────────────────────────────────────────┤
│ [상차지] 코덱트 후진입차              특이사항 2 │
│ 경기 여주시 산북면 후리                         │
│ 지금 · 지게차                                  │
│ ─ 상세 ─                                      │
│ 상세주소  18-1 코덱트 후진입차                 │
│ 담당자    김상차 / 010-1234-5678              │
│ 특이사항  진입로 협소, 10분 전 연락 필요        │
├──────────────────────────────────────────────┤
│ [하차지] 무갑리 현장                  특이사항 1 │
│ 경기 광주시 초월읍 무갑리                       │
│ 당일 · 지게차                                  │
│ ─ 상세 ─                                      │
│ 상세주소  554-7                               │
│ 담당자    박하차 / 010-9876-5432              │
│ 특이사항  지게차 대기 필요                     │
└──────────────────────────────────────────────┘
```

### 규칙

- 펼침 정보는 상차지와 하차지 각각의 행 아래에 붙는다.
- 상세주소, 담당자, 연락처, 특이사항은 기본 요약이 아니라 펼침에서만 보인다.
- 향후 실제 구현에서는 행별 `details` 또는 섹션 단위 토글을 사용할 수 있다.

## SCR-TR-D D안 워크플로우

```txt
1 주소 미적용
  └ 상차/하차 placeholder · 독차 기본 · 주소 입력 후 계산

2 상차 입력
  └ 상차 행만 D안 요약으로 표시 · 하차지 입력 전

3 닫힘 요약
  └ 상차/하차 2행 값 중심 표시 · 상세 라벨 숨김

4 상세 열림
  └ 상세주소 · 담당자/연락처 · 특이사항 요약 라벨 표시

5 계산 완료
  └ 톤수/차종 선택 후 거리 15.8km · 기준 금액 105,000원 표시
```

D안 워크플로우는 `transport-route-d-workflow.html`에서 확인한다.

## 반응형

| Viewport | 레이아웃 |
| --- | --- |
| Desktop `1280px+` | 전체 오더 화면은 2열일 수 있으나 운송구간 내부는 세로 2행 유지 |
| Tablet `768px-1279px` | 전체 화면이 1열로 접혀도 상차/하차 2행 유지 |
| Mobile `~767px` | 행 내부 메타 정보는 줄바꿈, 상차/하차 순서는 유지 |
