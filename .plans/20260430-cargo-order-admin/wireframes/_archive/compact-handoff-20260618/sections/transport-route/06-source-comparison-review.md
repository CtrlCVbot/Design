# 운송구간 Source Comparison Review

## 목적

`transport-route-section-variants.html`을 기존 기준 화면인 `Cargo Order Wireframe.html`, `Cargo Order Wireframe B Original Tone.html`과 비교해, 최종 통합 전에 추가로 반영해야 할 요소를 정리한다.

이번 리뷰는 운송구간 섹션 자체만 판단한다. 전체 오더 화면의 화물정보, 차주정보, 화주정보, 수수료율 정보는 운송구간에 직접 합치지 않고 연결 맥락만 점검한다.

## 최종 통합 권장안

운송구간 1차 기준안은 D `compact-hybrid`로 정리한다.

다만 전체 화면에 바로 합칠 때는 단독 variants 페이지처럼 큰 제목과 비교 섹션을 넣지 않는다. 실제 통합 화면에서는 다음 구조가 적합하다.

```text
1 운송 구간
┌────────────────────────────────────────────────────────────┐
│ 상차 │ 코덱트 후진입차 │ 경기 여주시 산북면 후리 │ 지금/지게차 │ 특이사항 2 │ 펼침 │
│ 하차 │ 무갑리 현장     │ 경기 광주시 초월읍 무갑리 │ 당일/지게차 │ 특이사항 1 │ 펼침 │
│ 펼침 │ 상세주소        │ 담당자/연락처             │ 특이사항 요약              │
└────────────────────────────────────────────────────────────┘
보조 메타: 배차 유형 독차(기본) · 예약 · 경유 없음
계산 메타: 톤수·차종 선택 후 거리/기준 금액 계산 · 실시간 운송료 조회 가능
```

화물 운송정보에서 `톤수`와 `차종`이 선택된 뒤에는 아래처럼 정밀값으로 바뀐다.

```text
보조 메타: 배차 유형 독차(기본) · 예약 · 경유 없음
계산 메타: 거리 15.8km · 기준 금액 105,000원 · 실시간 운송료 조회 가능
```

모바일에서는 D `compact-hybrid`를 유지하되, 행 내부는 필요한 경우 내부 가로 스크롤로 처리한다. A/B/C는 비교 근거로 보존한다.

## 다음 작업 체크리스트

1. D `compact-hybrid`는 전체 B 원본 톤 HTML의 1번 운송구간에 1차 통합 완료했다.
2. 운송구간 하단과 상단 subbar의 `거리`, `기준 금액`, `실시간 운송료` 표시 정책을 최종 결정한다. 현재 B 통합본은 `5톤 · 축카고 반영`과 `톤수·차종 변경 시 재계산`을 함께 표시한다.
3. 5번 `배차 유형`, 6번 `상차 조건`, 7번 `하차 조건`은 B 통합본에서 독립 섹션으로 제거하고 운송구간 내부로 병합 완료했다.
4. 펼침 상세에 `대기시간`과 `현장 연락처`를 넣을지 결정한다.
5. 운영자 검토 후 `07-cleanup-candidates.md`의 삭제 후보를 실제 삭제할지 결정한다.
