# Overview: 운송 관련 다이얼로그 최근 사용 리스트

## 사용자 가치

운영자는 같은 화주, 같은 현장, 같은 차종/품목 조합을 반복해서 입력하는 경우가 많습니다. `최근 사용` 리스트는 검색어나 필드값을 처음부터 다시 입력하지 않고, 바로 직전 또는 자주 반복되는 후보를 선택하게 해 입력 시간을 줄입니다.

| 대상 | 반복 입력 | 기대 효과 |
| --- | --- | --- |
| 상하차지 선택 다이얼로그 | 같은 화주의 상차 창고, 하차 현장, 담당자, 상세주소 | 검색 전 후보 선택으로 주소 입력 시간 단축 |
| 운송+품목 선택 다이얼로그 | `톤수`, `차종`, `품목`, `실중량` 조합 | 반복 조건을 한 번에 채워 금액 계산 전 단계 진입 단축 |

최근 사용 리스트는 자동 저장이나 자동 확정 기능이 아닙니다. 사용자가 선택한 후보를 다이얼로그 안의 미리보기 또는 입력값에 반영한 뒤, 기존 `적용` 버튼으로 확정합니다.

## 보조 영역 추가 원칙

기존 다이얼로그를 대체하지 않습니다.

| 원칙 | 기준 |
| --- | --- |
| 기존 검색 유지 | 주소 검색 결과 선택은 계속 주 흐름입니다. 최근 장소는 기존 검색 결과 영역의 초기 상태로만 표시합니다. |
| 기존 입력 유지 | 운송+품목 다이얼로그의 `톤수`, `차종`, `대수`, `실중량`, `품목` 입력 컴포넌트는 유지합니다. |
| 확정 책임 유지 | `상차지 적용`, `하차지 적용`, `운송+품목 적용` 버튼이 실제 반영을 담당합니다. |
| 보조 밀도 유지 | 최근 리스트는 새 패널로 분리하지 않고, 기존 검색 결과 리스트가 들어가는 표면을 재사용합니다. |
| 개인정보 최소화 | 연락처와 업체 정보는 항목 선택 판단에 필요한 범위로 줄이고, 전체 연락처 원문 노출은 미리보기에서만 검토합니다. |

## 적용 차이

### 신규 접수 wizard

신규 접수에서는 기존 입력 단위를 wizard frame 안에서 순차 표시합니다. 따라서 최근 리스트는 현재 단계의 완료를 돕는 보조 후보로만 동작합니다.

| 단계 | 최근 리스트 표시 | 적용 버튼 |
| --- | --- | --- |
| 상차지 | 현재 `상차지` 단계에서 `상차 최근` 우선 | `다음: 하차지` 또는 단계별 적용 성격의 CTA |
| 하차지 | 현재 `하차지` 단계에서 `하차 최근` 우선 | `다음: 운송+품목` |
| 운송+품목 | `톤수 + 차종 + 품목 + 중량` 최근 조합 표시 | `다음: 금액` |

왼쪽 프로세스 패널 정책은 유지합니다. 최근 리스트를 추가해도 wizard 단계 목록, 완료/현재/대기 상태, 완료한 이전 단계로만 돌아가는 정책은 바꾸지 않습니다.

### 일반 수정 다이얼로그

`idle-edit` 상태 또는 `new-submitted` 이후 개별 수정에서는 기존 독립 다이얼로그 방식으로 동작합니다.

| 흐름 | 최근 리스트 동작 |
| --- | --- |
| 주소 개별 수정 | `상차지 변경`, `하차지 변경`, 행정주소 클릭에서 독립 주소 검색 다이얼로그를 열고 최근 장소를 표시 |
| 운송+품목 개별 수정 | `운송+품목 입력` 또는 `수정`에서 독립 입력 다이얼로그를 열고 최근 조합을 표시 |
| 확정 | 최근 항목 클릭 후에도 기존 `적용` 버튼을 눌러야 main row에 반영 |

## MVP 정책

| 항목 | MVP 기준 |
| --- | --- |
| 표시 개수 | 최근 선택순 10개 |
| 중복 제거 | 같은 장소 또는 같은 조합은 최신 사용 시각 기준 1개만 표시 |
| 주소 scope | 화주별 최근과 사용자별 최근을 모두 검토하되, 기본 노출은 현재 화주 기준을 우선 제안 |
| 운송+품목 scope | 조합 단위 최근 사용을 기본으로 함 |
| 후속 확장 | 톤수/차종/품목 개별 추천, 빈도순 추천, 팀 공유 추천은 후속 |

## 디자인 방향

| 항목 | 기준 |
| --- | --- |
| 시각 기준 | `results/html/Design System.html`의 shadcn 기준 |
| 재사용 패턴 | `btn`, `chip`, `badge`, `empty`, `summary-row`, `dialog` |
| 배치 | 주소 최근은 기존 검색 결과 영역의 초기 상태. 운송+품목 최근은 입력폼 옆의 보조 후보 |
| 색상 | 기존 neutral + brand 계열 안에서 유지 |
| 장식 | 새 색상 체계, hero형 구성, 과한 카드 장식 제외 |

## Visual Thesis

운영자가 이미 아는 반복 후보를 조용히 앞에 꺼내 두는, 밀도 높은 업무용 다이얼로그입니다.

## Content Plan

| 영역 | 역할 |
| --- | --- |
| 다이얼로그 제목 | 현재 작업이 `상차지`, `하차지`, `운송+품목` 중 무엇인지 명확히 표시 |
| 최근 사용 | 바로 선택 가능한 보조 후보 |
| 검색/입력 | 기존 주 입력 흐름 유지 |
| 미리보기 | 주소는 기존 다이얼로그의 미리보기 위치를 우선하고, 현재 UI 톤은 가능한 범위에서 유지 |
| footer | 취소와 적용 버튼으로 확정 책임 분리 |

## Interaction Thesis

| 상호작용 | 기준 |
| --- | --- |
| 최근 항목 선택 | 주소는 선택 미리보기, 운송+품목은 입력폼을 갱신 |
| 검색 실행 | 주소 검색어 입력 후 `조회` 또는 Enter 실행 시 같은 리스트 영역이 기존 검색 결과로 전환 |
| 검색 결과 선택 | 최근 선택값보다 나중에 선택한 검색 결과가 미리보기를 덮어씀 |
| 적용 | 미리보기 또는 입력폼의 현재 값만 기존 적용 버튼으로 확정 |
