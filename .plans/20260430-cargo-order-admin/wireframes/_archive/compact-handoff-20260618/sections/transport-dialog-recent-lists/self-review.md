## 자동 반영한 피드백

| 항목 | 반영 내용 |
| --- | --- |
| 최근 클릭 저장 오해 | 모든 문서에 `최근 클릭은 미리보기/입력값만 갱신`, `적용 버튼으로 확정` 원칙을 반복 기준으로 넣음 |
| 개인정보 과노출 | 최근 row에는 연락처를 숨기고, 미리보기에서만 필요한 범위로 표시하는 기준 추가 |
| 최근 리스트 오류 혼동 | 최근 리스트 error는 검색/입력 흐름을 차단하지 않는다고 명시 |
| 주소 최근 리스트 위치 | 최근 리스트를 별도 3열 패널이 아니라 기존 검색 결과 영역의 초기 상태로 정리 |
| 운송+품목 요약 중복 | 별도 선택 요약 패널 없이 입력폼에서 현재 선택값을 확인하도록 정리 |

## 남은 리스크

| 항목 | 수준 | 후속 확인 |
| --- | --- | --- |
| 화주별/사용자별 최근 scope | high | 실제 데이터 소유권과 운영자 권한 기준 확인 필요 |
| 실중량 자동값과 최근 조합 값 충돌 | medium | `톤수` 변경 시 저장된 `실중량` 유지/재계산 확인 필요 |
| 최근 기록 저장 시점 | medium | 적용 성공 후 기록할지, 최종 API 저장 후 기록할지 제품 정책 확인 필요 |

## 후속 확인 필요 항목

| 항목 | 질문 |
| --- | --- |
| 최근 장소 scope | 같은 화주 기준만 보여줄지, 사용자별 최근을 fallback으로 허용할지 |
| 개인정보 마스킹 | 미리보기에서도 연락처 전체 노출이 필요한지, 뒷자리 마스킹이 필요한지 |
| 최근 기록 저장 | `적용` 직후 기록할지, 최종 `화물 등록` API 성공 후 기록할지 |
| 개별 추천 확장 | 톤수/차종/품목 개별 추천을 언제 별도 backlog로 둘지 |
