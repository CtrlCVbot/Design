# 운송+품목 다이얼로그 최근 사용 리스트 기획

## 대상 흐름

기존 `화물 정보` 또는 `운송+품목 입력` 흐름을 대상으로 합니다.

| 기준 | 현재 흐름 |
| --- | --- |
| 호출 | `openCargoInput()` |
| 입력 | `톤수`, `차종`, `대수`, `실중량`, `품목` |
| 확정 | `운송+품목 적용` |
| 반영 | `applyCargo()` 후 `화물 정보`의 `운송` row와 `품목` row 갱신 |

최근 사용 리스트는 입력폼 옆의 보조 후보로 배치합니다. 최근 조합을 선택하면 입력폼 값을 채우지만, 실제 row 반영은 `운송+품목 적용` 버튼을 눌렀을 때만 발생합니다.

## 최근 사용 기준

MVP는 조합 단위 최근 사용을 기본으로 합니다.

| 필드 | 조합 포함 | 설명 |
| --- | --- | --- |
| `톤수` | 포함 | 기준 금액과 `실중량` 자동값에 영향 |
| `차종` | 포함 | 운송구간 기준 금액에 영향 |
| `품목` | 포함 | 화물 정보 요약에 반영 |
| `실중량` 또는 `중량` | 포함 | 사용자가 직접 수정한 값까지 반복될 수 있음 |
| `대수` | 포함 권장 | 기존 다이얼로그 필수값이므로 기본 1대라도 조합에 포함 |

초안 문구의 `톤수 + 차종 + 품목 + 중량`은 구현 단계에서 현재 target field 기준인 `톤수 + 차종 + 품목 + 실중량 + 대수`로 확장해도 됩니다. 문서에서는 사용자 가치가 큰 4개 조합을 기준으로 설명하고, 데이터 저장에는 `대수`를 함께 기록하는 안을 권장합니다.

## 부분 선택과 전체 조합 선택

| 선택 방식 | MVP 여부 | 설명 |
| --- | --- | --- |
| 전체 조합 선택 | MVP | 최근 row 클릭 시 `톤수`, `차종`, `대수`, `실중량`, `품목`을 한 번에 입력폼에 반영 |
| 부분 선택 | 제한적 | row 안의 `톤수만 적용`, `차종만 적용` 같은 세부 버튼은 MVP에서 제외 |
| 개별 추천 | 후속 | 톤수 추천, 차종 추천, 품목 추천을 필드별 suggest로 분리 |

부분 선택은 운영자가 현재 입력 중인 값을 덮어쓰는지 판단하기 어렵습니다. MVP에서는 전체 조합 선택을 명확히 하고, 이미 입력된 값이 있을 때는 선택 후 미리보기에서 덮어쓴 필드를 강조하는 방향이 안전합니다.

## 최근 조합 항목 표시 정보

| 정보 | 표시 위치 | 기준 |
| --- | --- | --- |
| 주요 조합 | row title | `5톤 · 축카고 · 장비 운송` |
| 중량 | row meta 또는 chip | `실중량 5.50톤` |
| 대수 | row meta | `1대` |
| 사용 시각 | muted meta | `어제 14:20`, `최근 3회` |
| 사용 출처 | chip | `이 화주`, `내 최근`, `최근` |
| 금액 | 원칙적으로 제외 | 금액 조건은 별도 다이얼로그이므로 혼동 방지 |

금액 조건은 `운송+품목` 최근 조합에 포함하지 않습니다. 같은 운송 조건이라도 청구/운송비용은 상황에 따라 달라질 수 있으므로, 이번 범위에서는 분리합니다.

## 다이얼로그 배치

```text
[운송+품목 입력]

최근 사용 조합
○ 5톤 · 축카고 · 장비 운송 | 실중량 5.50톤 | 1대 | 최근 3회
○ 11톤 · 윙바디 · 파렛트 화물 | 실중량 12.10톤 | 1대 | 지난주

입력
톤수 [5톤]  차종 [축카고]  대수 [1]  실중량 [5.50]
품목 [장비 운송]

[취소] [운송+품목 적용]
```

별도 `선택 요약` 패널은 두지 않습니다. 사용자는 최근 조합 선택 후 입력폼에서 반영된 값을 확인하고 `운송+품목 적용` 버튼으로 확정합니다.

## 선택 시 입력값 반영

| 이벤트 | 처리 |
| --- | --- |
| 최근 조합 클릭 | 입력폼 전체 값 갱신 |
| 사용자가 필드 수정 | 최근 selected 상태는 유지하되 `수정됨` chip 표시 |
| 톤수 변경 | 기존 규칙대로 `실중량` 자동값을 제안하되, 최근 조합 선택으로 들어온 사용자 수정값은 덮어쓰기 전 확인 필요 |
| 적용 클릭 | 현재 입력폼 값을 `운송` row와 `품목` row에 반영 |
| 취소 클릭 | 기존 row 값 유지 |

## 실중량 자동 계산과 최근 조합

| 상황 | 기준 |
| --- | --- |
| 최근 조합 선택 | 저장된 `실중량`을 우선 입력 |
| 선택 후 톤수 변경 | 변경한 톤수 기준 110% 자동값을 다시 제안 |
| 선택 후 실중량 직접 수정 | 직접 수정값을 유지 |
| 적용 | 입력폼의 현재 값이 기록 대상 |

## Empty / Loading / Error

| 상태 | 표시 | 동작 |
| --- | --- | --- |
| empty | `최근 사용 조합이 없습니다.` + 직접 입력 안내 | 입력폼은 계속 사용 가능 |
| loading | skeleton row 2~3개 또는 `최근 조합을 불러오는 중` | 입력폼은 사용 가능 |
| error | `최근 조합을 불러오지 못했습니다.` + `다시 시도` | `운송+품목 적용` 흐름은 차단하지 않음 |

## 데이터 정책 초안

| 항목 | 기준 |
| --- | --- |
| key | `shipperId + ton + vehicleType + itemName + weight + count` |
| dedupe | 같은 조합은 최신 사용 시각 기준 1개 |
| 정렬 | `lastSelectedAt desc` |
| limit | 10개 |
| 저장 시점 | 사용자가 `운송+품목 적용`으로 실제 row에 반영한 뒤 기록 |
| 후속 확장 | 필드별 추천, 빈도순 추천, 화주별/사용자별 scope toggle |

## 구현 인계 메모

| 항목 | 메모 |
| --- | --- |
| 기존 함수 보호 | `openCargoInput()`의 입력폼과 `applyCargo()` 확정 책임 유지 |
| 계산 의존성 | 최근 조합 선택으로 `톤수`, `차종`이 채워지면 운송구간 기준 금액 계산 가능성이 높아짐 |
| 요약 연결 | `품목`, `실중량`, `톤수`, `차종`, `대수`가 기존 `화물 정보 요약` 원천값으로 이어짐 |
| 개인정보 | 운송+품목 최근 항목에는 업체명, 담당자, 연락처를 표시하지 않음 |
