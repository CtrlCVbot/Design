## 확정 판단

- 다음 HTML variants의 기준 운송 조건은 `톤수`, `차종`, `대수`, `실중량`입니다.
- B 통합본 F안에서는 `품목`을 `운송+품목` 2번째 row로 배치합니다.
- `광폭`, `도착`은 기준 필드에서 제외합니다.
- `실중량`은 톤수 선택 시 110% 자동 입력 후 `0.00` 단위로 직접 수정할 수 있게 둡니다.
- 금액 조건에는 `결제방법`을 포함하고, 금액 입력 필드는 `청구비용`, `운송비용`, `수수료`, 결과 금액 순서로 둡니다.
- `청구비용` 입력 시 `운송비용`을 같은 금액으로 자동 입력합니다.
- `운송비용` 수정 시 `청구비용`은 유지합니다.
- `수수료` 입력 시 자동 `선착불`, `청구비용 0원 고정`, `운송비용 포커스` 규칙을 적용합니다.
- 기존 `합계`는 인수증에서 `수익`, 선착불에서 `차주운임`으로 대체합니다.
- B 통합본 HTML의 `3. 화물 운송정보`에는 F안 구조를 반영했고, 입력 전 CTA row와 적용 후 정보 row의 최종 밀도까지 검증했습니다.
