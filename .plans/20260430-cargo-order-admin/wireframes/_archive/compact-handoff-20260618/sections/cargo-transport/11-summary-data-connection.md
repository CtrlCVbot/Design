# Summary Data Connection: 3. 화물 운송정보 -> 4. 화물정보 요약

## 목적

이 문서는 `sections/cargo-summary-docs/06-cargo-transport-summary-data-contract.md`의 요약 데이터 계약을 `3. 화물 운송정보` 패키지 관점에서 연결합니다.

화물 운송정보는 원본 입력 영역이고, 화물정보 요약은 이 원본을 읽기 쉬운 문장으로 압축한 파생 영역입니다.

## 연결 원칙

| 원칙 | 내용 |
| --- | --- |
| source of truth | `톤수`, `차종`, `대수`, `실중량`, `품목`은 `3. 화물 운송정보`가 원본 |
| summary output | `4. 화물정보 요약`은 원본 값을 문장화한 결과 |
| 금액 제외 | `결제방법`, `청구비용`, `운송비용`, `수수료`, `수익/차주운임`은 기본 요약 문장에 넣지 않음 |
| 운송구간 보조 | 상하차 일시/방법은 `2. 운송 구간`에서 받아 요약 문장 앞뒤 조건으로 사용 |
| override 분리 | 사용자가 요약 문장을 직접 수정하면 원본 필드와 별도 override로 관리 |

## 화물 운송정보 필드별 연결

| 필드 | 요약 영향 | 연결 규칙 |
| --- | --- | --- |
| `품목` | 높음 | 문장의 핵심 물품명으로 사용 |
| `실중량` | 높음 | `5.5톤`처럼 실제 중량으로 표현 |
| `톤수` | 중간 | 실중량이 없을 때 대체하거나 차량 조건으로 보조 사용 |
| `차종` | 중간 | `축차`, `윙바디` 등 차량 조건으로 표현 |
| `대수` | 낮음 | 1대는 생략, 2대 이상만 문장에 포함 |
| `결제방법` | 없음 | 정산 정보이므로 요약 문장에 기본 미포함 |
| `청구비용` | 없음 | 정산 정보이므로 요약 문장에 기본 미포함 |
| `운송비용` | 없음 | 정산 정보이므로 요약 문장에 기본 미포함 |
| `수수료` | 없음 | 정산 정보이므로 요약 문장에 기본 미포함 |
| `수익/차주운임` | 없음 | 정산 결과이므로 요약 문장에 기본 미포함 |

## 상태 연결

| 화물 운송정보 상태 | 요약 상태 | 처리 |
| --- | --- | --- |
| `activation-ready` | `empty` | `화물 운송정보 입력 후 요약 생성` 표시 |
| `activated-empty` | `auto` 후보 | 입력값이 충분하면 자동 생성 |
| `selected` | `auto` | 원본 필드 변경 시 요약 자동 갱신 |
| `weight-auto-filled` | `auto` | 실중량 자동값을 요약에 반영 |
| `weight-overridden` | `auto` 또는 `stale` | auto 요약이면 재생성, override가 있으면 stale 처리 |
| 사용자 요약 수정 | `overridden` | 원본 필드는 바꾸지 않고 요약 override만 저장 |

## 구현 인계 기준

1. `3. 화물 운송정보`의 inline edit 저장 이벤트가 발생하면 요약 생성 함수를 호출합니다.
2. 요약 상태가 `auto`이면 `4. 화물정보 요약` 값을 즉시 갱신합니다.
3. 요약 상태가 `overridden`이면 자동 갱신하지 않고 `stale` 후보로 표시합니다.
4. 금액 조건 변경은 요약 재생성 트리거에서 제외합니다.
5. `품목`, `실중량`, `차종` 변경은 요약 재생성 우선순위가 높습니다.

## B 통합본 샘플 반영

| 항목 | 상태 |
| --- | --- |
| `품목` inline edit 후 요약 갱신 | 반영 완료 |
| `톤수` 변경 후 `실중량 110%`와 요약 중량 갱신 | 반영 완료 |
| 수동 요약 수정 후 원본 변경 시 `stale` 상태 전환 | 반영 완료 |
| 금액 조건 변경이 요약에 영향 없음 | 반영 완료 |
