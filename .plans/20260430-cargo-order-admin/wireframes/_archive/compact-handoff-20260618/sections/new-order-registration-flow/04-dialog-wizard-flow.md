# 다이얼로그 wizard 플로우

## 원칙

신규 접수 최초 입력은 wizard 다이얼로그로 진행한다.

기존 화면의 독립 다이얼로그는 화주, 주소, 운송+품목, 금액, 차주 입력 단위가 이미 분리되어 있다. 신규 접수에서는 같은 입력 단위를 순차 wizard로 묶고, 화물 수정에서는 기존 독립 다이얼로그 방식을 유지한다.

## wizard 단계

| 단계 | 제목 | 필수 여부 | 기존 화면 단서 | 완료 조건 |
| --- | --- | --- | --- | --- |
| 1 | 화주 정보 | 필수 | `openShipperLookup()` | 화주 선택 또는 필수 화주 입력 완료 |
| 2-1 | 상차지 | 필수 | `openAddrSearch('load')` | 상차지 주소와 필수 상세 정보 완료 |
| 2-2 | 하차지 | 필수 | `openAddrSearch('unload')` | 하차지 주소와 필수 상세 정보 완료 |
| 3 | 운송+품목 | 필수 | `openCargoInput()` | 톤수, 차종, 수량, 실중량 등 완료 |
| 4 | 금액 | 필수 | `openMoneyInput()` | 결제방법, 청구비용, 운송비용 완료 |
| 5 | 차주 정보 선택 | 선택 | `openDriverLookup()` | 선택 완료 또는 건너뛰기 |

## 왼쪽 프로세스 패널

신규 접수 wizard에서만 다이얼로그 왼쪽에 프로세스 패널을 표시한다.

| 표시 상태 | 정책 |
| --- | --- |
| `new-wizard-active` | 표시 |
| `new-required-complete` | 표시 |
| `new-driver-optional` | 표시 |
| `idle-edit` | 숨김 |
| `new-submitted` | 숨김 |

## 프로세스 패널 구조

| 요소 | 설명 |
| --- | --- |
| 플로우 제목 | `신규 접수` |
| 단계 목록 | 화주 정보, 상차지, 하차지, 운송+품목, 금액, 차주 정보 |
| 단계 상태 | 현재, 완료, 대기, 선택 |
| 필수 표시 | 1~4단계는 필수, 차주 정보는 선택 |
| 진행률 | 필수 단계 기준 완료 수를 우선 표시 |

프로세스 패널은 navigation처럼 보일 수 있지만 MVP에서는 단계 이동을 제한한다.

사용자는 완료한 이전 단계로만 돌아갈 수 있고, 아직 완료하지 않은 다음 필수 단계로 건너뛰지는 않는다. 차주 정보는 금액 완료 후에만 진입할 수 있다.

## 다이얼로그 footer 정책

| 단계 | 주요 버튼 | 보조 버튼 |
| --- | --- | --- |
| 화주 정보 | `다음: 상차지` | `취소` |
| 상차지 | `다음: 하차지` | `이전`, `취소` |
| 하차지 | `다음: 운송+품목` | `이전`, `취소` |
| 운송+품목 | `다음: 금액` | `이전`, `취소` |
| 금액 | `화물 등록 완료`, `차주 정보로 이동` | `이전`, `취소` |
| 차주 정보 | `화물 등록 완료` | `건너뛰기`, `이전` |

`취소`는 신규 접수 작성 중단이므로 확인 다이얼로그가 필요할 수 있다. MVP에서는 문구와 동작만 정의하고, 입력값 보존/복구 정책은 후속으로 둔다.

## 마지막 금액 다이얼로그 분기

금액 입력 완료 시 상태는 `new-required-complete`가 된다.

이때 금액 다이얼로그는 일반 `적용`으로 닫히지 않는다. 대신 사용자의 다음 목적을 명확히 선택하게 한다.

`화물 등록 완료`는 실제 API 저장이 아니다. wizard에 입력된 전체 데이터를 메인 화면에 적용하고, 실제 등록 요청은 메인 화면의 `화물 등록` 버튼으로 분리한다.

| 선택 | 상태 전환 | UI 결과 |
| --- | --- | --- |
| `화물 등록 완료` | `new-required-complete -> new-submitted` | API 통신 없이 메인 화면에 전체 데이터 적용, 신규 접수 UI 종료, 헤더 숨김, 메인 `화물 등록` 버튼 표시 |
| `차주 정보로 이동` | `new-required-complete -> new-driver-optional` | 같은 wizard에서 차주 정보 선택 단계 표시 |

## 차주 정보 선택 단계

차주 정보는 선택 사항이다.

| 사용자 선택 | 결과 |
| --- | --- |
| 차주 선택 | 차주 정보까지 메인 화면에 적용 후 `new-submitted` |
| 건너뛰기 | 차주 미지정 상태로 메인 화면에 적용 후 `new-submitted` |
| 이전 | 금액 단계로 돌아감 |
| 닫기/취소 | 신규 접수 중단 확인 필요 |

## 신규 접수 후 개별 수정

`new-submitted` 이후에는 wizard를 재사용하지 않는다.

예를 들어 데이터 적용 후 사용자가 금액만 다시 수정하면 `openMoneyInput()` 성격의 독립 다이얼로그가 열린다. 왼쪽 프로세스 패널, wizard 단계, 신규 접수 섹션 헤더는 표시하지 않는다.

메인 화면의 `화물 등록` 버튼은 수정된 최신 화면 데이터를 기준으로 실제 API 통신을 시작하는 최종 CTA다. API 통신 세부 정책은 `08-main-submit-api-policy.md`를 따른다.

## 확인 필요

| 항목 | 확인 이유 |
| --- | --- |
| wizard 중 닫기 처리 | 입력값 폐기, 유지, 임시 저장 중 정책 필요 |
| 이전 단계 수정 시 후속 단계 유지 여부 | 상차지 변경 시 거리/금액 재계산이 필요할 수 있음 |
| 메인 `화물 등록` API 연결 | 실제 endpoint, payload schema, 서버 validation code 매핑 필요 |
| 차주 선택 후 화물맨 연동 | 선택 단계에서 단순 차주 선택만 할지, 화물맨 연동까지 허용할지 결정 필요 |
