# PRD: 화주 정보 섹션

## 목적

`화주 정보` 섹션은 화물 오더의 청구 대상과 실제 연락 담당자를 빠르게 확인하고 수정하는 영역입니다.

이번 1차 기준에서는 기존 `의뢰자` 중심 표현을 `화주` 중심 표현으로 정리하고, 필수 정보 5개만 남겨 화면 밀도를 낮춥니다.

## 사용자 문제

| 문제 | 영향 |
| --- | --- |
| 기존 화면에 조회, 연락, 점검, 의뢰자구분, 실적신고 조건이 한 섹션에 섞여 있음 | 사용자가 지금 입력해야 할 핵심 화주 정보가 흐려짐 |
| 화주 업체와 담당자 정보가 분리되어 보이지 않음 | 연락 대상과 청구 대상 확인이 느려짐 |
| 값이 입력된 후에도 입력창처럼 보이면 수정 대상이 많아 보임 | 운영자가 화면을 스캔할 때 피로도가 높아짐 |

## 사용자 목표

| 사용자 | 목표 |
| --- | --- |
| 접수/배차 운영자 | 화주 업체명과 담당자 연락처를 빠르게 확인한다 |
| 정산 담당자 | 사업자 번호와 이메일을 쉽게 찾는다 |
| 수정 담당자 | 필요한 항목만 클릭해서 빠르게 고친다 |

## 기능 요구사항

| ID | 요구사항 | 상태 |
| --- | --- | --- |
| `REQ-SI-001` | 화주 정보 필드는 `화주 업체명`, `사업자 번호`, `담당자명`, `담당자 연락처`, `담당자 이메일` 5개로 구성한다 | 1차 확정 |
| `REQ-SI-002` | 값이 입력된 상태에서는 입력창 대신 요약 row로 표시한다 | 제안 |
| `REQ-SI-003` | 값이 없는 상태에서는 `화주 정보 입력` CTA를 보여준다 | 제안 |
| `REQ-SI-004` | CTA 클릭 후 `화주` 라벨 방향으로 흡수되는 전환을 사용해 항상 노출 row로 바꾼다 | 제안 |
| `REQ-SI-005` | 데이터가 있는 항목을 클릭하면 같은 위치에서 input으로 전환한다 | 제안 |
| `REQ-SI-006` | 조회, 전화, 점검, 실적신고 조건은 1차 필드에서 제외한다 | 1차 확정 |
| `REQ-SI-007` | `화주 정보 입력` 클릭 시 기존 슬라이딩 애니메이션을 유지하고, 완료 후 통합 조회 다이얼로그를 연다 | 제안 |
| `REQ-SI-008` | 조회 결과는 업체명, 사업자 번호, 담당자명, 담당자 연락처, 담당자 이메일을 한 행에 모두 표시한다 | 제안 |
| `REQ-SI-009` | 결과 행 클릭은 선택만 하고, `화주 정보에 적용` 버튼으로만 화주 row에 일괄 반영한다 | 제안 |
| `REQ-SI-010` | 선택 없이 닫기/취소하면 빈 row를 닫고 `화주 정보 입력` CTA 상태로 복귀한다 | 제안 |
| `REQ-SI-011` | 담당자 추가 UI는 문서에서 3개 대안을 비교하고, HTML에서는 선택 이후 추천안만 노출한다 | 제안 |
| `REQ-SI-012` | 담당자 추가 폼에는 역할 구분 `배차`, `정산`, `관리`를 멀티 선택으로 제공한다 | 제안 |
| `REQ-SI-013` | 역할 구분 기본값은 `배차` 선택 상태로 둔다 | 제안 |
| `REQ-SI-014` | 담당자 추가 시 선택된 화주 기준으로 새 담당자 행을 조회 결과 리스트에 추가한다 | 제안 |
| `REQ-SI-015` | 새 담당자 행은 추가 즉시 자동 선택되고 미리보기에도 반영된다 | 제안 |
| `REQ-SI-016` | 다이얼로그에서 선택·적용된 뒤 현재 화주 정보 row에 표시된 `담당자 연락처`, `담당자 이메일` 라벨을 클릭하면 같은 위치의 텍스트 입력 컴포넌트로 전환한다 | 제안 |
| `REQ-SI-017` | 연락처/이메일 인라인 수정값은 현재 오더의 화주 정보 row에만 반영하고, 다이얼로그 조회 리스트와 마스터 데이터는 수정하지 않는다 | 제안 |
| `REQ-SI-018` | 현재 화주 정보 요약 영역의 `수정` 버튼 문구는 `화주/담당자 변경`으로 고정한다 | 제안 |
| `REQ-SI-019` | `화주/담당자 변경` 버튼은 `화주 정보 입력`과 동일한 통합 조회 다이얼로그를 열고, 현재 적용된 화주/담당자를 우선 선택 상태로 보여준다 | 제안 |
| `REQ-SI-020` | 독립 HTML은 `적용 후 row`, `입력 전/조회`, `수정 기준`, `변경 애니메이션` 상태를 탭으로 전환해 보여준다 | 제안 |
| `REQ-SI-021` | 연락처/이메일 값에는 클릭 가능성을 알 수 있는 점선 밑줄을 제공하고, `수정` 배지는 기본 숨김 후 hover/focus 때만 노출한다 | 제안 |
| `REQ-SI-022` | 담당자 추가 영역은 선택 미리보기와 같은 우측 영역 하단에 배치한다 | 제안 |
| `REQ-SI-023` | `화주/담당자 변경` 버튼의 전환 방식은 A안 `버튼 흡수형`으로 확정하고, A~E 후보 비교는 참고용으로 유지한다 | 1차 확정 |

## 비기능 요구사항

| ID | 요구사항 | 설명 |
| --- | --- | --- |
| `NFR-SI-001` | B 통합본의 밀도와 섹션 톤을 크게 흔들지 않는다 | 사용자가 화면 변화에 불편함을 느끼지 않도록 한다 |
| `NFR-SI-002` | 값 있음 상태는 한 줄에서 스캔 가능해야 한다 | 업체, 사업자 번호, 담당자를 한 번에 확인한다 |
| `NFR-SI-003` | 수정 진입은 별도 팝업 없이 같은 위치에서 이루어진다 | 현재 운송정보 섹션의 inline edit 방향과 맞춘다 |
| `NFR-SI-004` | 다이얼로그 조회 리스트는 가로 스크롤 의존을 줄인다 | 업체명, 연락처, 이메일 비교를 빠르게 한다 |

## 제외 범위

- 실제 사업자 번호 검증 로직
- 화주 업체 검색 API
- 담당자 자동완성
- 이메일 형식 validation 문구
- 다이얼로그 조회 리스트 안의 연락처/이메일 직접 편집
- 연락처/이메일 임시 수정값의 마스터 데이터 저장
- 저장 API 연동

| `AC-SI-001` | 독립 HTML에서 필수 5개 항목이 모두 표시된다 | 검증 완료 |
| `AC-SI-002` | 값 없음 상태에서 `화주 정보 입력` CTA 전환을 확인할 수 있다 | 검증 완료 |
| `AC-SI-003` | 값 있음 상태가 한 줄 요약 row로 표시된다 | 검증 완료 |
| `AC-SI-004` | 클릭 시 inline edit 예시를 확인할 수 있다 | 검증 완료 |
| `AC-SI-005` | 이전 1차 제외 항목이 독립 와이어프레임에 다시 섞이지 않는다 | 검증 완료 |
| `AC-SI-006` | 슬라이딩 애니메이션 완료 후 통합 조회 다이얼로그가 열린다 | 검증 완료 |
| `AC-SI-007` | 조회 결과 행 선택 후 `화주 정보에 적용`으로 5개 정보가 일괄 반영된다 | 검증 완료 |
| `AC-SI-008` | 선택 없이 닫기/취소하면 다시 `화주 정보 입력` CTA 상태로 돌아간다 | 검증 완료 |
| `AC-SI-009` | 담당자 추가 추천안이 HTML 안에서 확인 가능하다 | 검증 완료 |
| `AC-SI-010` | 담당자 역할 기본값이 `배차`로 선택되어 있다 | 검증 완료 |
| `AC-SI-011` | `정산`, `관리`를 추가 선택할 수 있다 | 검증 완료 |
| `AC-SI-012` | 담당자 추가 시 리스트에 `신규` 행이 생기고 자동 선택된다 | 검증 완료 |
| `AC-SI-013` | 새 담당자 선택값을 `화주 정보에 적용`하면 row에 반영된다 | 검증 완료 |

| `AC-SI-023` | B 통합본에서 화주 정보, 운송구간, 화물 운송정보의 주요 row가 32px 전후 높이로 정렬되고 가로 overflow 없이 표시된다 | 검증 완료 |
| `AC-SI-024` | B 통합본 조회 다이얼로그에서 담당자 추가, 역할 멀티선택, 신규 행 자동 선택, row 반영을 확인할 수 있다 | 검증 완료 |
