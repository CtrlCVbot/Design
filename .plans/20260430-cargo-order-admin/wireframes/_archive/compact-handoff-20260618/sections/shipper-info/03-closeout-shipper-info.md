# Closeout: 화주 정보 1차 기준 고정

## 목적

이 문서는 `화주 정보` 섹션 1차 정리의 종료 상태를 고정한다.

다음 작업인 `차주 정보` 섹션 정리를 시작하기 전에, 어떤 HTML과 문서를 기준으로 이어가야 하는지 명확히 남긴다.

## 1차 기준 상태

| 항목 | 상태 | 기준 |
| --- | --- | --- |
| 기준 표시안 | 확정 | 한 줄 요약 row |
| 필수 항목 | 완료 | 화주 업체명, 사업자 번호, 담당자명, 담당자 연락처, 담당자 이메일 |
| 값 없음 진입 | 완료 | `화주 정보 입력` CTA가 `화주` 라벨 방향으로 흡수되는 전환 |
| 조회 방식 | 완료 | `화주/담당자 통합 조회` 다이얼로그 |
| 선택 반영 | 완료 | 결과 행 선택 후 `화주 정보에 적용`으로 5개 항목 일괄 반영 |
| 담당자 추가 | 완료 | 선택된 화주 기준 담당자 추가, 역할 멀티선택, 신규 행 자동 선택 |
| 담당자 역할 | 완료 | `배차`, `정산`, `관리` 멀티 선택. 기본값은 `배차` |
| 임시 수정 | 완료 | 적용 후 row의 담당자 연락처, 담당자 이메일만 inline edit |
| 변경 버튼 | 완료 | `화주/담당자 변경` 버튼에 A안 `버튼 흡수형` 전환 적용 |
| B 통합본 반영 | 완료 | `Cargo Order Wireframe B Original Tone.html` 1번 섹션에 반영 |
| B 통합본 정렬 | 완료 | 화주 정보, 운송구간, 화물 운송정보 row를 32px 리듬으로 정렬 |

## B 통합본 최종 반영 요약

| 변경 | 결과 |
| --- | --- |
| 화주 정보 상단 배치 | 운송구간 위에 같은 폭의 1열 섹션으로 배치 |
| 한 줄 요약 row | 업체명, 사업자 번호, 담당자명, 연락처, 이메일을 한 줄에서 확인 |
| 변경 진입 | `화주/담당자 변경` 버튼 클릭 시 A안 흡수 전환 후 조회 다이얼로그 오픈 |
| 조회 다이얼로그 | 업체명, 사업자 번호, 담당자명, 담당자 연락처, 담당자 이메일을 한 행으로 표시 |
| 담당자 추가 | 선택된 화주 기준으로 담당자를 추가하고, 신규 행을 자동 선택 |
| 역할 선택 | `배차` 기본 선택, `정산`, `관리` 추가 선택 가능 |
| 임시 수정 | 다이얼로그가 닫힌 뒤 row의 연락처/이메일만 같은 위치에서 input으로 전환 |
| 수정 배지 | 기본 숨김, hover/focus 때만 노출 |

## 다음 섹션으로 넘길 기준

다음 작업 대상은 `5. 차주 정보` 섹션이다.

차주 정보는 화주 정보처럼 `사람/주체 정보`를 다루지만, 역할이 다르다. 화주는 청구와 연락 대상이고, 차주는 실제 배차된 기사와 차량 식별 대상이다. 따라서 두 섹션은 UI 패턴은 공유하되 데이터 책임은 섞지 않는다.

| 차주 정보 후보 항목 | 1차 검토 방향 |
| --- | --- |
| 차주명 | 핵심 row 항목으로 유지 |
| 차량번호 | 핵심 row 항목으로 유지 |
| 차주 연락처 | 핵심 row 항목으로 유지 |
| 복지/상태 | chip 또는 보조 액션으로 정리 |
| 사업자 번호 | 필요 시 상세 또는 조회 다이얼로그에서 검토 |
| 차량 톤수/차종 | 화물 운송정보와 중복 가능성이 있어 표시 책임 분리 필요 |

## 1차 마무리 판정

화주 정보는 1차 기준안으로 마무리한다.

다음 작업은 `driver-info` 섹션 패키지를 시작하고, B 통합본의 `5. 차주 정보`를 화주 정보와 같은 방식으로 PRD, wireframe, user flow, source mapping, HTML 반영 순서로 정리한다.
