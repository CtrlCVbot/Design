# Field State Mapping - 화물맨 Phase 2

## 1. 상태 매핑

| 상태 ID | 표시명 | row 표시 | 주요 액션 | 다음 상태 |
| --- | --- | --- | --- | --- |
| `hm_not_linked` | 연동 전 | `차주 [화물맨 연동] ... [차주/차량 입력]` | 차주/차량 입력, 화물맨 연동 | Phase 1, `hm_linking` |
| `manual_new_driver` | 일반 신규 차주 등록 | 다이얼로그 우측 등록 폼 | 중앙/플로팅 `차주 등록`, 등록 저장 | 내부 DB 차주 흐름 |
| `hm_linking` | 화물맨 연동중 | `화물맨 화물 연동중` | 확인 후 화물맨 API 취소 요청, 차주/차량 입력 | `hm_canceling`, `hm_dispatched`, `hm_failed` |
| `hm_dispatched` | 화물맨 배차완료 | `화물맨 화물 배차완료` | 확인 후 화물맨 API 취소 요청, 화물맨 차주 적용 | `hm_canceling`, `hm_existing_driver`, `hm_new_driver` |
| `hm_canceling` | 화물맨 취소 요청중 | `화물맨 API 취소 요청중` | API 응답 대기 | `hm_not_linked`, `hm_cancel_failed` |
| `hm_manual_driver_applied` | 내부 DB 차주 적용 + 화물맨 연동중 | 내부 DB 차주 row + `차주/차량 변경` + `연동 취소` | 차주 변경, 화물맨 API 취소 | `hm_canceling`, Phase 1 row 적용 |
| `hm_existing_driver` | 기존 등록 차주 | `기존 등록` | 선택 미리보기, 차주 정보에 적용 | Phase 1 row 적용 |
| `hm_new_driver` | 신규 차주 | `신규 차주` | 차주 등록 | `hm_save_required` |
| `hm_save_required` | 저장 필요 | `저장 필요` | 차주 등록, 수동 DB 저장 | `hm_saved`, `hm_duplicate`, `hm_save_failed` |
| `hm_saved` | DB 저장 완료 | `DB 저장됨` | 일반 차주처럼 선택/적용 | Phase 1 DB 차주 흐름 |
| `hm_failed` | 배차 실패 | `배차 실패` | 재연동, 내부 차주 조회 | `hm_linking`, Phase 1 |
| `hm_cancel_failed` | 취소 실패 | `화물맨 취소 실패` 알림 + `연동 취소` 버튼 유지 | 확인 후 취소 재시도 | `hm_canceling` |
| `hm_duplicate` | 중복 발견 | `중복 후보 있음` | 기존 차주 연결, 저장 보류 | `hm_saved`, `hm_save_required` |
| `hm_save_failed` | 저장 실패 | `저장 실패` | 저장 재시도 | `hm_saved`, `hm_save_required` |

## 2. 화물맨 배차 행 필드

| 필드 | 표시 | 저장 전 | 저장 후 | 비고 |
| --- | --- | --- | --- | --- |
| source | row column | `화물맨 배차` | `출처 / 화물맨` | 적용 후 일반 DB 차주와 구분 |
| saveState | 배지 | `저장 필요` | `DB 저장됨` | 수동 저장 기준 |
| registrationState | 배지 | `기존 등록` 또는 `신규 차주` | `DB 저장됨` | 적용 가능 여부 판단 |
| driverName | row value | 표시 | DB 차주명으로 저장 | 화물맨 수신값 |
| vehicle | row value | 표시 | DB 차량번호로 저장 | 중복 판단 핵심 |
| phone | row value | 표시 | DB 연락처로 저장 | 중복 판단 보조 |
| tonnage | row value/select | 표시/보정 가능 | DB 차량 스펙 또는 오더 보정값 | 선택형 |
| truckType | row value/select | 표시/보정 가능 | DB 차량 스펙 또는 오더 보정값 | 선택형 |
| externalDispatchId | hidden value | 보관 | 매핑값으로 보관 | 화물맨 배차 식별값 |
| receivedAt | meta | 표시 가능 | 저장 로그로 보관 | 운영 추적 |

## 3. 조회 리스트 구조

| 영역 | 행 타입 | 검색어 영향 | 설명 |
| --- | --- | --- | --- |
| 상단 고정 영역 | 화물맨 배차 행 | 영향 없음 | 해당 화물의 화물맨 배차 결과가 있으면 항상 표시 |
| 일반 결과 영역 | 내부 DB 차주 행 | 검색어 적용 | 기존 Phase 1 검색 결과 |
| 빈 결과 안내 | 안내 행 | 검색어 적용 | 일반 결과가 없을 때 표시 |

## 4. 다이얼로그 레이아웃 유지 매핑

| 기존 다이얼로그 영역 | Phase 2 유지/추가 기준 | 변경 수준 |
| --- | --- | --- |
| 검색 input | 기존 위치와 검색 역할 유지 | 유지 |
| 중앙/플로팅 `차주 등록` | Phase 1의 일반 신규 차주 등록 진입 유지 | 유지 |
| 조회 리스트 | 기존 열 구조 유지, 최상단 화물맨 우선 행만 추가 | 최소 추가 |
| 선택 미리보기 | 기존 등록 차주일 때 기존 미리보기 컴포넌트 재사용 | 유지 |
| 차주 등록 폼 | 일반 신규 등록은 기본값, 화물맨 신규 차주는 수신값 자동 입력 | 최소 추가 |
| 하단 CTA | 기존 `차주 등록`, `차주 정보에 적용` 위치 유지 | 유지 |
| 적용 버튼 상태 | 신규 차주 등록 전 비활성화, 등록 후 활성화 | 상태 추가 |
| 출처 컬럼 | 적용 완료 row에 `출처 / 화물맨` 필드 추가 | 최소 추가 |
| 적용 row `연동 취소` | 내부 DB 차주가 적용됐지만 화물맨 연동이 살아 있을 때만 표시 | 상태 추가 |

## 5. 버튼 및 액션 매핑

| 버튼 | 노출 조건 | 처리 |
| --- | --- | --- |
| 차주/차량 입력 | 연동 전/연동중/배차완료 전 | 기존 내부 DB 차주 조회 |
| 화물맨 연동 | 연동 전 | 확인 다이얼로그 후 외부 연동 요청 |
| 연동 취소 | 연동중/배차완료 | 취소 확인 다이얼로그 오픈, `YES` 선택 시 화물맨 API 취소 요청 |
| 적용 row 연동 취소 | 내부 DB 차주 적용 + 화물맨 연동중 | 현재 차주 배차는 유지하고 화물맨 API 취소 요청 |
| API 취소 성공 | 취소 요청중 | 화물맨 연동 상태 해제 후 입력 전 상태 복귀 |
| API 취소 실패 | 취소 요청중 | `화물맨 취소 실패` 알림 표시, `연동 취소` 버튼 유지로 재시도 |
| 화물맨 차주 적용 | 배차 결과 수신 | 통합 조회 다이얼로그 오픈 |
| 차주 정보에 적용 | 기존 등록 화물맨 배차 행 또는 DB 차주 행 선택, 신규 차주 등록 완료 후 | 기존 row 적용 |
| 중앙/플로팅 차주 등록 | 조회 결과에 원하는 차주/차량이 없음 | 일반 신규 차주 등록 폼 표시 |
| 등록 폼 차주 등록 | 일반 신규 등록 또는 신규 화물맨 배차 행 등록 전 | 내부 DB 수동 저장 시작 |
| DB 저장 | 저장 필요 상태 | 내부 DB 저장 |
| 기존 차주 연결 | 중복 후보 있음 | 중복 차주와 외부 배차 정보 연결 |
| 다시 연동 | 실패 상태 | 화물맨 연동 재시도 |

## 6. 상태 표현 기준

| 표현 | 색상 성격 | 의미 |
| --- | --- | --- |
| `화물맨 화물 연동중` | 대기 | 외부 배차 대기 |
| `화물맨 화물 배차완료` | 정보 | 외부에서 배차 완료됨 |
| `화물맨 배차` | 정보 | 외부에서 배차 완료된 차주 |
| `기존 등록` | 완료 | 내부 DB에 이미 등록된 차주 |
| `신규 차주` | 경고 | 등록 후 배차 적용 가능 |
| `저장 필요` | 경고 | 내부 DB 미저장 |
| `DB 저장됨` | 완료 | 내부 DB 저장 완료 |
| `중복 후보 있음` | 경고 | 기존 차주/차량과 연결 검토 필요 |
| `배차 실패` | 오류 | 외부 배차 실패 |
| `저장 실패` | 오류 | 내부 DB 저장 실패 |

## 7. 중복 처리 기준

| 비교 기준 | 우선순위 | 처리 |
| --- | --- | --- |
| 차량번호 일치 | 1 | 기존 차량 연결 후보로 표시 |

## 8. 적용 row 반영 기준

| 상태 | row 적용 가능 여부 | 표시 |
| --- | --- | --- |
| 기존 등록 차주 | 가능 | 다이얼로그에서는 `화물맨 배차`, `기존 등록` 보조 배지 |
| 일반 신규 차주 등록 전 | 불가 | 중앙/플로팅 `차주 등록`으로 등록 폼 진입, `차주 정보에 적용` 비활성화 |
| 일반 신규 차주 등록 완료 | 가능 | 조회 목록에 신규 행 추가 후 자동 선택 |
| 신규 차주 등록 전 | 불가 | `차주 등록` 노출, `차주 정보에 적용` 비활성화 |
| 신규 차주 등록 완료 | 가능 | `차주 등록` 숨김, `차주 정보에 적용` 활성화, 적용 row에서는 `출처 / 화물맨` |
| 저장 완료 | 가능 | `DB 저장됨` 보조 배지 표시 |
| 중복 후보 있음 | 제한적 가능 | 기존 차주 연결 또는 저장 보류 선택 후 적용 |
| 실패 | 불가 | 실패 상태와 재시도 액션 우선 |
