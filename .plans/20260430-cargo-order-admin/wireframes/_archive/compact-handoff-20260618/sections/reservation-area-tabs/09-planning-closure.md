# 보조 정보 섹션 기획 마무리 체크리스트

## 목적

이 문서는 이번 `보조 정보` 섹션 기획을 어떤 상태로 마무리하는지 정리합니다.

현재 범위는 구현이 아니라 기획 마무리입니다. 따라서 이 문서는 개발 실행 계획이 아니라, 섹션 기획 결과물의 기준 파일, 확인 방법, 남은 정책 확인 항목을 정리하는 용도입니다.

## 기준 결과물

| 대상 | 위치 | 상태 | 확인 방법 |
| --- | --- | --- | --- |
| 통합 HiFi master | `../../results/html/cargo-order-admin-hifi-master.html` | 기준 파일 | 기본 열람 시 데이터 있음 상태 |
| 보조 정보 데이터 없음 | 같은 master 파일 | 반영 완료 | `신규(F3)` 클릭 후 확인 |
| 보조 정보 부분 데이터 | 같은 master 파일 | 반영 완료 | 화주 선택, 상차지+하차지 적용, 정산 정보 적용 순서로 확인 |
| 과거 후보 HTML | `../../results/html/*reservation-tabs*` | 참고용 | 비교, rollback, 출처 확인 용도로만 사용 |

`cargo-order-admin-hifi-master.html`은 이번 섹션이 최종 결과물에 반영된 기준 HTML입니다. 과거 후보 HTML은 비교와 이력 확인 용도로만 둡니다.

## 기획 마무리 판정

| 영역 | 판정 | 근거 |
| --- | --- | --- |
| 탭 구조 | 완료 | `메모`, `금액 로그`, `운송 구간 지도` 순서 확정 |
| 화면 노출명 | 완료 | 우측 패널명은 `보조 정보`로 확정 |

| 부분 데이터 상태 | 완료 | 화주, 운송 구간, 정산 정보 적용 단계별 mock 기준 확정 |
| 금액 로그 용어 | 완료 | `조정금`과 +/- 표기로 통일 |
| 메모 입력 방식 | 완료 | 리스트 우선, 추가는 dialog 방식 |
| 지도 중복 정보 | 완료 | 좌측 `2. 운송 구간`과 겹치는 전체 주소 반복 표시 제외 |
| 디자인 기준 | 완료 | `../../results/html/Design System.html`의 shadcn 기준 사용 |

## 최종 확인 결과

| 검증 | 결과 | 기준 |
| --- | --- | --- |
| 3개 탭 목적 분리 | 통과 | 메모는 사람 기록, 금액 로그는 금액 명세, 지도는 위치/경로 확인 |
| MVP/후속 범위 분리 | 통과 | 각 상세 문서에 MVP와 후속 확장 범위 분리 |
| 기존 접수/수정 화면 충돌 여부 | 통과 | 주소와 금액 직접 수정 책임은 좌측 기존 섹션 유지 |
| 데이터 있음/없음 상태 | 통과 | master 기본 화면과 `신규(F3)` 흐름으로 확인 |
| 부분 데이터 상태 | 통과 | 화주, 운송 구간, 정산 정보 적용 단계별 mock 노출 기준 정리 |
| acceptance criteria | 통과 | `RAT`, `MAP`, `MEMO`, `AMT`, `STATE`, `VIS` 기준 작성 |
| self-review | 통과 | 리스크와 후속 확인 사항 정리 |

## 후속 정책 확인 항목

아래 항목은 기획 마무리를 막지는 않습니다. 다만 다음 차수에서 실제 데이터 구조나 운영 정책을 정할 때 확인해야 합니다.

| 우선순위 | 영역 | 확인 필요 내용 | 필요한 이유 |
| --- | --- | --- | --- |
| P0 | 신규 접수 저장 정책 | 실제 endpoint, payload schema, 서버 validation | `화물등록` 버튼의 실제 저장 책임과 오류 처리를 정해야 함 |
| P0 | 조정금 저장 | `청구금`, `배차금`, `둘 다` 대상 조정금을 단일 item으로 둘지 분리 저장할지 | 금액 로그, 정산, 감사 이력의 데이터 구조에 직접 영향 |
| P0 | 정산 후 수정 | 정산 완료 후 조정금 수정 가능 여부와 승인 예외 | 잘못 정하면 정산 오류와 운영 신뢰 리스크 발생 |
| P0 | 금액 권한 | 금액 로그 조회, 조정금 입력, 마스킹 기준 | 민감 금액 정보 노출 범위를 정해야 함 |
| P1 | 메모 권한 | 작성, 수정, 삭제, 공개 범위, 보존 기간 | 메모 dialog와 리스트 액션 노출 기준에 영향 |
| P1 | 지도 provider | 지도 API, 비용, 쿼터, 장애 fallback | 지도 탭의 실제 표시 방식과 오류 상태에 영향 |
| P1 | 좌표/예상 시간 | 좌표 저장 방식, 예상 시간 산정 기준, 실시간 교통 포함 여부 | 거리/예상 시간이 운영 판단에 쓰일 때 기준 필요 |
| P2 | 입력 옵션 | 드롭다운 전체 옵션, 필수값, 상태별 버튼 활성화 | 캡처 기반 문서의 남은 불확실성 제거 |
| P2 | 목록/팝업 | 숨은 컬럼, 조회/점검/위탁증/정보변경 팝업 | 전체 현대화 범위 확정에 필요 |

## 종료 판정

보조 정보 섹션 기획은 현재 범위에서 완료입니다.

남은 항목은 화면 기획 누락이 아니라 다음 차수에서 결정할 운영 정책입니다. 따라서 현재 산출물은 기획 완료본으로 닫고, 다음 작업은 정책 확인이 필요할 때 별도 차수로 진행합니다.
