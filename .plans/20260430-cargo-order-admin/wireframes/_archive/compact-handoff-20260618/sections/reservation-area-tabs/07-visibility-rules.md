# 보조 정보 표시/접힘/숨김 규칙

## 목적

`보조 정보`는 좌측 접수/수정 입력을 보조하는 우측 정보 패널입니다. 내부 작업명과 폴더명은 기존 흐름을 따라 `예약 영역`과 `reservation-area-tabs`를 유지합니다.

따라서 데이터가 없다는 이유만으로 바로 숨기지 않습니다. 운영자가 현재 오더의 메모, 경로, 금액 맥락을 확인할 수 있도록 기본적으로는 보여주고, 화면 폭이나 권한, 오더 컨텍스트에 따라 `표시`, `접힘`, `숨김`을 나눕니다.

## 최종 결정

| 상태 | 의미 | 기본 사용 시점 |
| --- | --- | --- |
| 표시 | 우측 `보조 정보` 전체를 보여줌 | 데스크톱 접수/수정 화면, 오더가 선택된 상태 |
| 접힘 | 얇은 요약 rail 또는 `보조 정보 열기` 버튼만 보여줌 | 화면이 좁거나 좌측 입력 집중이 필요한 상태 |
| 숨김 | 화면에서 `보조 정보`를 제외함 | 오더 없음, 목록 화면, 권한 없음, 인쇄/외부 공유 화면 |

## 기본 탭 결정

| 항목 | 결정 |
| --- | --- |
| 첫 번째 탭 | `메모` |
| 최초 진입 기본 탭 | `메모` |
| 이유 | 운영자가 배차 전후로 가장 먼저 확인해야 하는 사람의 기록과 주의사항을 먼저 노출하기 위함 |
| 다음 탭 순서 | `금액 로그`, `운송 구간 지도` |

## 표시 상태

다음 조건에서는 `보조 정보`를 전체 표시합니다.

| 조건 | 표시 방식 |
| --- | --- |
| 오더 상세 또는 접수/수정 화면에 진입 | `보조 정보` 전체 표시 |
| 데스크톱 또는 충분한 가로 폭 | 우측 패널로 표시 |
| 오더가 선택되어 있음 | `메모` 기본 탭 표시 |
| 주소가 비어 있음 | 섹션은 표시하고 `운송 구간 지도` 탭에서 메모와 같은 빈 상태 안내 박스 표시 |
| 메모가 없음 | 섹션은 표시하고 `메모` 탭에서 빈 상태와 `메모 추가` 표시 |
| 금액 로그 데이터가 없음 | 섹션은 표시하고 `금액 로그` 탭에서 메모와 같은 빈 상태 안내 박스 표시 |
| 기준금액은 있고 조정 내역만 없음 | 섹션은 표시하고 `금액 로그` 탭에서 명세 안 조정금 0원 또는 상세 항목 없음 상태 표시 |
| 정산 완료 오더 | 섹션은 표시하되 금액 수정 제한 안내를 read-only로 표시 |

## 접힘 상태

다음 조건에서는 `보조 정보`를 접힘 상태로 전환할 수 있습니다.

| 조건 | 접힘 방식 |
| --- | --- |
| 화면 폭이 좁음 | 우측 rail 또는 하단 drawer 진입 버튼만 표시 |
| 좌측 입력 집중이 필요함 | 사용자가 접기 버튼으로 `보조 정보`를 접음 |
| 주소 검색, 금액 입력, 메모 작성 dialog가 열림 | 배경 패널을 비활성 또는 접힘 처리 |
| 사용자가 이전에 접음 | 같은 세션 또는 사용자 설정에서 접힘 상태 유지 가능 |

접힘 상태의 요약 예시는 아래처럼 짧게 둡니다.

```txt
보조 정보
메모 3 · 금액 확인 · 지도 확인
```

## 숨김 상태

다음 조건에서는 `보조 정보`를 숨깁니다.

| 조건 | 이유 |
| --- | --- |
| 오더가 선택되지 않음 | 표시할 `보조 정보` 컨텍스트가 없음 |
| `cargo-list` 같은 목록 화면 | 목록 우선 화면이므로 우측 보조 패널을 노출하지 않음 |
| 모바일 초소형 화면 | 우측 패널 유지가 입력 흐름을 방해함 |
| 인쇄, PDF, 외부 공유 화면 | 내부 운영 메모와 금액 정보 노출 위험 |
| 사용자에게 `보조 정보` 조회 권한이 없음 | 민감 정보 접근 제한 |
| 금액 로그 권한만 없음 | 전체 숨김보다 `금액 로그` 탭만 숨김 또는 권한 안내를 우선 검토 |

## 탭별 데이터 없음 처리

| 탭 | 데이터 없음 상태 | 섹션 숨김 여부 |
| --- | --- | --- |
| `메모` | 등록된 메모 없음, `메모 추가` 버튼 표시 | 숨기지 않음 |
| `금액 로그` | 등록된 금액 로그 없음, `금액 입력으로 이동` 버튼 표시 | 숨기지 않음 |
| `운송 구간 지도` | 운송 구간 계산 불가, `운송 구간 입력` 버튼 표시 | 숨기지 않음 |

## 권한 기준

| 권한 상태 | 처리 |
| --- | --- |
| `보조 정보` 전체 조회 권한 없음 | `보조 정보` 숨김 |
| 메모 조회 권한 없음 | `메모` 탭 숨김 또는 권한 안내 |
| 메모 작성 권한 없음 | `메모 추가` 버튼 숨김, 리스트는 권한에 따라 표시 |
| 금액 조회 권한 없음 | `금액 로그` 탭 숨김 또는 권한 안내 |
| 지도 조회 권한 없음 | `운송 구간 지도` 탭 숨김 또는 주소 상태만 표시 |

## Responsive 기준

| 화면 조건 | 처리 |
| --- | --- |
| 넓은 데스크톱 | 좌측 입력 + 우측 `보조 정보` 전체 표시 |
| 중간 폭 | `보조 정보` 표시 유지, 내부 탭/요약 줄 wrap 허용 |
| 좁은 폭 | 접힘 상태 또는 drawer 전환 |
| 모바일 초소형 | 기본 숨김, 필요 시 별도 진입 버튼 |

## Acceptance Criteria

| ID | 기준 |
| --- | --- |
| `VIS-AC-01` | 오더가 선택된 데스크톱 접수/수정 화면에서는 `보조 정보`가 기본 표시되어야 합니다. |
| `VIS-AC-02` | 최초 진입 시 첫 번째 탭과 기본 탭은 `메모`여야 합니다. |
| `VIS-AC-03` | 주소, 메모, 금액 로그 데이터가 없어도 `보조 정보` 전체를 숨기지 않고 세 탭 모두 같은 empty box 패턴으로 처리해야 합니다. |
| `VIS-AC-04` | 화면이 좁거나 사용자가 접기를 선택하면 `보조 정보`는 접힘 상태로 전환할 수 있어야 합니다. |
| `VIS-AC-05` | 오더가 없거나 목록/인쇄/외부 공유 화면에서는 `보조 정보`가 숨김 처리되어야 합니다. |
| `VIS-AC-06` | 금액 권한이 없는 사용자는 `금액 로그` 탭을 볼 수 없거나 명확한 권한 안내를 받아야 합니다. |
| `VIS-AC-07` | 접힘 상태의 요약은 `메모 수`, `지도 상태`, `수익 또는 정산 상태` 정도로 제한해야 합니다. |

## 후속 확인 필요

| 항목 | 확인 내용 |
| --- | --- |
| breakpoint | 실제 구현에서 접힘 전환 기준 px 확정 |
| 사용자 설정 저장 | 접힘/펼침 상태를 세션만 유지할지 사용자 설정으로 저장할지 결정 |
| 권한 정책 | 메모, 지도, 금액 로그 탭별 조회/작성 권한 확정 |
| 모바일 대응 | drawer 방식, 별도 화면 이동 방식 중 선택 |
