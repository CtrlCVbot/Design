# 화물정보 요약 섹션 기획 패키지

## 목적

이 패키지는 화물 오더 접수/수정 화면의 `4. 화물정보 요약` 파트만 독립적으로 정리하기 위한 문서 묶음입니다.

기존 `운송구간`, `화주 정보` 섹션처럼 한 줄 요약 row를 기준으로 정리합니다. 중량, 계산 기준, 이하 여부, 안내문, 증빙 옵션은 이번 분리 화면에서 제외합니다.

| 항목 | 기준 |
| --- | --- |
| 기준 화면 | `Cargo Order Wireframe B Original Tone.html` |
| 기준 섹션 | `4. 화물정보 요약` |
| 디자인 톤 | 운송구간/화주 정보와 동일한 종이 배경, 얇은 선, compact row, 손글씨형 값 표현 |
| 범위 | 화물 운송정보에서 파생된 요약 문장 표시와 수정 진입 |

## 1차 결정

| 결정 | 내용 |
| --- | --- |
| 표시 방식 | `요약` 라벨 + `화물정보 요약` 값 + `수정` 버튼의 한 줄 row |
| 값 있음 | 운송 조건, 품목, 차종, 상하차 조건을 한 문장으로 요약 |
| 값 없음 | `화물 운송정보 입력 후 요약 생성` placeholder 표시 |
| 수정 진입 | row 안에서 같은 위치에 input으로 전환 |
| 원본 데이터 | `톤수`, `차종`, `대수`, `실중량`, `품목`은 `3. 화물 운송정보`가 source of truth |
| 보조 데이터 | 상하차 일시와 상하차 방법은 `2. 운송 구간`에서 받아 요약 조건으로 사용 |
| 금액 조건 | `결제방법`, `청구비용`, `운송비용`, `수수료`, `수익/차주운임`은 요약 문장에 기본 포함하지 않음 |
| 사용자 수정 | 자동 생성 요약과 사용자가 수정한 override 문장을 분리해 관리 |
| 삭제 항목 | 중량, 계산 기준, 이하 여부, 안내문, 증빙 옵션, 하단 PRD/User Flow 보조 패널 |

## 사용 방법

1. `cargo-summary-docs-section-wireframe.html`을 브라우저에서 열어 값 있음, 값 없음, 수정 중 상태를 확인합니다.
2. `02-wireframe-cargo-summary-docs.md`에서 B 통합본에 들어갈 row 구조를 확인합니다.
3. `06-cargo-transport-summary-data-contract.md`에서 자동 요약 생성 필드와 override 정책을 확인합니다.
4. B 통합본의 4번 섹션에서 동일 구조가 반영됐는지 확인합니다.
## 2026-06-08 Clean 반영 고정 규칙

| 항목 | 기준 |
| --- | --- |
| 입력전 상태 | 화물 운송정보가 아직 입력되지 않았을 때 `운송 정보 입력 후 요약됩니다` placeholder를 표시한다. |
| 적용 후 상태 | 운송+품목과 운송구간의 핵심 값으로 요약 문장을 생성한다. |
| 금액 제외 | 결제방법, 청구비용, 운송비용, 수수료, 수익/차주운임은 요약 문장에 기본 포함하지 않는다. |
| 수정 | 요약 row에서 수정 버튼을 눌렀을 때 같은 자리 input으로 전환한다. |
| 표시 | 요약 문장은 불필요하게 `...`으로 줄이지 않고, 필요한 경우 줄바꿈으로 전체 문맥을 보존한다. |
