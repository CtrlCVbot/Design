## 운송 관련 다이얼로그 최근 사용 후보

최근 사용 후보는 기존 입력 필드를 대체하지 않고, 다이얼로그 안에서 선택을 빠르게 돕는 보조 데이터입니다.

| 다이얼로그 | 항목 | 구분 | 현재 prototype 값/옵션 | 보존 메모 |
| --- | --- | --- | --- | --- |
| 상차지 주소 검색 | 상차 최근 장소 | 후보 리스트 | 장소명, 주소, 출처, 상태 | 기존 검색 결과 영역의 초기 상태로 표시 |
| 하차지 주소 검색 | 하차 최근 장소 | 후보 리스트 | 장소명, 주소, 출처, 상태 | `조회` 또는 Enter 후 기존 검색 결과로 전환 |
| 주소 검색 | 선택 미리보기 | 미리보기 | 주소, 지명, 상세, 담당, 연락처 | 최근 장소 클릭 시 미리보기만 갱신 |
| 운송+품목 입력 | 최근 조합 | 후보 리스트 | 품목, 톤수, 차종, 중량 | 클릭 시 입력폼 값만 채움 |
| 운송+품목 입력 | 확정 버튼 | 버튼 | `운송+품목 적용` | 적용 전까지 `화물 정보` row에 반영하지 않음 |

## 보조 정보 패널

`보조 정보`는 기존 우측 `예약 영역`을 대체하거나 정리하는 패널입니다. 직접 입력을 늘리는 영역이 아니라, 선택된 화물 운송건의 후속 판단 정보를 탭으로 확인하는 영역입니다.

| 탭 | 항목 | 구분 | 현재 값/옵션 | 보존 메모 |
| --- | --- | --- | --- | --- |
| 공통 | 패널명 | 표시 | `보조 정보` | 섹션 헤더는 유지하되 설명 문구는 최소화 |
| 공통 | 탭 순서 | 탭 | `메모`, `금액 로그`, `운송 구간 지도` | 메모를 기본 탭으로 사용 |
| 공통 | 데이터 없음 | 빈 상태 | 신규 접수 초기화 직후 | 세 탭 모두 같은 톤의 빈 상태 사용 |
| 공통 | 상태 전환 | 프로토타입 동작 | `resetAll`, `applyShipper`, `applyAddr`, `applyMoney` | 별도 URL이 아니라 실제 입력 흐름에 맞춰 표시 전환 |
| 메모 | 메모 목록 | 리스트 | 운영자 메모, 고객 요청, 배차 특이사항 | 첫 화면은 리스트만 표시 |
| 메모 | mock 표시 조건 | 상태 | 화주 선택 후 | 화주만 선택해도 메모 mock 데이터 표시 |
| 메모 | 메모 추가 | 버튼/다이얼로그 | `메모 추가` | 클릭 시 다이얼로그에서 유형과 내용을 입력 |

| 금액 로그 | 기준 배차금 | 금액 | 예: `420,000원` | 차주에게 지급하는 기본 배차금 |

| 금액 로그 | 수익 | 금액 | 청구금 합계 - 배차금 합계 | 운영자가 가장 빠르게 확인해야 하는 결과값 |
| 금액 로그 | 조정금 상세 | 상세 목록 | 사유, 대상, 금액, 담당자, 마지막 일시, 설명 | 기본 화면은 컴팩트하게, `상세 항목 보기`에서 확인 |
