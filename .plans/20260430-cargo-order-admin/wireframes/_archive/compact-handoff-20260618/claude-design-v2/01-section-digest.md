# 01. 섹션 다이제스트 (5개 섹션 정제 요약)

> `sections/` 기획 문서를 섹션별로 취합·압축한 것. 클로드 디자인이 **필드·상태·인터랙션을 보존**하도록 하는 기준.
> 상세 원본은 각 섹션 폴더(`../sections/<key>/`)에 있으며 수정하지 않는다.

---

## 1. 화주 정보 (shipper-info)

**목적** — 오더의 청구 대상 업체와 연락 담당자를 확인·수정. 기존 `의뢰자` 표현을 `화주`로 정리, 필수 5개 항목만 유지.

**핵심 필드**

| 필드 | 컨트롤 | 필수 | 비고 |
| --- | --- | --- | --- |
| 화주 업체명 | 다이얼로그 선택(직접입력 불가) | 필수 | — |
| 사업자 번호 | text `000-00-00000` | 필수 | `사업자 번호 검증 대기` 배지(1차 검증 제외) |
| 담당자명 | 다이얼로그 선택 | 필수 | — |
| 담당자 연락처 | text → inline input | 필수 | 적용 후 row에서만 임시 수정(점선 밑줄) |
| 담당자 이메일 | text → inline input | 필수 | 적용 후 임시 수정 |
| 담당자 역할 | chip 멀티선택 | — | `배차`/`정산`/`관리`, 기본 `배차` |

**상태** — `empty`(placeholder+CTA) / `summary`(5필드 한 줄 row + 변경 버튼) / `editing`(연락처·이메일만 inline) / `dialog`(조회·선택·담당자추가).

**인터랙션** — `화주 정보 입력` CTA가 라벨 방향으로 **흡수 전환** → 통합 조회 다이얼로그(검색 input + 6기준 필터 + 결과 리스트 + 우측 선택 미리보기) → 행 선택 후 `화주 정보에 적용`으로 5개 항목 일괄 반영. 담당자 추가 = 우측 미니 패널(C안).

**확정** — 표시: 한 줄 요약 row / 전환 애니메이션: **A안 버튼 흡수형** / 담당자 추가: **C안 우측 미니 패널**.

**보존/주의** — 업체명·사업자·담당자명은 **직접 편집 불가**(클릭 어포던스 주지 말 것). 연락처·이메일만 편집 가능 표시. 행 클릭(선택)과 적용(반영)을 분리, 단일 클릭 즉시반영 금지. inline 임시 수정은 현재 오더에만 반영(마스터·조회 리스트 불변).

## 2. 운송 구간 (transport-route)

**목적** — 상차지(출발)·하차지(도착)를 한 곳에서 확인하고 펼쳐 수정. 기존 5(배차 유형)/6(상차 조건)/7(하차 조건)번 섹션을 흡수 통합.

**핵심 필드**

| 필드 | 컨트롤 | 필수 | 비고 |
| --- | --- | --- | --- |
| 행정주소 | text + 주소검색 다이얼로그 | 필수 | inline edit 금지(좌표/거리 의존) |
| 상세주소 | text / inline edit | 선택 | 펼침/1열 행에서만 |
| 지명(placeName) | text | 선택 | 값 없으면 줄 미렌더 |
| 일시(timeLabel) | 칩(지금/당일/내일) | 선택 | 방법과 한 줄, 선택 UI는 별도 화면 |
| 방법(method) | 선택(지게차/호이스트/수작업/크레인/컨베이어) | 선택 | 일시와 한 줄 |
| 담당자/연락처 | text / inline edit | 선택 | 펼침에서만 |
| 거리/기준금액 | 읽기전용 칩(계산값) | 자동 | 주소+톤수+차종 충족 시만 확정 표시 |
| 배차 대표 유형 | 칩(독차/혼적) | 필수 | 단일선택, 기본 `독차` |
| 배차 옵션 | 멀티칩(긴급/왕복/예약) | 선택 | 선택된 것만 |

**상태** — 요약(상/하차 2행) / 빈 상태(주소 미입력) / 펼침(상세·담당자·특이사항) / 계산 메타(`주소 입력 후 계산`→`톤수·차종 선택 후 계산`→`거리 15.8km·기준 105,000원`→`다시 계산`).

**인터랙션** — **주소 검색/적용 다이얼로그**(상·하차 공용, 화주 정보 검색과 동일 레이아웃): 화주 주소록 + **Kakao** 통합 검색, 출처 배지 구분. 주소 원문은 항상 읽기 전용, 상세주소/담당자/연락처만 펼침 inline edit.

**확정** — 1차 기준안 **D `compact-hybrid`** / 배치 **상차·하차 세로 2행** / B 통합본 **full-width**(좌측 route + 우측 배차·계산 메타 패널) / 주소 적용 = **1열 단일 행**.

**보존/주의** — 지명·행정주소 = 검색 다이얼로그로만 변경. 거리/기준금액 대기값을 확정값처럼 보이게 하지 말 것(회색 대기 chip). 상차/하차 row 안에 배차/조건 칩 반복 금지(보조 메타에만). full-width시 route·meta 패널 y좌표·높이 정렬 일치.

## 3. 화물 운송정보 (cargo-transport)

**목적** — 차량/화물 조건과 정산 금액을 확인·수정. 운송구간과 중복(`광폭`·`도착`) 제거. 화주에게 받을 `청구비용`과 차주 지급/청구 `운송비용`·`수수료`를 분리. **F안 = `운송+품목` / `금액` 2블록 분리**.

**핵심 필드**

| 필드 | 컨트롤 | 필수 | 비고 |
| --- | --- | --- | --- |
| 톤수 | select | 필수 | 선택 시 `실중량`=톤수 110% 자동. 운송구간 기준금액에 영향 |
| 차종 | select | 필수 | 운송구간 기준금액에 영향 |
| 대수 | number | 필수 | 1대는 요약에서 생략 |
| 실중량 | decimal(0.00) | 필수 | 110% 자동·수정 가능, 셀 내부 보조 힌트 |
| 품목 | text | 선택(저장 전 안내) | `4.요약`의 핵심 원본 |
| 결제방법 | select/segment | 필수 | `인수증`/`선불`/`착불`/`선착불`. 수수료 표시·계산식 제어 |
| 청구비용 | money | 필수 | 입력 시 `운송비용` 같은 값 자동 입력 |
| 운송비용 | money | 필수 | 수정해도 청구비용 유지 |
| 수수료 | money(조건부) | 조건부 | 인수증=숨김/비활성. 입력 시 선착불+청구비용 0원+운송비용 포커스 |
| 수익/차주운임 | calculated readonly | 자동 | 인수증=`수익`(청구−운송), 그 외=`차주운임`(운송+수수료). 기존 `합계` 대체 |

**상태** — empty→activating→activated-empty 흐름 / inline: read·hover·focus·editing·`auto-filled`·`overridden`·readonly·error / `weight-auto-filled↔overridden`, `locked`(선착불 청구비용 0원), `conditional-visible/hidden`(수수료).

**인터랙션** — inline 편집(한 번에 한 셀, 계산값 readonly, 팝업 미사용). 자동 규칙(톤수→실중량 110% / 청구→운송 동일값 / 수수료 입력→선착불 세트). F안 최초 입력 = `운송+품목 입력`·`금액 조건 선택` 2개 CTA 흡수 → 다이얼로그(계산 미리보기 포함).

**확정** — F안 구조 채택. 최종 배치 기준 = **B 통합본의 `3. 화물 운송정보`**. F안 단독 HTML은 reference 유지.

**보존/주의** — 수수료는 **금액**으로 좌측 `금액` 박스에 위치(수수료율 %와 혼동 금지). 인수증에서 `차주운임` 라벨 금지(=수익). `수익/차주운임` 셀만 결과값으로 강조, 계산식은 muted. 입력 전 보조 chip은 dashed muted로 약화.

**→ 요약 데이터 계약** — `3.화물 운송정보`가 source of truth. 요약으로 흐르는 원본: 품목(높음)·실중량(높음)·톤수(중간)·차종(중간)·대수(낮음, 2대+만). 금액 필드는 요약 미포함.

## 4. 화물정보 요약 (cargo-summary-docs)

**목적** — `3.화물 운송정보`(+`2.운송 구간`)에서 파생되는 **한 문장 read-only 요약** row. 자동 생성값과 사용자 override를 분리 관리. 증빙 docs·금액은 이 섹션에서 제외.

**표시 항목** — `요약` 라벨 chip + 파생 문장(품목·실중량/톤수·차종·대수·상하차 일시/방법) + `수정` 버튼. 모두 read-only 파생값.

**상태** — `Auto`(원본 변경 시 즉시 재생성) / `Overridden`(사용자 보정, 즉시 덮어쓰지 않음) / `Stale`(override 후 원본 변경, 재생성/유지 선택) / `Empty`(placeholder `운송 정보 입력 후 요약됩니다`, action `요약 대기` disabled) / `Editing`(같은 row input).

**인터랙션** — `수정` 또는 값 클릭 → 같은 위치 input 전환 → `저장`. 원본 변경 시 auto=재생성, overridden=stale 분기.

**자동 생성 템플릿** — `{상차조건} {하차조건} {품목} {거점요약}. {실중량 or 톤수} {차종}, {상하차방법} 조건.` (실중량 trailing zero 제거, 대수 1대 생략, 방법 같으면 병합).

**보존/주의** — 모든 값 read-only(입력 필드로 오인 금지, 수정은 명시적 진입). 금액 토큰 요약 배제 → 디자인에서 금액 강조 추가 금지. **긴 문장 처리(말줄임 vs 줄바꿈) 문서/HTML 불일치** — 최신 규칙은 줄바꿈 보존(`white-space:normal`). 디자인 확정 전 1개로 통일 필요.

## 5. 차주 정보 (driver-info)

**목적** — 배차된 차주·차량을 확인/변경. 화주 정보와 **동일 패턴**(한 줄 요약 row, CTA 흡수, 통합 조회 다이얼로그, inline edit). Phase 1 = 내부 DB 조회·배차, Phase 2 = 외부 `화물맨` 연동 배차.

**핵심 필드**

| 필드 | 컨트롤 | 필수 | 비고 |
| --- | --- | --- | --- |
| 차주명 | row value(읽기) | 필수 | 조회로만 변경 |
| 차량번호 | row value(읽기) | 필수 | 조회/등록으로만 변경 |
| 차주 연락처 | inline input | 필수 | 적용 후 inline 수정 가능 |
| 톤수 | inline select | 필수 | **차량 스펙**(≠화물 운송정보의 운송 요구 조건) |
| 차종 | inline select | 필수 | 차량 스펙 |
| 출처 | row column(읽기) | Phase 2 | `화물맨`/`내부 DB` 구분 |

**상태** — Phase 1: Ready/Empty/Dialog/Preview/Register/Editing. Phase 2(화물맨, 다단계): `hm_not_linked→hm_linking→hm_dispatched→적용`, 분기 `hm_canceling`/`hm_cancel_failed`/`hm_manual_driver_applied`/`hm_new_driver→hm_save_required→hm_saved`/`hm_duplicate`/실패. 배지: `화물맨 배차`/`기존 등록`/`신규 차주`/`저장 필요`/`DB 저장됨`/`중복 후보 있음`.

**인터랙션** — 통합 조회 다이얼로그(검색 전 중앙 CTA, 조회 후 우상단 플로팅 `차주 등록`). 차주 등록 폼은 클릭 시에만 생성(미리보기와 동시 노출 금지). 화물맨: `연동` → `화물에 화물을 공유?` 확인 → `연동중`→`배차완료`→`화물맨 차주 적용`. 연동 취소 = 확인 다이얼로그 → 성공 시 입력 전 복귀, 실패 시 버튼 유지 재시도.

**확정** — 진입 UI **A안**(실제 배치 row) / 차주 등록 **A안 하단 CTA** / B 통합본은 Phase 1/Phase 2 **탭형**. Phase 1 먼저 개발, Phase 2는 화물맨 API 계약 확정 후 확장.

**보존/주의** — 화물맨 연동과 내부 DB 조회는 **대체 아닌 병렬** 배차 옵션. 차주 정보의 톤수/차종(차량 스펙)과 화물 운송정보의 톤수/차종(운송 요구)을 **혼동 금지**. 다이얼로그 골격은 화주 정보와 공유 — 화물맨은 상단 우선 행+상태 배지만 얹음. 차주 정보 ≠ 화주 정보(데이터 책임 분리).

## 섹션 공통 패턴 (컴포넌트화 1순위)

| 패턴 | 사용 섹션 | 비고 |
| --- | --- | --- |
| 한 줄 요약 row(라벨 컬럼 + 값 + 액션 버튼) | 화주·운송구간·화물운송정보·요약·차주 | 32px 행 리듬 통일 |
| 통합 조회 다이얼로그(검색→결과→미리보기→적용) | 화주·차주·운송구간(주소) | 동일 레이아웃 재사용 |
| CTA 흡수 전환(빈 상태→다이얼로그) | 화주·차주·운송구간·화물운송정보 | A안 흡수형 |
| inline edit(read↔control 토글) | 전 섹션 | 한 번에 한 셀, Enter/blur 저장, Esc 복원 |
| 상태 배지/칩 | 전 섹션 | 색상만 아닌 텍스트 병행 |
