# 디자인 고도화 결과 검수 체크리스트

> 클로드 디자인 결과를 받은 뒤 점검. **보존(깨지면 안 됨) → 고도화(끌어올렸나) → 미결정(임의 확정 안 했나)** 순.

## A. 보존 (실패 시 재작업)

- [ ] 5개 섹션 순서 유지: 화주 → 운송구간 → 화물운송정보 → 화물요약 → 차주
- [ ] `cargo-list`(화물 목록 테이블)를 임의로 추가하지 않음
- [ ] [01-section-digest.md](01-section-digest.md)의 모든 입력 필드가 매핑됨
- [ ] 금액 의미 구분: 청구비용 / 운송비용 / 수수료(금액) / 수익·차주운임(자동계산)
- [ ] 수수료가 좌측 "금액" 영역에 위치(비율 수수료율과 혼동 없음)
- [ ] 인수증=수익 / 그 외=차주운임 라벨 분기
- [ ] 직접 편집 불가 필드(업체명·사업자번호·담당자명·차주명·차량번호·행정주소)에 편집 어포던스 없음
- [ ] 화주 정보 ≠ 차주 정보, 차량 스펙(차주) ≠ 운송 요구 조건(화물운송정보)
- [ ] inline 편집 read↔control 흐름 유지, 통합 조회 다이얼로그 패턴 유지
- [ ] 화물맨 phase 상태(연동중/배차완료/취소요청중/취소실패) 표현 가능
- [ ] 상태를 색상 + 텍스트 배지 병행으로 구분
- [ ] 라벨 보기 토글 / `aria-label`·`role=dialog`·tablist 등 접근성 골격 유지

## B. 고도화 (목표 달성 여부)

- [ ] 단일 디자인 토큰 체계로 통합(accent/ink vs pen/paper/note 이중 체계 해소)
- [ ] 색 팔레트 정규화(토큰 밖 하드코딩 색 최소화) + 상태색 규칙 일관
- [ ] 모듈러 타입 스케일 + 4/8 스페이싱 스케일 적용
- [ ] 공통 컴포넌트(Button/Field/SummaryRow/Chip/Dialog/EmptyState) 추출·일관 적용
- [ ] 위험 액션(화물취소·배차취소·연동취소·정산처리) danger로 시각 분리
- [ ] 가시적 섹션 헤더(번호+타이틀+구분선)로 스캔 가능
- [ ] 손글씨 폰트 제거(또는 컨셉 명시 채택) — vestigial 자산 정리
- [ ] 포커스 링/키보드 가시성 강화
- [ ] 고밀도 유지(핵심 필드를 탭 뒤에 숨기지 않음)
- [ ] 업무 화면 톤(랜딩/카드형 대시보드 아님)

## C. 반응형 / 상태 예시

- [ ] Desktop 1280px+: 운송구간 full-width 2열 + 섹션 세로 스택
- [ ] Tablet 세로 스택(섹션 묶음 유지) / Mobile 핵심입력 + 하단 시트 제안
- [ ] 페이지 전체 가로 overflow 없음
- [ ] 상태 예시 제공: empty / applied / inline 편집 / overridden / readonly / error
- [ ] 다이얼로그 예시: 검색 전 / 결과 / 미리보기 / 적용 / confirm

## D. 미결정 (임의 확정 안 했는지)

- [ ] 요약 긴 문장: 줄바꿈 보존(말줄임 강제 안 함)
- [ ] 수수료 숨김 vs 비활성을 한쪽으로 못박지 않음(옵션/확장 여지)
- [ ] 상태 chip 노출 여부를 임의 확정하지 않음
- [ ] 화물맨 중복/기존 차주 판정 로직을 화면이 강제하지 않음
- [ ] API·validation 미가정(화면은 상태 표현만)
