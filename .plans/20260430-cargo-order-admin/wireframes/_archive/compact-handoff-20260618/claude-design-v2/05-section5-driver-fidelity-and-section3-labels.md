## 1. 섹션 5 — 차주 정보 (원본 정밀 일치)

### 원본 차주 섹션 UX/UI (기준)

- **패널/탭**: phase 탭 `[내부 DB][화물맨]`(role=tablist) — 패널 본문 상단.
- **Phase 1 행**: `차주`(kind) → 차주명(잠금) → 차량번호(잠금) → 연락처(inline) → 톤수(inline select) → 차종(inline select) → **출처(값 텍스트 "내부 DB")** → `[차주/차량 변경]`. 셀마다 label+value.
- **Phase 1 meta-row**(aria-live): `chip.on 상태` + `chip 출처` + 결과문구(예: "임동진 차주가 적용되었습니다.").
- **Phase 2(화물맨) 상태**:
  - 입력 전: `[화물맨 연동]` + placeholder + **`[차주/차량 입력]` 병렬**(3요소).
  - 연동중 → `[연동 취소][차주/차량 입력]` / 배차완료 → `[연동 취소][화물맨 차주 적용]`.
  - 취소요청중 → `[취소 요청중(disabled)]` / 취소실패 → `[연동 취소][차주/차량 입력]`.
  - meta-row: 상태 chip + 출처 chip + 결과문구 + **mini-action `[배차완료 수신][취소실패 수신]`**(상태별 hidden 토글).
- **차주 조회 다이얼로그**:
  - status-strip: `모드배지 chip` + 상태문구 + **모드도구 2버튼 `[내부 DB 조회][화물맨 배차 결과]`**.
  - 검색바: input + `[조회]` + **`[검색 초기화]`**(버튼 2개).
  - filter-row: **검색 기준 6칩** `[통합검색(on)][차주명][차량번호][연락처][톤수][차종]`.
  - 결과(좌): **화물맨 외부 블록(상단 고정, "검색어와 무관하게 상단 고정" chip)** + presearch 안내 + floating-register + **내부 DB 결과(건수 chip)**. 7열 헤더(선택/차주명/차량번호/연락처/톤수/차종/**상태 or 최근 운행**).
  - preview(우): side-guide + **7필드**(차주명·차량번호·연락처·톤수·차종·**출처·상태**) + register-card 인페이지 마운트.
  - register-card: badge "신규 등록" + 5필드(차주명·차량번호·연락처·톤수 select·차종 select) + `[취소][차주 등록]`.
  - foot: **apply-note chip**("선택된 차주 정보를 현재 row에 적용합니다.") + `[취소][차주 정보에 적용]`.

## 2. 섹션 3 — 화물 운송정보 라벨 중첩 정리

### 중첩 원인 (현재본)

섹션 3은 F안 2블록이라 각 블록에 **블록 헤더**(`money-block__head`)가 있다. 그런데 블록 본문의 인라인 행에도 **`kind` 칩**이 있어 같은 단어가 겹친다:

| 블록 | 블록 헤더 | 행 내 kind 칩 | 셀 라벨 | 중복 |
|---|---|---|---|---|
| 운송+품목 | `운송 + 품목` | `운송`(cond 행), `품목`(item 행) | …, `품목`(품목 셀) | **"운송"×2, "품목"×3** |
| 금액 | `금액` | `금액`(money 행) | 결제방법/청구비용/… | **"금액"×2** |

→ 블록 헤더가 이미 블록을 라벨링하므로, 행의 `kind` 칩("운송"/"품목"/"금액")은 **중복**이고, "품목"은 헤더·kind·셀 라벨로 **3중** 노출된다. (좁은 좌측 열에서 더 어수선해 보임.)

### 교정

- 섹션 3의 인라인 행에서 **`kind` 칩(운송/품목/금액)을 제거**한다(블록 헤더가 라벨 역할).
- 그리드의 **첫 kind 컬럼(48px)도 제거**해 빈 칸이 생기지 않게 한다.
  - `irow--cond`: `48px 70px 80px 54px minmax(96px,1fr)` → 앞 48px 제거.
  - `irow--item`: `48px minmax(0,1fr)` → 앞 48px 제거.
  - `irow--money`: `44px 78px 80px 80px 64px 84px auto` → 앞 44px 제거.
- 품목 행은 kind 제거 후 **셀 라벨 "품목"만 단일** 유지.
- **빈 상태(입력 전)도 동일**(기본 화면에서 가장 잘 보이는 중복): 두 블록의 빈 행(`emptyRow`)이 블록 헤더와 같은 "운송+품목"/"금액" 라벨을 또 표시한다. 빈 행의 맨 앞 라벨(kind)도 제거하고 placeholder + CTA만 둔다. (헤더가 라벨 역할.) → prompt-06은 채워진 행만 정리했으므로, 이 빈-상태 건은 **prompt-07(현 PROMPT-LOG.md Step 6)로 분리 처리**(차주 탭 통합·2행 배치와 함께).
- 다른 섹션(화주·운송구간·차주)의 `kind` 칩은 **그대로 둔다**(블록 헤더가 없어 중복 아님 — 빈 상태 포함).
- 좌측 절반 열 폭에서 행이 넘치면 줄바꿈/내부 스크롤로, 라벨이 시각적으로 겹치지 않게.
