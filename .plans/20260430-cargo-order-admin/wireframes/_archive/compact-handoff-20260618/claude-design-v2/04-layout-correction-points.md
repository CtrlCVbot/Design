# 04. 레이아웃 교정 포인트 — Planned.html을 원본 배치에 정렬

> **배치 권위(SSOT)**: `../results/wireframe/order-register-new2.0/Cargo Order B Implementation Clean.html`
> **교정 대상**: `Cargo Order Planned.html`
> **원칙**: **섹션 내부 컴포넌트의 배치·크기만** 원본과 일치시킨다. 비주얼(토큰·컴포넌트)은 ds.css 유지,

## 핵심 차이 (한 줄 요약)

원본은 각 섹션을 **"한 줄에 모든 필드를 펼친 고밀도 가로 인라인 행"**(셀마다 라벨+값+인라인 편집)으로 둔다.
Planned.html은 이를 **세로 summary-row 스택 / 좌우 2열 분리 / 펼침(detail) 패널**로 재해석해 밀도와 위치가 어긋났다.
→ 교정 방향: **세로 분해·펼침 패널을 걷어내고, 원본의 가로 인라인 행 배치로 복원**(스타일은 ds.css).

## 확인 필요 / 충돌 메모

- **다이얼로그 vs 인라인 (섹션 2·3)** *(정정)*: 원본은 **둘 다** 갖는다 — resting/applied 상태는 가로 인라인 행,
  최초 입력은 다이얼로그(`b-cargo-dialog` 운송+품목/금액)와 주소 검색 다이얼로그. 일시/방법은 인라인 트리거 + 팝오버 + 지정 다이얼로그.
  → 결정: **배치(resting/applied)는 원본의 가로 인라인 행**으로 맞추고, **입력/조회 다이얼로그도 원본의 내부 배치로 맞춘다**
