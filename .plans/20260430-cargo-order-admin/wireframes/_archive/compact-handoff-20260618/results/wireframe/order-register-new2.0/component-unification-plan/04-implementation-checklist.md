# Implementation Checklist

## 목적

이 체크리스트는 문서 검토 후 `Cargo Order B Implementation Clean.html`에 실제 통일 기준을 반영할 때 사용합니다.

이번 문서 패키지 작성 단계에서는 HTML을 직접 수정하지 않았습니다. 아래 항목은 후속 작업용입니다.

## 적용 전 준비

| 항목 | 상태 |
| --- | --- |
| 기준 HTML 백업 또는 원본 보존 확인 | 필요 |
| 현재 clean HTML 브라우저 캡처 | 필요 |
| 추천안 `Option D+` 확정 | 필요 |
| 우측 빈공간 제외 범위 재확인 | 필요 |
| 차주 정보 기준이 화물맨 연동 버전임을 재확인 | 필요 |

## 작업 목록

### T1. `라벨 보기` 토글 추가

| 항목 | 내용 |
| --- | --- |
| 우선순위 | 높음 |
| 대상 | 화면 전체 또는 좌측 작업 영역 상단 |
| 작업 | `라벨 보기` 토글을 추가하고 OFF/ON 상태를 class로 제어 |
| 검증 | OFF에서는 compact, ON에서는 대상 row가 2행 라벨 모드로 전환되는지 확인 |
| 완료 기준 | 사용자가 한 번의 토글로 compact/detail 모드를 전환할 수 있음 |

세부 항목:

| 항목 | 기준 |
| --- | --- |
| 기본값 | OFF |
| 저장 | 사용자별 preference |
| 위치 | 좌측 작업 영역 상단 또는 고정 toolbar |
| 문구 | `라벨 보기` 우선 검토 |
| 범위 | 화주 정보, 화물 운송정보, 차주 정보 우선 적용 |

### T5. 라벨 적용 범위 정리

| 항목 | 내용 |
| --- | --- |
| 우선순위 | 높음 |
| 대상 | 전체 section |

| 검증 | compact, hover/focus, 라벨 보기 ON 상태가 각각 문서와 일치하는지 확인 |
| 완료 기준 | 라벨을 항상 보이는 값과 토글/hover로만 보이는 값이 설명 가능 |

세부 항목:

| 필드 | 권장 라벨 방식 |

| 결제방법, 청구비용, 운송비용, 수수료, 수익/차주운임 | compact에서도 최소 라벨 유지 가능 |
| 요약 본문 | 기본 라벨 생략, hover 수정만 유지 |
| 차주명, 차량번호, 연락처, 톤수, 차종, 출처 | OFF hover/focus, ON 2행 |

## 검증 방법

| 검증 | 방법 | 기준 |
| --- | --- | --- |
| 화면 캡처 비교 | 적용 전/후 desktop viewport 캡처 | 중첩 박스와 버튼 불일치가 줄었는지 |
| 라벨 모드 비교 | 라벨 보기 OFF/ON 캡처 | compact와 2행 라벨 모드가 모두 읽히는지 |
| hover/focus 확인 | 필드 hover와 keyboard focus | 임시 라벨이 뜨고 row height가 변하지 않는지 |
| DOM 확인 | 주요 section text와 hidden 상태 확인 | 불필요한 설명 문구 미노출 |
| interaction smoke test | 주요 버튼 클릭 | 다이얼로그 열림/닫힘 유지 |
| 상태 전환 확인 | 화물 운송정보 적용, 화물맨 연동 상태 | row 위치와 문구 안정성 유지 |
| 접근성 기본 확인 | focus 가능한 요소 순서 확인 | keyboard로 주요 action 접근 가능 |

## 적용 후 체크포인트

| 체크포인트 | 통과 기준 |
| --- | --- |
| 라벨 토글 | OFF compact, ON 2행 라벨 모드가 명확함 |
| 임시 라벨 | hover/focus 시 해당 field 상단에만 표시됨 |
| 버튼 | 같은 역할 버튼이 같은 스타일과 위치 규칙을 따름 |
| 입력 전 | 각 입력 전 row가 짧고 명확함 |
| 적용 후 | 상태가 바뀌어도 필드 위치가 안정적 |
| 수정 가능 | inline edit와 dialog 변경이 구분됨 |
| 구분선 | section 외곽과 내부 row 구분이 겹치지 않음 |

## 권장 실행 순서

1. `라벨 보기` 토글과 상태 class를 먼저 추가합니다.
2. 화주 정보, 화물 운송정보, 차주 정보에 2행 라벨 모드를 적용합니다.
3. hover/focus 임시 라벨을 적용하고 row height jump를 확인합니다.
4. 버튼 역할 체계를 CSS class 수준으로 정리합니다.
5. 입력 전 상태 문구를 축약합니다.
6. 캡처 비교 후 row 간격과 컬럼 폭을 미세 조정합니다.
