# 화물 오더 Wireframe UX/UI 리뷰

## 1. 요약

두 wireframe 중 High Fidelity 작업의 주 기반으로는 `Cargo Order Wireframe.html`이 더 적합합니다. 원본 화면의 핵심 구조인 `상단 입력/수정 영역 + 하단 목록`, `좌측 입력 + 우측 조건` 구성이 비교적 잘 유지되어 있기 때문입니다.

다만 확정 B안 기준으로 보면 `화물 운송정보`, `상차/하차 조건`, `배차 유형`, `수수료 금액`, `수수료율 정보`의 위치가 일부 섞여 있습니다. High Fidelity로 가기 전에 이 정보 구조를 먼저 바로잡아야 합니다.

`specs/04 Wireframe Spec Form.html`은 명세서/출력물/위탁증 스타일로는 장점이 큽니다. 하지만 일상적인 오더 입력·수정 화면으로 쓰기에는 원본과 업무 감각이 많이 달라져 기존 사용자의 적응 비용이 큽니다.

## 7. UX/UI 개선점

### 7.1 1안에서 유지하면 좋은 점

| 장점 | 이유 |
| --- | --- |
| 원본과 유사한 전체 골격 | 기존 사용자 적응 비용이 낮음 |
| 섹션 번호와 라벨 | 초보자가 입력 순서를 따라가기 쉬움 |
| 상차 조건/하차 조건 분리 | 작업 방식과 시간 조건이 명확함 |
| 화주 정보 분리 | 의뢰자/화주 의미가 잘 드러남 |
| 상태 탭과 목록 | 하단 목록 기반 운영 흐름을 유지함 |
| 상태 배지 | 접수/완료/취소를 색상과 텍스트로 구분함 |

### 7.2 1안에서 개선해야 할 점

| 개선점 | 현재 문제 | 권장 조치 |
| --- | --- | --- |
| `화물 운송정보` 재정렬 | 상차/하차일시와 배차 유형이 들어가 있음 | 톤수, 차종, 도착, 대수, 품목, 운송료, 수수료, 합계 중심으로 재구성 |
| `품목` 위치 이동 | 현재 `화물 요약·중량·증빙`에 있음 | `화물 운송정보`로 이동 |
| `수수료`와 `합계` 복구 | 우측 정산 영역에 수수료율/운임이 섞임 | 좌측 `화물 운송정보`에 운송료/수수료/합계를 함께 배치 |
| `수수료율 정보` 순수화 | 기준 운송료, 차주 지급, 마진까지 섞임 | 우측에는 주선약관 수수료율 표만 남기고 정산 보조로 처리 |
| 버튼 라벨 복원 | `저장`, `오더취소` 등으로 바뀜 | `화물등록`, `화물수정`, `화물취소`, `배차취소` 원문 라벨 병기 |
| 목록 컬럼 보강 | 정산/사업자/계산서 컬럼이 축약됨 | 원본 컬럼을 기본/확장 컬럼으로 보존 |
| 날짜 정합성 | 예시가 `2025-04-30`으로 되어 있음 | 기준 캡처 흐름에 맞춰 `2026-04-30` 또는 중립 placeholder 사용 |

### 7.3 2안에서 가져오면 좋은 점

| 장점 | 활용 방식 |
| --- | --- |
| 문서형 번호 체계 | 팝업 도움말, 출력물, 위탁증, 거래명세서 화면에 활용 |
| 명세서형 화물 목록 | `화물 요약·중량·증빙`의 상세 보기로 활용 |
| 서명/확인 영역 | 정산/서명확인/위탁증 출력 흐름에 활용 |

### 7.4 2안을 주 화면으로 쓰기 어려운 이유

| 문제 | 영향 |
| --- | --- |
| 입력 화면보다 문서 양식에 가까움 | 실제 오더 접수 속도가 떨어질 가능성 |
| 원본 화면의 앱형 조작 감각이 약함 | 기존 사용자 적응 비용 증가 |

| 화면 세로 길이가 길어짐 | 한 화면 확인 가치가 약해짐 |

## 9. 권장 반영안

### 9.1 High Fidelity 기준안

High Fidelity는 `Cargo Order Wireframe.html`을 기반으로 진행하는 것을 권장합니다.

이유:

1. 원본 업무 화면의 큰 골격을 가장 잘 유지합니다.
2. 기존 사용자에게 낯선 변화가 상대적으로 적습니다.
3. 하단 목록 기반 업무가 유지됩니다.
4. B안 구조로 수정하기 위한 변경량이 2안보다 적습니다.

### 9.2 1안 수정 방향

```text
좌측 오더 입력
1. 운송 구간
   - 상차지 / 하차지

2. 화물 운송정보
   - 톤수 / 광폭 / 차종 / 도착 / 대수 / 품목
   - 운송비구분 / 운송료 / 수수료 / 합계

3. 화물 요약·중량·증빙
   - 화물정보 요약
   - 중량(톤수 110%)
   - 전자세금계산서 / 인수증 우편발송 / 이사화물 / 안내문

4. 차주 정보
   - 차주명 / 차량번호 / 차주 연락처 / 복지

우측 조건
5. 배차 유형
   - 독차 / 혼적 / 긴급 / 왕복 / 예약 / 경유

6. 상차 조건
   - 상차방법 / 상차일시 / 상차지 전화 / 초기화

7. 하차 조건

   - 주선약관 수수료율 표
```

### 9.3 2안 활용 방향

2안은 주 입력 화면으로 쓰기보다 아래 용도로 분리하는 것이 좋습니다.

| 활용처 | 이유 |
| --- | --- |
| 위탁증 출력 화면 | 서명/확인/보관용 양식에 적합 |
| 거래명세서 미리보기 | 명세서형 구조와 잘 맞음 |
| 인수증/세금계산서 보조 화면 | 증빙 문서 감성이 강함 |
| 오더 상세 인쇄 양식 | 번호형 항목과 고정 양식이 유리 |
