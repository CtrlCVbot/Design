# 상태 및 상호작용 매트릭스

## 목적

이 문서는 화면 노출명 `보조 정보`, 내부 작업명 `예약 영역` 3개 탭의 상태, 액션, 전환, motion/feedback 기준을 한 곳에서 확인하기 위한 매트릭스입니다.

구현팀은 이 문서를 기준으로 상태별 UI 누락을 확인하고, 기존 접수/수정 섹션의 motion 방향과 충돌하지 않도록 구현합니다.

| 탭 | 액션 | 결과 | 실패 시 |
| --- | --- | --- | --- |
| `메모` | `메모 추가` | 메모 작성 dialog 열기 | dialog를 열 수 없으면 오류 notice 표시 |
| `메모` | `저장` | 새 메모가 리스트 상단에 추가되고 dialog 닫힘 | dialog 입력값 유지, 오류 notice 표시 |
| `메모` | `취소` | dialog 닫기, 입력값 폐기 또는 이탈 확인 | 별도 실패 없음 |
| `금액 로그` | `상세 항목 보기` | 금액 명세 안의 `대기비`, `경유비`, `할인`, `기타 조정` row 펼침 | 기존 명세 유지, 오류 notice 표시 |
| `금액 로그` | `금액 입력으로 이동` | 기존 금액 입력 다이얼로그 또는 금액 영역으로 이동 | 정산 처리됨이면 수정 제한 안내 |
| `금액 로그` | `다시 불러오기` | 로그 재조회 | 오류 상태 유지 |
| `운송 구간 지도` | `주소 입력으로 이동` | 기존 `2. 운송 구간`의 주소 입력 위치로 focus 이동 | 이동 대상이 없으면 안내 notice |
| `운송 구간 지도` | `경로 다시 계산` | 지도와 메타 재조회 | 기존 값 유지, 오류 notice 표시 |

## 탭 전환 규칙

| 상황 | 규칙 |
| --- | --- |
| 최초 진입 | `메모`를 기본 탭으로 엽니다. |
| 같은 오더에서 탭 전환 | 선택한 탭 상태를 유지합니다. |
| 다른 오더로 변경 | 탭 본문 데이터를 새 오더 기준으로 다시 로딩합니다. |
| 메모 dialog 작성 중 탭 전환 | dialog가 열린 상태에서는 배경 탭 전환을 막습니다. 닫기 시에는 입력값 폐기 확인을 검토합니다. |
| 오류 탭에서 다른 탭으로 이동 | 이동을 막지 않습니다. 오류는 해당 탭 안에만 남깁니다. |
| keyboard | 탭은 좌우 방향키 또는 Tab focus로 이동 가능해야 합니다. |

## Motion 및 Feedback 기준

기존 결정 사항을 유지합니다.

| 기준 | 예약 영역 적용 방식 |
| --- | --- |
| 입력 버튼은 Clean 기준의 sliding absorb 참고 | 예약 영역에서는 직접 입력 버튼이 적으므로, 기존 섹션으로 이동하는 버튼에만 가볍게 적용 검토 |
| 적용 후 row 흔들림 없이 field flash 사용 | 새 메모 item, 금액 명세 row, 지도 메타처럼 실제 변경된 요소만 flash |
| `화물맨 연동`은 별도 handoff dot/pulse | 예약 영역 탭에서는 화물맨 motion을 재사용하지 않음 |
| 상태 안내 | `aria-live` 또는 visible status text로 저장/오류/계산 상태 안내 |
| reduced motion | `prefers-reduced-motion`에서는 animation 없이 상태만 즉시 전환 |
| 디자인 토큰 | 신규 animation 색상은 만들지 않고 `Design System.html`의 색상/표면 토큰 안에서만 처리 |

## 접근성 기준

| 항목 | 기준 |
| --- | --- |
| 탭 semantics | 탭 목록은 `tablist`, 탭은 `tab`, 본문은 `tabpanel` 구조를 권장합니다. |
| 현재 탭 | 선택된 탭에는 `aria-selected="true"`를 둡니다. |
| 숨김 본문 | 비활성 본문은 focus가 들어가지 않게 처리합니다. |
| 로딩 | 로딩 중에는 visible text와 `aria-busy`를 함께 검토합니다. |
| 오류 | 오류는 색상만으로 표현하지 않고 문구와 재시도 액션을 제공합니다. |
| 메모 dialog | dialog title, textarea label, 글자 수 제한, 저장 실패 안내가 필요합니다. |

## 기존 섹션과 충돌 방지 규칙

| 충돌 가능성 | 방지 규칙 |
| --- | --- |
| 주소 수정 위치 중복 | 지도 탭은 주소를 직접 수정하지 않고 기존 운송 구간으로 이동합니다. |
| 금액 수정 위치 중복 | 금액 로그 탭은 읽기 중심이며 수정은 기존 금액 입력 다이얼로그로 연결합니다. |
| 메모와 금액 사유 중복 | 공식 금액 변경 사유는 금액 로그에 남기고, 사람의 보조 설명만 메모에 남깁니다. |
| 지도 주소 표시 중복 | 지도 탭은 상차/하차 전체 주소를 하단에 반복하지 않고 핀, 상태 chip, 기존 섹션 이동으로 처리합니다. |
| 금액 로그 과밀 | 첫 화면에는 기준금액/조정금/합계/수익만 두고 항목별 담당자/시각/사유는 펼침 또는 dialog로 분리합니다. |
| 화면 밀도 과다 | 우측 탭은 요약과 리스트 중심으로 구성하고 장문 설명을 줄입니다. |
| motion 중복 | 좌측 입력 적용 feedback과 우측 탭 feedback을 동시에 크게 흔들지 않습니다. |

## 탭별 MVP 체크리스트

| 탭 | MVP 체크 |
| --- | --- |
| `메모` | 메모 없음, dialog 입력 중, 저장 중, 저장 실패, 데이터 있음 상태가 모두 존재 |
| `금액 로그` | 조정 내역 없음, 조회 중, 조회 실패, 요약 있음, 상세 펼침, 정산 처리됨 상태가 모두 존재 |
| `운송 구간 지도` | 운송 구간 없음, 부분 입력, 계산 중, 오류, 데이터 있음 상태가 모두 존재 |

## 공통 Acceptance Criteria

| ID | 기준 |
| --- | --- |
| `STATE-AC-01` | 모든 탭은 빈 상태, 로딩 상태, 오류 상태, 데이터 있음 상태를 가져야 합니다. |
| `STATE-AC-02` | 탭 전환은 한 탭의 오류 때문에 막히지 않아야 합니다. |
| `STATE-AC-03` | 기존 접수/수정 섹션의 입력 책임과 `보조 정보` 탭의 확인 책임이 분리되어야 합니다. |
| `STATE-AC-04` | 데이터가 변경되었을 때는 변경된 field 또는 item만 feedback을 받아야 합니다. |
| `STATE-AC-05` | keyboard와 screen reader 사용자가 현재 탭과 오류 상태를 이해할 수 있어야 합니다. |
| `STATE-AC-06` | reduced motion 환경에서도 상태 변경 의미가 사라지지 않아야 합니다. |
| `STATE-AC-07` | 우측 `보조 정보` 첫 화면은 요약 중심이어야 하며, 메모 작성과 금액 상세는 dialog 또는 펼침으로 분리되어야 합니다. |
