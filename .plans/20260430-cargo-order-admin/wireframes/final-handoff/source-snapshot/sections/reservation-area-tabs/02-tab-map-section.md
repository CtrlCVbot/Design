# 운송 구간 지도 탭 상세 기획

## 목적

`운송 구간 지도` 탭은 상차지와 하차지를 지도 위에 표시하고, 운영자가 이동 구간의 현실성을 빠르게 판단하게 만드는 탭입니다.

이 탭은 주소를 수정하는 곳이 아닙니다. 주소 입력과 수정은 기존 `2. 운송 구간` 섹션에서 처리하고, 이 탭은 그 결과를 시각적으로 확인하는 역할을 맡습니다.

## 핵심 질문

| 질문 | 표시해야 할 정보 |
| --- | --- |
| 어디에서 어디로 가는가? | 상차 핀, 하차 핀, 짧은 지명 또는 상태 |
| 거리가 어느 정도인가? | 거리 km, 산정 기준 |
| 얼마나 걸리는가? | 예상 시간, 교통 반영 여부 |
| 경유가 있는가? | 경유 여부, 경유 수, 경유지 표시 정책 |
| 주소가 충분한가? | 상차지/하차지 주소 미입력 또는 좌표 변환 실패 상태 |

## 데이터 입력

| 데이터 | 출처 | 필수 여부 | 메모 |
| --- | --- | --- | --- |
| 상차지 주소 | 기존 `2. 운송 구간` | 필수 | 주소가 없으면 지도 대신 입력 유도 상태 표시 |
| 하차지 주소 | 기존 `2. 운송 구간` | 필수 | 주소가 없으면 경로 계산 불가 |
| 상차/하차 상세주소 | 기존 `2. 운송 구간` | 선택 | 지도 핀 라벨에는 너무 길게 표시하지 않음 |
| 거리 | 기존 계산 메타 또는 지도 API | 선택 | 현재 화면에는 `계산 메타`의 거리 chip이 존재 |
| 예상 시간 | 지도/경로 API | 선택 | MVP에서는 mock 또는 `계산 대기` 상태 허용 |
| 경유 여부 | 배차/운송 조건 또는 후속 정책 | 선택 | MVP에서는 `경유 없음`, `경유 있음`, `확인 필요`로 표현 |

## UI 구성 요소

| 영역 | 구성 요소 | 설명 |
| --- | --- | --- |
| 탭 요약 | 상태 chip | `경로 계산 완료`, `주소 필요`, `계산 실패` 등 |
| 지도 영역 | 지도 canvas 또는 지도 placeholder | MVP에서는 실제 지도 provider 확정 전에도 정적 지도 placeholder로 설계 가능 |
| 핀 | 상차 핀, 하차 핀 | 색과 라벨로 상차/하차를 구분 |
| 경로선 | 상차지에서 하차지까지의 선 | 경유가 있으면 꺾인 선 또는 경유 점 추가 |
| 메타 행 | 거리, 예상 시간, 경유 여부 | 1줄 또는 2줄 이내로 빠르게 스캔 가능해야 함 |
| 주소 기준 안내 | `2. 운송 구간 기준`, 주소 확인 상태 | 상차/하차 전체 주소를 하단에 반복하지 않고, 필요할 때 기존 운송 구간 섹션으로 연결 |
| 보조 액션 | `주소 입력으로 이동`, `경로 다시 계산` | 직접 주소 수정이 아니라 기존 섹션 또는 계산 재시도에 연결 |

## 중복 표시 정책

좌측 `2. 운송 구간` 섹션이 이미 상차지와 하차지 주소의 원본 입력 위치입니다. 따라서 지도 탭 하단에 상차/하차 주소를 다시 긴 summary row로 반복하지 않습니다.

지도 탭은 아래 정보에 집중합니다.

| 표시 | 방식 |
| --- | --- |
| 상차/하차 구분 | 지도 핀 라벨 또는 짧은 tooltip |
| 주소 상태 | `주소 확인됨`, `상차지 필요`, `하차지 필요`, `좌표 확인 필요` chip |
| 상세 주소 확인 | `2. 운송 구간 기준` 안내 또는 기존 주소 입력 위치로 이동 |
| 경로 판단 | 거리, 예상 시간, 경유 여부, 계산 상태 |

## 상태별 기획

| 상태 | 조건 | UI | 주요 액션 |
| --- | --- | --- | --- |
| 빈 상태 | 상차지와 하차지가 모두 없음 | 메모 탭과 같은 empty box 패턴으로 `운송 구간을 계산할 수 없습니다.`와 짧은 안내 표시 | `운송 구간 입력` |
| 부분 입력 | 상차지 또는 하차지 중 하나만 있음 | 입력된 쪽은 핀 또는 상태 chip으로 표시, 없는 쪽은 placeholder | `운송 구간 입력` |
| 로딩 상태 | 좌표 변환 또는 경로 계산 중 | 지도 skeleton, `경로 계산 중` chip | 탭 전환 가능, 계산 취소는 MVP 제외 |
| 오류 상태 | 좌표 변환 실패, API 실패, 경로 계산 실패 | `경로를 계산하지 못했습니다` notice | `다시 계산`, `주소 확인` |
| 데이터 있음 | 상차/하차 좌표와 거리 또는 경로가 있음 | 지도, 핀, 경로선, 거리/예상 시간/경유 메타 | `경로 다시 계산` |
| 주소 불완전 | 주소는 있으나 상세 위치가 불분명함 | 지도는 대략 위치, warning notice | `주소 확인` |

## 데이터 있음 상태 예시

| 요소 | 예시 |
| --- | --- |
| 핀 라벨 | `상차`, `하차` |
| 주소 상태 | `상차/하차 좌표 확인됨` |
| 주소 기준 | `2. 운송 구간 기준` |
| 거리 | `391km` |
| 예상 시간 | `4시간 35분` |
| 경유 | `경유 없음` |
| 기준 문구 | `예상 시간은 교통 상황과 상하차 대기 시간을 포함하지 않습니다.` |

## 기존 화면과의 연결

| 기존 요소 | 연결 방식 |
| --- | --- |
| `renderRoute()`의 상차/하차 데이터 | 지도 탭의 핀/주소 상태를 결정합니다. 전체 주소 표시는 기존 `2. 운송 구간`을 기준으로 합니다. |
| `route-meta`의 거리 chip | 지도 탭의 거리 메타와 같은 값을 사용합니다. 값이 다르면 기존 `route-meta`를 우선합니다. |
| 주소 검색 버튼 | 지도 탭에서는 주소 검색을 직접 열기보다 기존 운송 구간의 주소 입력 위치로 focus 이동하는 방식을 권장합니다. |
| field flash | 주소 적용 후 지도 탭이 열려 있으면 지도 메타 영역만 flash합니다. 좌측 row 흔들림은 사용하지 않습니다. |

## MVP 범위

| 포함 | 제외 |
| --- | --- |
| 상차/하차 핀과 주소 상태 표시 | 실제 실시간 교통 반영 |
| 지도 placeholder 또는 provider 확정 시 지도 표시 | 복수 경유지 편집 |
| 거리와 예상 시간 표시 영역 | 경로 최적화 |
| 주소 없음/부분 입력/오류 상태 | 지도 위 직접 핀 드래그 |
| 기존 주소 입력 섹션으로 이동하는 액션 | 주소 자동 보정 저장 |

## 후속 확장 범위

| 확장 | 설명 |
| --- | --- |
| 지도 provider 연동 | Kakao Maps, Naver Maps, Google Maps 등 실제 provider 정책 결정 후 연결 |
| 다중 경유지 | 경유지가 여러 개인 경우 순번과 거리 변화를 표시 |
| 교통/차종 반영 | 톤수, 차종, 도로 제한, 실시간 교통을 예상 시간에 반영 |
| 지도 기반 검증 | 상차지와 하차지가 동일하거나 비정상적으로 가까운 경우 경고 |
| 배차 리스크 표시 | 장거리, 산간/도서, 야간 상하차 등 운영 리스크 표시 |

## Acceptance Criteria

| ID | 기준 |
| --- | --- |
| `MAP-AC-01` | 상차지와 하차지가 모두 있으면 지도 또는 지도 placeholder와 함께 경로 요약이 표시되어야 합니다. |
| `MAP-AC-02` | 상차지와 하차지가 모두 없으면 메모 탭과 같은 empty box 패턴으로 표시하고, 일부만 없으면 누락된 주소가 무엇인지 명확히 보여야 합니다. |
| `MAP-AC-03` | 거리, 예상 시간, 경유 여부는 각각 독립 label로 표시되어야 합니다. |
| `MAP-AC-04` | 지도 탭에서 주소를 직접 수정하지 않고 기존 `2. 운송 구간` 입력 흐름으로 연결해야 합니다. |
| `MAP-AC-05` | 경로 계산 실패 시 탭 전체가 비지 않고 오류 메시지와 재시도 액션이 보여야 합니다. |
| `MAP-AC-06` | 좌측 운송 구간의 주소 적용 motion과 충돌하지 않도록 지도 탭 feedback은 지도 메타 영역 중심으로 제한해야 합니다. |
| `MAP-AC-07` | 좌측 `2. 운송 구간`과 겹치지 않도록 상차/하차 전체 주소를 하단 summary row로 반복 표시하지 않아야 합니다. |

## 확인 필요 정책

| 항목 | 확인 필요 내용 |
| --- | --- |
| 지도 provider | 어떤 지도 API를 사용할지, 사용량/요금/쿼터 정책은 무엇인지 확인 필요 |
| 좌표 저장 | 주소를 좌표로 변환한 값을 저장할지, 매번 계산할지 확인 필요 |
| 예상 시간 기준 | 실시간 교통 포함 여부와 상하차 대기 시간 포함 여부 확인 필요 |
| 경유 데이터 출처 | 경유 여부가 운송 조건, 주소 데이터, 별도 입력 중 어디에서 오는지 확인 필요 |
| 개인정보 | 상세주소와 담당자 정보가 지도 탭에 얼마나 노출되어야 하는지 확인 필요 |
