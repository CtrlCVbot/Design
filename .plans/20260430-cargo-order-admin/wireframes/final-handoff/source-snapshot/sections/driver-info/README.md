# 차주 정보 섹션 기획 패키지

## 목적

이 패키지는 화물 오더 접수/수정 화면의 `5. 차주 정보` 섹션을 `1. 화주 정보` 섹션과 동일한 디자인/기능 패턴으로 정리하기 위한 문서 묶음입니다.

화주 정보에서 확정한 `한 줄 요약 row`, `입력 전 CTA 흡수 전환`, `차주/차량 통합 조회 다이얼로그`, `적용 후 일부 항목 inline edit`, `변경 버튼 A안 흡수 애니메이션`을 차주 정보에 맞게 변환합니다.

## 기준

| 항목 | 기준 |
| --- | --- |
| 기준 화면 | B 통합본 `../reference-snapshot/wireframe/Cargo Order Wireframe B Original Tone.html` |
| 기준 섹션 | `5. 차주 정보` |
| 참조 섹션 | `1. 화주 정보` |
| 디자인 톤 | 화주 정보와 동일한 sheet, compact row, 손글씨형 값, 조회 다이얼로그 |
| 핵심 필드 | 차주명, 차량번호, 차주 연락처, 톤수, 차종 |

## 1차 결정

| 결정 | 내용 |
| --- | --- |
| 표시 방식 | `차주` 라벨 + 5개 핵심 필드 + `차주/차량 변경` 버튼 |
| 조회 방식 | 화주 정보와 동일한 통합 조회 다이얼로그 |
| 조회 결과 | 차주명, 차량번호, 차주 연락처, 톤수, 차종을 한 행으로 표시 |
| 적용 방식 | 행 선택 후 `차주 정보에 적용` |
| 임시 수정 | 적용 후 row에서 `차주 연락처`는 input, `톤수`, `차종`은 select 전환 |
| 추가 기능 | `차주 등록`으로 신규 차주명과 차량 정보를 함께 등록 |
| 애니메이션 | 화주 정보의 A안 흡수 전환을 기본값으로 사용 |

## Phase 2 결정

| 결정 | 내용 |
| --- | --- |
| 연동 대상 | `화물맨` 외부 배차 서비스 |
| 입력 전 상태 | `차주` 라벨 옆 `화물맨 연동`, row 우측 `차주/차량 입력` 공존 |
| 진입 UI 추천 | 실제 배치 row 기준 `차주 [화물맨 연동] ... [차주/차량 입력]` |
| 화물맨 연동 상태 | 확인 다이얼로그에서 `YES` 선택 후 `화물맨 화물 연동중` 표시 |
| 화물맨 연동 취소 | `연동 취소` 클릭 시 확인 다이얼로그를 띄우고 `YES`에서 화물맨 API 취소 요청 후 입력 전 상태로 복귀 |
| 다른 차주 배차 | 화물맨 연동 후 내부 DB 차주로 배차되면 `차주/차량 변경` 옆 `연동 취소`를 유지 |
| 배차 완료 알림 | `화물맨 화물 연동중`이 `화물맨 화물 배차완료`로 전환, `화물맨 차주 적용` 클릭 시 통합 조회 다이얼로그 오픈 |
| 배차 완료 표시 | 화물맨 배차 차주 정보를 차주 조회 리스트 상단에 고정 |
| 다이얼로그 레이아웃 | 기존 `차주/차량 통합 조회 다이얼로그` 구조 유지, 화물맨 우선 행과 상태만 추가 |
| 검색 정책 | 검색어와 무관하게 해당 화물의 화물맨 배차 차주 행은 항상 상단 노출 |
| 신규 차주 규칙 | 신규 차주는 `차주 등록` 완료 후에만 배차 적용 가능 |
| 일반 차주 등록 CTA | 검색 전에는 결과 영역 중앙 CTA, 검색 후에는 결과 리스트 우측 상단 플로팅 CTA로 노출 |
| 화물맨 신규 등록 CTA | 화물맨 신규 차주 등록 폼 하단에 `차주 등록` primary 버튼 노출, 등록 전 `차주 정보에 적용` 비활성 |
| 기존 차주 규칙 | 기존 등록 차주는 선택 미리보기 후 바로 적용 가능 |
| 저장 정책 | 자동 저장하지 않고 사용자가 수동으로 `차주 등록` |
| 저장 후 재사용 | `DB 저장됨` 상태가 되면 내부 DB 차주처럼 재사용 |
| 비교/동작 시안 | 기존 Phase 1 HTML은 유지하고 전체 플로우/비교/탭별 상태 섹션/실제 다이얼로그 플로우와 화물맨 연동 CTA, 연동중/배차완료 상태, `출처 / 화물맨` 컬럼, 일반 `차주 등록`, 화물맨 신규 차주 저장 CTA를 포함한 실제 배치형 차주 정보 row 컴포넌트를 별도 HTML로 추가 |
| B 통합본 적용 | `Cargo Order Wireframe B Original Tone.html`의 `5. 차주 정보`를 Phase 1/Phase 2 탭형 실제 컴포넌트로 교체 |
| B Phase 1 탭 | 내부 DB 차주/차량 조회, 차주 등록, 연락처 input, 톤수/차종 select 수정 확인 |
| B Phase 2 탭 | 화물맨 연동, 배차완료 수신, 화물맨 차주 적용, 내부 DB 대체 배차, 연동 취소/취소 실패 재시도 확인 |
| B 통합본 다이얼로그 | 최신 참조 기준의 status strip, 검색 전 안내, 검색/필터, 선택 컬럼, 최근 운행/상태 컬럼, 화물맨 상단 고정 그룹, 우측 가이드/미리보기/등록 전환, 하단 리스트 선명도 보정 반영 |

## 사용 방법

1. `driver-info-section-wireframe.html`을 브라우저에서 열어 상태별 화면을 확인합니다.
2. `02-wireframe-driver-info.md`에서 B 통합본에 들어갈 row와 dialog 구조를 확인합니다.
3. `09-hwamulman-coexistence-ux-plan.md`에서 Phase 1/화물맨 공존 진입 UX를 확인합니다.
4. `10-general-driver-register-phase2-plan.md`에서 일반 `차주 등록`과 화물맨 신규 등록의 역할 분리를 확인합니다.
5. `driver-info-hwamulman-phase2-wireframe.html`에서 화물맨 Phase 2 비교 시안과 화물맨 연동 포함 실제 배치형 차주 정보 row 컴포넌트를 확인합니다.
6. B 통합본의 `5. 차주 정보`에서 Phase 1/Phase 2 탭을 전환하며 실제 배치 컴포넌트 흐름을 확인합니다.
7. B 통합본의 `차주/차량 통합 조회` 다이얼로그에서 내부 DB 조회와 화물맨 배차 결과 상단 고정 표시를 확인합니다.

## 다음 단계

1. 차량번호/연락처 기준 중복 판단 정책을 확정합니다.
2. 화물맨 배차 차주가 기존 등록 차주인지 판정하는 기준을 확정합니다.
3. B 통합본의 Phase 탭 비교안을 기준으로 최종 구현 범위와 배포 순서를 확정합니다.
