# PRD - 차주 정보

## 1. 배경

`5. 차주 정보`는 배차된 차주와 차량을 확인하는 운영 핵심 섹션입니다. 현재 정리안에서는 `차주명`, `차량번호`, `차주 연락처`, `톤수`, `차종`을 핵심 필드로 봅니다.

이번 단계에서는 `1. 화주 정보`와 같은 디자인/기능 패턴을 적용해, 값이 있을 때는 한 줄 요약 row로 보여주고 값이 없을 때는 조회 다이얼로그로 진입할 수 있게 정리합니다. 차주 차량의 `톤수`와 `차종`을 함께 보여줍니다.

## 2. 목표

| 목표 | 설명 |
| --- | --- |
| 디자인 통일 | 화주 정보와 같은 compact row, 변경 버튼, 조회 다이얼로그 사용 |
| 배차 정보 확인성 강화 | 차주명, 차량번호, 연락처, 톤수, 차종을 한 줄에서 확인 |
| 변경 흐름 단순화 | `차주/차량 변경`으로 통합 조회 후 한 번에 적용 |
| 임시 수정 제공 | 연락처는 텍스트 입력, 톤수와 차종은 선택 변경으로 빠르게 보정 가능 |
| 신규 등록 분리 | 조회 목록 선택 미리보기와 신규 차주 등록 폼을 분리해 한 번에 하나의 상태만 표시 |
| 조회 전 혼잡도 축소 | 검색 전에는 결과 목록을 숨기고 `차주 등록` CTA만 안내 영역에 노출 |

## 3. 범위

### 포함

| 항목 | 처리 |
| --- | --- |
| 차주명 | 조회/적용으로 변경 |
| 차량번호 | 조회/등록으로 변경, 적용 후 직접 수정 불가 |
| 차주 연락처 | 적용 후 inline edit 가능 |
| 톤수 | 적용 후 select 변경 가능 |
| 차종 | 적용 후 select 변경 가능 |
| 차주/차량 변경 | 조회 다이얼로그 오픈 |
| 차주 등록 | 신규 차주명과 차량번호, 연락처, 톤수, 차종을 함께 등록 |

### 제외

| 제외 항목 | 이유 |
| --- | --- |
| 실제 차주 조회 API | 이번 단계는 wireframe |
| 차량번호 실검증 | 검증 규칙은 후속 단계 |
| 정산/계약 정보 | 차주 정보 row의 기본 확인 범위를 넘음 |
| 화물 운송 조건 입력 | `3. 화물 운송정보`에서 관리 |

## 4. 요구사항

| ID | 요구사항 | 우선순위 |
| --- | --- | --- |
| DI-01 | 값 있음 상태는 화주 정보와 같은 한 줄 row로 표시한다 | High |
| DI-02 | 값 없음 상태는 `차주 정보 입력` CTA가 `차주` 라벨로 흡수되는 전환을 사용한다 | High |
| DI-03 | `차주/차량 변경` 클릭 시 통합 조회 다이얼로그를 연다 | High |
| DI-04 | 조회 결과 행 선택 후 차주 정보 row에 한꺼번에 적용한다 | High |
| DI-05 | 조회 결과에는 차주 차량의 톤수와 차종을 함께 표시한다 | High |
| DI-06 | 연락처는 같은 위치 input, 톤수와 차종은 같은 위치 select로 임시 수정 가능하다 | Medium |
| DI-07 | `차주 등록` 버튼 클릭 시 선택 미리보기 대신 신규 차주 등록 폼을 표시한다 | Medium |
| DI-08 | 변경 버튼은 화주 정보 A안 흡수 애니메이션과 동일하게 동작한다 | Medium |
| DI-09 | 신규 차주 등록 후 조회 목록에 새 행을 추가하고 자동 선택해 미리보기에 반영한다 | Medium |
| DI-10 | 조회 전에는 결과 리스트를 숨기고, 조회 후에는 리스트 우측 상단에 플로팅 `차주 등록` CTA를 표시한다 | Medium |
| DI-11 | 조회 다이얼로그 결과 행은 고정 컬럼 폭과 정수 line-height/padding으로 하단 리스트 흐림을 방지한다 | Medium |

## 5. Acceptance Criteria

| 기준 | 통과 조건 |
| --- | --- |
| 디자인 | 화주 정보와 같은 row/card/dialog 톤으로 보인다 |
| 상태 | 적용 후 row, 입력 전/조회, 수정 기준, 변경 애니메이션을 HTML에서 확인할 수 있다 |
| 조회 | 다이얼로그에서 행 선택 후 row에 적용 가능하다 |
| 등록 | 차주 등록 후 신규 행이 목록에 추가되고 자동 선택된다 |
| 수정 | 연락처는 input, 톤수와 차종은 select로 전환된다 |
| CTA | 검색 전 중앙 CTA와 검색 후 플로팅 CTA가 중복 없이 전환된다 |
| 시각 품질 | 결과 리스트와 선택 미리보기 텍스트가 반 픽셀 좌표에 걸리지 않는다 |
| 접근성 | 주요 버튼과 행은 keyboard focus가 가능하다 |
