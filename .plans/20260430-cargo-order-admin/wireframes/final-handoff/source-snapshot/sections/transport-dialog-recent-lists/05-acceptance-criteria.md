# Acceptance Criteria

## 주소 최근 리스트

| ID | 기준 | 검증 방법 |
| --- | --- | --- |
| `AC-ADDR-001` | `openAddrSearch('load')` 성격의 다이얼로그에는 `상차 최근`을 기본 최근 리스트로 표시한다. | 상차지 샘플 HTML에서 최근 리스트 있음 상태 확인 |
| `AC-ADDR-002` | `openAddrSearch('unload')` 성격의 다이얼로그에는 `하차 최근`을 기본 최근 리스트로 표시한다. | 문서 정책과 HTML 탭 문구 확인 |
| `AC-ADDR-003` | 최근 장소 row에는 장소명, 행정주소, 사용 맥락을 표시하고 연락처는 기본 row에서 숨긴다. | HTML row와 문서 개인정보 기준 확인 |
| `AC-ADDR-004` | 최근 장소 클릭은 `선택 미리보기`만 갱신하고, main row 저장은 실행하지 않는다. | HTML의 `적용 전` 표시와 문서 우선순위 확인 |
| `AC-ADDR-005` | 최근 리스트는 별도 3열 패널이 아니라 기존 검색 결과 영역의 초기 상태로 표시된다. | HTML 구조와 문서 정책 확인 |
| `AC-ADDR-006` | `조회` 또는 Enter 검색 실행 후 같은 영역이 기존 검색 결과 리스트로 전환된다. | HTML 안내 문구와 상태 정책 확인 |
| `AC-ADDR-007` | 선택 미리보기 내부 UI 톤은 유지하되 위치와 전체 레이아웃은 기존 주소 검색 다이얼로그 구조를 우선한다. | master 구조 비교와 HTML 확인 |
| `AC-ADDR-008` | 최근 리스트 empty/loading/error 상태에서도 검색 입력과 검색 결과 선택 흐름은 차단하지 않는다. | empty 상태 샘플과 상태 정책 확인 |

## 운송+품목 최근 리스트

| ID | 기준 | 검증 방법 |
| --- | --- | --- |
| `AC-CARGO-001` | 최근 조합은 `톤수 + 차종 + 품목 + 중량` 조합 단위로 표시한다. | 문서와 HTML 조합 row 확인 |
| `AC-CARGO-002` | 최근 조합 클릭은 입력폼 값을 채우지만, `운송+품목 적용` 전에는 row에 반영하지 않는다. | HTML 선택 후 미리보기 상태 확인 |
| `AC-CARGO-003` | `대수`, `실중량`은 기존 target field 기준에 맞춰 입력폼에 남긴다. | HTML 입력폼 확인 |
| `AC-CARGO-004` | 부분 선택은 MVP 기본 기능이 아니며, 전체 조합 선택을 우선한다. | `03-cargo-item-dialog-recent-list.md` 확인 |
| `AC-CARGO-005` | 운송+품목 최근 항목에는 금액 조건을 포함하지 않는다. | HTML row와 문서 데이터 정책 확인 |
| `AC-CARGO-006` | 별도 선택 요약 패널 없이 최근 조합 선택값은 입력폼에서만 확인한다. | HTML과 문서 확인 |
| `AC-CARGO-007` | 최근 조합 empty/loading/error 상태에서도 직접 입력과 적용 흐름은 차단하지 않는다. | empty 상태 샘플과 상태 정책 확인 |

## 기존 flow 회귀 방지

| ID | 기준 | 검증 방법 |
| --- | --- | --- |
| `AC-REG-001` | 기존 `applyAddr()` 확정 책임은 유지한다. | 문서에서 최근 클릭과 적용 클릭을 분리했는지 확인 |
| `AC-REG-002` | 기존 `applyCargo()` 확정 책임은 유지한다. | 문서에서 `운송+품목 적용` 전 row 미반영 기준 확인 |
| `AC-REG-003` | 신규 접수 wizard에서는 왼쪽 프로세스 패널 정책을 유지한다. | `04-state-and-interaction-matrix.md` 확인 |
| `AC-REG-004` | `new-submitted` 이후 개별 수정에서는 독립 다이얼로그를 사용한다. | `01-overview.md`와 상태 매트릭스 확인 |
| `AC-REG-005` | 최근 리스트는 검색 결과 선택 흐름을 대체하지 않고 검색 전 초기 상태로만 쓰인다. | 주소 다이얼로그 문서와 HTML 검색 영역 확인 |
| `AC-REG-006` | `master.html`에는 static HiFi prototype 범위로만 반영하고 실제 API/DB 구현은 제외한다. | 파일 변경 목록과 문서 범위 확인 |

## HTML 산출물 기준

| ID | 기준 | 검증 방법 |
| --- | --- | --- |
| `AC-HTML-001` | standalone HTML은 전체 화면이 아니라 다이얼로그 2개와 상태 샘플만 포함한다. | HTML 구조 확인 |
| `AC-HTML-002` | 상하차지 최근 리스트 있음 상태가 확인된다. | HTML 내 `상하차지 / 최근 있음` 샘플 확인 |
| `AC-HTML-003` | 상하차지 최근 리스트 없음 상태가 확인된다. | HTML 내 `상하차지 / 최근 없음` 샘플 확인 |
| `AC-HTML-004` | 운송+품목 최근 리스트 있음 상태가 확인된다. | HTML 내 `운송+품목 / 최근 있음` 샘플 확인 |
| `AC-HTML-005` | 운송+품목 최근 리스트 없음 상태가 확인된다. | HTML 내 `운송+품목 / 최근 없음` 샘플 확인 |
| `AC-HTML-006` | 최근 항목 선택 후 미리보기 반영 상태가 확인된다. | HTML 내 selected row와 preview panel 확인 |
| `AC-HTML-007` | `btn`, `chip`, `badge`, `empty`, `summary-row`, `dialog` 계열 패턴을 우선 재사용한다. | CSS class와 화면 구조 확인 |
| `AC-HTML-008` | 운송+품목 최근 있음 샘플에는 `선택 요약` 패널이 없다. | HTML 문구와 레이아웃 확인 |

## 용어 기준

| 용어 | 사용 기준 |
| --- | --- |
| `운송 구간` | 상차/하차 주소와 조건 섹션 |
| `화물 정보` | 기존 통합 HiFi의 사용자-facing 섹션 표현 |
| `운송+품목` | 톤수, 차종, 대수, 실중량, 품목 입력 묶음 |
| `최근 사용` | 최근 선택순으로 정렬된 보조 후보 |
| `선택 미리보기` | 적용 전 확인 영역 |
