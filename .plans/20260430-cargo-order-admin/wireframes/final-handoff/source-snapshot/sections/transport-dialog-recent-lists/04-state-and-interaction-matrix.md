# 상태와 상호작용 매트릭스

## 상태 기준

| 상태 | 최근 리스트 표시 | 다이얼로그 방식 | 주요 정책 |
| --- | --- | --- | --- |
| `idle-edit` | 표시 | 독립 다이얼로그 | 기존 수정 버튼에서 열고, 최근 선택 후 `적용`으로 확정 |
| `new-reset` | 표시 전 단계 | 독립 다이얼로그 사용 안 함 | 신규 접수 초기화와 화주 focus가 우선 |
| `new-wizard-active` | 현재 wizard 단계에 표시 | wizard frame 내부 | 왼쪽 프로세스 패널 유지, 현재 단계 후보만 표시 |
| `new-required-complete` | 필요 시 현재 단계에 표시 | wizard frame 내부 | 금액 완료 후 분기 정책 유지 |
| `new-driver-optional` | 대상 아님 | wizard frame 내부 | 운송 관련 최근 리스트는 표시하지 않음 |
| `new-submitted` | 표시 | 독립 다이얼로그 | 메인 적용 후 개별 수정은 `idle-edit`과 동일 |

## 신규 접수 wizard 상태

| wizard 단계 | 최근 리스트 | 기본 탭 또는 scope | CTA |
| --- | --- | --- | --- |
| 상차지 | 최근 장소 | `상차 최근` + 현재 화주 | `다음: 하차지` |
| 하차지 | 최근 장소 | `하차 최근` + 현재 화주 | `다음: 운송+품목` |
| 운송+품목 | 최근 조합 | 현재 화주 조합 최근 | `다음: 금액` |

wizard에서는 최근 리스트가 단계 이동을 바꾸지 않습니다. 최근 선택은 현재 단계의 입력값 또는 미리보기를 채우는 동작이고, 다음 단계 이동은 기존 CTA와 validation을 통과해야 합니다.

## 일반 수정 다이얼로그 상태

| 흐름 | 최근 리스트 | 확정 |
| --- | --- | --- |
| 상차지 수정 | `상차 최근` 기본 | `상차지 적용` |
| 하차지 수정 | `하차 최근` 기본 | `하차지 적용` |
| 운송+품목 수정 | 최근 조합 | `운송+품목 적용` |

일반 수정 다이얼로그에서는 왼쪽 프로세스 패널을 표시하지 않습니다. `new-submitted` 이후 개별 수정도 같은 원칙입니다.

## 최근 리스트 있음/없음

| 상태 | 주소 다이얼로그 | 운송+품목 다이얼로그 |
| --- | --- | --- |
| 있음 | 최근 장소 row 1~10개 표시, 기본 selected 없음 또는 첫 항목 preview 가능 | 최근 조합 row 1~10개 표시, 클릭 시 입력폼 채움 |
| 없음 | `최근 사용 장소가 없습니다.` empty 표시, 검색 사용 가능 | `최근 사용 조합이 없습니다.` empty 표시, 직접 입력 가능 |
| loading | 최근 영역만 loading | 최근 영역만 loading |
| error | 최근 영역만 error, 검색 차단 없음 | 최근 영역만 error, 입력 차단 없음 |

## 최근 항목 선택 후 상태

| 항목 | 주소 최근 선택 | 운송+품목 최근 선택 |
| --- | --- | --- |
| selected 표시 | 최근 row selected | 최근 row selected |
| 반영 위치 | `선택 미리보기` | 입력폼 |
| 저장 여부 | 저장 안 됨 | 저장 안 됨 |
| 확정 조건 | `상차지 적용`/`하차지 적용` 클릭 | `운송+품목 적용` 클릭 |
| 취소 시 | 기존 row 유지 | 기존 row 유지 |

## 검색 결과 선택과 최근 항목 선택 우선순위

마지막 명시 선택이 우선입니다.

| 시나리오 | 적용 대상 |
| --- | --- |
| 최근 항목 선택 -> 적용 | 최근 항목 |
| 최근 항목 선택 -> 검색 결과 선택 -> 적용 | 검색 결과 |
| 검색 결과 선택 -> 최근 항목 선택 -> 적용 | 최근 항목 |
| 최근 항목 선택 -> 직접 입력 수정 -> 적용 | 수정된 입력값 |
| 검색어 변경만 하고 결과 미선택 -> 적용 | 기존 미리보기 또는 입력값 |

## 충돌 방지 기준

| 충돌 가능성 | 방지 기준 |
| --- | --- |
| 최근 선택이 바로 저장으로 오해됨 | row 선택 후 `적용 전` chip 또는 helper text 표시 |
| wizard 단계와 독립 다이얼로그 혼동 | `new-wizard-active`에서는 프로세스 패널 유지, `idle-edit`/`new-submitted`에서는 숨김 |
| 검색 결과 흐름 대체 | 최근 리스트는 기존 검색 결과 영역의 초기 상태로 표시하고, 검색 실행 후 같은 영역을 기존 검색 결과로 전환 |
| 개인정보 과노출 | 최근 row에는 연락처를 숨기고, 미리보기에서 필요한 경우에만 표시 |
| 금액 조건 혼입 | 운송+품목 최근 조합에는 금액 필드를 포함하지 않음 |
| 불필요한 요약 중복 | 운송+품목 다이얼로그에는 별도 선택 요약 패널을 두지 않고 입력폼에서 현재값을 확인 |

## 저장 기록 시점

| 대상 | 기록 시점 | 이유 |
| --- | --- | --- |
| 주소 최근 | `상차지 적용` 또는 `하차지 적용` 성공 후 | 클릭만으로는 사용 확정이 아니기 때문 |
| 운송+품목 최근 | `운송+품목 적용` 성공 후 | 사용자가 직접 수정한 최종값을 기록해야 함 |
