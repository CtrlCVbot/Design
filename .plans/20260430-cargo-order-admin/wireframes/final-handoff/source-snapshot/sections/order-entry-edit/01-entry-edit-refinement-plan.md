# 접수/수정 섹션 보강 계획

## 목표

신규 HiFi를 `시각 고도화 기준본`으로 삼되, 실제 업무 흐름과 동작 계약은 B Original Tone과 Clean을 유지한다.

이번 계획은 상단 입력/수정 업무면을 하나의 `접수/수정 섹션`으로 보고, 5개 입력 섹션과 전역 액션바가 어떻게 연결되는지 정리한다.

신규 접수 플로우의 최종 기준은 `../new-order-registration-flow`가 담당한다. 이 문서는 기존 오더 수정, 전역 액션, 저장 실패, 위험 액션과의 연결을 보조한다.

## 범위

| 포함 | 제외 |
| --- | --- |
| 신규 접수, 등록, 수정, 복사, 조회 흐름 | 하단 화물 목록 상세 기획 |
| 화물 취소, 배차 취소, 정산 처리 위험 흐름 | 실제 API 계약 |
| 5개 입력 섹션의 저장 전 상태 연결 | 서버 validation 메시지 전문 |
| 전역 액션바 그룹화 | cargo-list 테이블 컬럼 설계 |
| HiFi 시각 톤 적용 기준 | Clean hook 제거 또는 대체 |

## 상태 모델 초안

| 상태 | 설명 | 주요 UI |
| --- | --- | --- |
| `new-reset` | 신규 접수 시작, 전체 초기화, 오더 ID 미발번 | 신규 접수 안내형 헤더, 화주 정보 focus |
| `new-wizard-active` | 신규 접수 wizard 진행 중 | 왼쪽 프로세스 패널, 단계별 다이얼로그 |
| `new-required-complete` | 금액까지 필수 입력 완료 | `화물 등록 완료` 또는 `차주 정보로 이동` 선택 |
| `new-driver-optional` | 차주 정보 선택 입력 | 차주 선택 또는 건너뛰기 |
| `new-submitted` | wizard 데이터가 API 없이 메인 화면에 적용된 상태 | 메인 `화물 등록` primary, 섹션 헤더 숨김 |
| `submit-complete` | 실제 API 등록 완료, 오더 ID 발번 | 접수 상태 badge, 수정/복사/취소 가능 |
| `edit-loaded` | 목록 또는 조회로 기존 오더 로드 | `화물 수정` primary, 변경 전/후 구분 |
| `edit-dirty` | 기존 오더 수정 중 | 변경 항목 요약, 저장/되돌리기 |
| `save-error` | 저장 실패 또는 validation 실패 | 필드별 오류 + 상단 요약 |
| `cancel-confirm` | 화물 취소 또는 배차 취소 확인 중 | 영향 범위, 사유, NO/YES |
| `settlement-locked` | 정산 처리 후 금액 수정 제한 | 금액 read-only, 정산 상태 badge |

## 저장 전 검증 묶음

| 검증 묶음 | 연결 섹션 | 검증 방향 |
| --- | --- | --- |
| 화주 적용 여부 | `shipper-info` | 업체명, 사업자번호, 담당자, 연락처, 이메일 |
| 운송구간 적용 여부 | `transport-route` | 상차/하차 주소, 상세주소, 일시/방법 |
| 운송+품목 | `cargo-transport` | 톤수, 차종, 대수, 실중량, 품목 |
| 금액 조건 | `cargo-transport` | 결제방법, 청구비용, 운송비용, 수수료, 수익/차주운임 |
| 요약 상태 | `cargo-summary-docs` | 자동/수정됨/stale 여부 |
| 차주 상태 | `driver-info` | 내부 DB 또는 화물맨 적용 상태 |

## HiFi 이식 원칙

| 원칙 | 내용 |
| --- | --- |
| 시각은 HiFi 기준 | 색, 타입, 버튼 variant, 섹션 헤더, 액션 그룹은 HiFi를 따른다. |
| 구조는 Clean 기준 | 기존 `data-*` hook과 dialog 접근성 구조는 제거하지 않는다. |
| 필드는 섹션 문서 기준 | 보이는 값과 입력 가능 범위는 각 섹션 README와 mapping을 따른다. |
| 목록은 placeholder | `cargo-list`는 지금 상세 설계하지 않고 자리만 둔다. |
| 위험 액션은 confirm | 취소/배차취소/정산 처리에는 영향 요약과 명시 확인을 둔다. |
