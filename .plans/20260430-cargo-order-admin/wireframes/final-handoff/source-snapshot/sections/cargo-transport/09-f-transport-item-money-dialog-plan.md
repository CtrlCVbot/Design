# F안: 운송+품목 / 금액 분리형 다이얼로그 기획

## 목적

F안은 기존 `운송`, `금액`, `품목` 3개 row 구조를 업무 성격 기준으로 다시 묶는 안입니다.

- `운송+품목`: 실제 화물 운송에 필요한 물리 정보와 품목 요약을 함께 확인합니다.
- `금액`: 청구, 운송비, 수수료, 결과 금액처럼 정산에 필요한 값을 별도 영역으로 분리합니다.

기존 D안의 장점인 compact row, inline edit, 손글씨 wireframe 톤은 유지합니다. 다만 최초 입력 전 상태는 다른 섹션과 맞춰 `버튼 슬라이딩 -> 다이얼로그 오픈 -> 적용 후 row 반영` 흐름으로 정리합니다.

## 정보 구조

```text
[운송+품목]
운송   톤수   차종   대수   실중량(화물중량 110% 이하 안내 포함)
품목   품목명 또는 품목 입력값

[금액]
금액   결제방법   청구비용   운송비용   수수료   수익/차주운임
```

## 운송+품목 섹션

| 항목 | 표시 방식 | 수정 방식 | 규칙 |
| --- | --- | --- | --- |
| `톤수` | 라벨 | select | 선택 시 `실중량`을 톤수의 110%로 자동 계산 |
| `차종` | 라벨 | select | 차량 형태 선택 |
| `대수` | 라벨 | number input | 기본 1대 |
| `실중량` | 라벨 + 보조 안내 | decimal input | `0.00` 단위, 자동 계산 후 직접 수정 가능. `화물중량 110% 이하` 안내는 별도 칩이 아니라 `실중량` 셀 내부에 표시 |
| `품목` | 라벨 | text input | 같은 박스의 2번째 row에 표시 |

품목은 별도 카드가 아니라 `운송+품목` 박스 내부 2행에 둡니다. 긴 품목명은 row 높이를 늘리지 않도록 한 줄 표시와 말줄임을 기본으로 하고, hover/focus 시 전체 값을 확인할 수 있게 합니다.

컴포넌트 내부에는 실제 오더 데이터만 표시합니다. `같은 박스 2행`, `요약 전달`처럼 설계 의도를 설명하는 문구는 row 안에 배지로 넣지 않고, 기획 문서나 섹션 외부 설명에서만 다룹니다.

운송 row에서는 고정 `수정 가능` 칩을 두지 않습니다. 운송구간 1열 정리안과 동일하게 각 수정 가능한 셀에 hover/focus가 들어왔을 때만 작은 `수정` 배지를 노출합니다. 이렇게 하면 실제 데이터 폭을 유지하면서도 어떤 값이 수정 가능한지 즉시 알 수 있습니다.

## 금액 섹션

| 항목 | 표시 방식 | 수정 방식 | 규칙 |
| --- | --- | --- | --- |
| `결제방법` | 라벨 | select 또는 segmented control | `인수증`, `선불`, `착불`, `선착불` |
| `청구비용` | 금액 라벨 | money input | 입력 시 `운송비용`에 같은 금액 자동 입력 |
| `운송비용` | 금액 라벨 | money input | 직접 수정해도 `청구비용`은 유지 |
| `수수료` | 조건부 금액 라벨 | money input | 입력 시 자동 `선착불`, `청구비용=0원`, `운송비용` 포커스 |
| `수익/차주운임` | 계산 라벨 | readonly | 인수증은 `수익`, 그 외는 `차주운임` |

## 입력 전 상태

입력 전에는 두 개의 CTA를 분리합니다.

```text
[운송+품목 입력]  -> 운송+품목 라벨 방향으로 슬라이딩 -> 운송+품목 입력 다이얼로그 오픈
[금액 조건 선택]  -> 금액 라벨 방향으로 슬라이딩 -> 금액 조건 입력 다이얼로그 오픈
```

다이얼로그에서 `적용`하면 해당 row가 표시됩니다. `취소`하면 입력 전 상태를 유지합니다.

## 입력 후 수정 방식

최초 입력은 다이얼로그로 처리합니다.

값이 반영된 이후에는 기존 D안의 inline edit 기능을 유지합니다. 각 값은 독립적으로 클릭 가능해야 하며, 사용자가 특정 값을 클릭하면 해당 값만 같은 자리에서 select/input/text input으로 전환되어 수정됩니다.

수정 가능 여부는 별도 row 안내가 아니라 셀 상호작용으로 표현합니다. 기본 상태에서는 라벨과 값만 보이고, 마우스를 올리거나 키보드 포커스가 들어오면 셀 배경과 dashed outline이 표시되며 오른쪽 위에 `수정` 배지가 나타납니다.

| 필드 | 입력 후 수정 컴포넌트 |
| --- | --- |
| `톤수` | select/dropdown |
| `차종` | select/dropdown |
| `대수` | number input |
| `실중량` | decimal input |
| `품목` | text input |
| `결제방법` | select 또는 segmented control |
| `청구비용` | money input |
| `운송비용` | money input |
| `수수료` | money input |
| `수익/차주운임` | readonly |

수정 중 Enter 또는 blur 시 반영하고, Esc 입력 시 수정 전 값으로 돌아갑니다.

## 다이얼로그 구성

### 운송+품목 입력 다이얼로그

- 상단: 섹션 목적과 적용 후 row 안내
- 본문: `톤수`, `차종`, `대수`, `실중량`, `품목`
- 하단: `취소`, `운송+품목 적용`

### 금액 조건 입력 다이얼로그

- 상단: 정산 구조와 결제방법 안내
- 본문: `결제방법`, `청구비용`, `운송비용`, `수수료`, 계산 미리보기
- 하단: `취소`, `금액 조건 적용`

## 장단점

| 항목 | 장점 | 주의점 |
| --- | --- | --- |
| `운송+품목` 묶음 | 물리 운송정보와 품목 요약을 함께 읽을 수 있음 | 품목이 길면 말줄임 처리가 필요 |
| `금액` 분리 | 정산 흐름과 계산 결과가 별도 영역으로 명확해짐 | 운송 조건과 금액 조건을 한 번에 입력하려는 사용자는 클릭이 2회 필요 |
| 다이얼로그 최초 입력 | 값 없음 상태에서 입력 범위가 명확함 | 입력 후에는 inline edit를 유지해 재수정 부담을 줄여야 함 |

## B 통합본 적용 시 고려사항

1. B 통합본에서는 현재 `3. 화물 운송정보` 섹션 안에 F안 구조를 적용했습니다.
2. 기존 `운송`, `금액`, `품목` 3개 row를 `운송+품목`, `금액` 2개 그룹으로 줄였습니다.
3. 입력 전 상태는 `운송+품목 입력`, `금액 조건 선택` CTA를 유지하되 보조 chip은 muted dashed 스타일로 낮춥니다.
4. 적용 후 상태는 `운송` row, `품목` row, `금액` row가 모두 같은 폭 리듬으로 정렬됩니다.
5. `실중량`의 `110% 이하` 안내는 별도 chip이 아니라 `실중량` 셀 내부에 붙입니다.
6. `품목` row 내부의 중복 라벨은 숨기고 실제 품목값만 노출합니다.
7. 금액 row에서는 `수익/차주운임` 셀만 강조하고, `청구-운송` 공식은 보조 정보로 낮춥니다.
8. 운송구간에서 사용하는 거리/기준금액 메타는 `톤수`, `차종` 선택 이후 정확해진다는 의존성을 유지합니다.
