# Final Adoption Criteria: 화물 운송정보 F안 -> B 통합본

## 목적

이 문서는 `cargo-transport-section-f-wireframe.html`의 F안과 `Cargo Order Wireframe B Original Tone.html`의 B 통합본 차이를 정리하고, 실제 후속 구현에서 어떤 기준을 최종 채택할지 고정합니다.

결론부터 말하면, **B 통합본의 `3. 화물 운송정보` 섹션을 최종 배치 기준으로 채택**합니다. F안 단독 HTML은 구조 검토와 기능 후보 확인용 reference로 유지합니다.

## 비교 대상

| 구분 | 파일 | 역할 |
| --- | --- | --- |
| F안 단독 HTML | `sections/cargo-transport/cargo-transport-section-f-wireframe.html` | `운송+품목`, `금액` 분리 구조와 다이얼로그/inline edit를 독립적으로 검토하는 비교안 |
| B 통합본 | `results/wireframe/order-register-new2.0/Cargo Order Wireframe B Original Tone.html` | 실제 오더 접수/수정 화면 안에 들어간 최종 배치 기준 |

| 다이얼로그 안정성 | 기본 close 동작 | `open` 제거, `display:none`, animation 중복 실행 방지 | B 통합본 |

## 최종 채택 기준

### 1. 정보 구조

최종 구조는 F안의 핵심 정보 구조를 유지합니다.

```text
[운송+품목]
운송  톤수  차종  대수  실중량
품목  장비 운송

[금액]
금액  결제방법  청구비용  운송비용  수수료  수익/차주운임  계산식
```

`광폭`, `도착`은 최종 구조에 포함하지 않습니다. `광폭`은 차량 세부 옵션 후보이고, `도착`은 운송구간 상하차 일정/조건과 중복됩니다.

### 2. 입력 전 상태

입력 전 상태는 B 통합본 기준을 채택합니다.

| row | 최종 표시 |
| --- | --- |
| `운송+품목` | `[운송+품목] placeholder [운송+품목 입력] [톤수/차종/품목]` |
| `금액` | `[금액] placeholder [금액 조건 선택] [인수증 기본]` |

채택 기준:
- CTA 버튼이 가장 먼저 보이게 둡니다.
- 보조 chip은 dashed muted 스타일로 낮춥니다.
- 입력 전 row는 29px 기준으로 맞춥니다.
- placeholder는 현재 B 통합본의 종이/명세표 감각을 유지합니다.

### 3. 적용 후 상태

적용 후 상태는 B 통합본의 compact row를 최종 채택합니다.

| row | 최종 표시 | 높이 기준 |
| --- | --- | --- |
| `운송` | `톤수`, `차종`, `대수`, `실중량`, `110% 이하` | 28px |
| `품목` | 실제 품목값만 표시 | 29px |
| `금액` | `결제방법`, `청구비용`, `운송비용`, `수수료`, `수익/차주운임`, 계산식 | 28px |

채택 기준:
- `품목` row 내부의 중복 `품목` 라벨은 숨깁니다.
- `실중량`의 `110% 이하` 안내는 셀 내부 보조 안내로 둡니다.
- `수익/차주운임` 셀은 결과값이므로 강조합니다.
- `청구-운송` 또는 `운송+수수료` 공식은 보조 정보로 약하게 표시합니다.

### 4. 다이얼로그와 전환

| 항목 | 최종 기준 |
| --- | --- |
| CTA 이동 | 대상 라벨 방향으로 340ms 이동 |
| fallback | animation event 누락 시 560ms 안에 open |
| 중복 실행 방지 | animationend와 fallback timer 중 먼저 실행된 1회만 처리 |
| 적용 후 닫힘 | `<dialog open>` 제거 및 `display:none` 상태 확인 |
| row reveal | 위치 이동 없이 opacity/background 중심으로 140ms |

F안 단독 HTML의 기본 close 동작은 비교용으로 유지할 수 있지만, 실제 구현 또는 B 통합본 반영 시에는 B 통합본의 안정화 규칙을 따릅니다.

### 5. 수정 방식

적용 후 수정은 F안과 B 통합본 모두 같은 방향을 채택합니다.

| 필드 | 수정 방식 |
| --- | --- |
| `톤수` | select |
| `차종` | select |
| `대수` | number input |
| `실중량` | decimal input |
| `품목` | text input |
| `결제방법` | select 또는 segmented control |
| `청구비용` | money input |
| `운송비용` | money input |
| `수수료` | money input |
| `수익/차주운임` | readonly |

기본 상태에서는 라벨과 값만 보입니다. hover/focus 시에만 `수정` 배지를 노출합니다.

## F안 단독 HTML 유지 기준

F안 단독 HTML은 삭제하지 않습니다.

유지 이유:
- `운송+품목`, `금액` 분리 구조를 독립적으로 설명하기 좋습니다.
- 입력 전/적용 후/다이얼로그/inline edit 기능을 한 화면에서 비교하기 쉽습니다.
- B 통합본 수정 전 기능 후보를 검토하는 reference 역할을 합니다.

단, F안 단독 HTML의 시각 밀도와 설명 chip은 최종 제품 기준이 아닙니다. 최종 제품 기준은 B 통합본을 따릅니다.

## 구현 인계 기준

| 구현 항목 | 인계 기준 |
| --- | --- |
| 실제 화면 기준 | B 통합본 `3. 화물 운송정보` |
| 구조 검토 reference | F안 단독 HTML |
| CTA animation | B 통합본의 340ms/560ms/1회 실행 가드 |
| dialog close | B 통합본의 `open=false`, `display=none` 보강 |
| row density | B 통합본의 28~29px row rhythm |
| chip 강도 | 입력 전 보조 chip은 muted dashed |
| 결과 강조 | `수익/차주운임`만 강조, 계산식은 보조 |
| 요약 연결 | `품목`, `실중량`, `톤수`, `차종`, `대수`는 `4. 화물정보 요약`의 원본 데이터 |
| 요약 갱신 샘플 | B 통합본에서 `auto`, `overridden`, `stale` 상태 기준으로 동작 확인 |

## 후속 작업

1. validation message와 `자동/수정됨` chip 필요 여부를 결정합니다.
2. 차종 축약어와 일정 토큰의 운영 표기를 확정합니다.
3. 실제 개발 단계에서는 B 통합본 기준으로 컴포넌트를 구현하고, F안 단독 HTML은 참고 자료로만 사용합니다.
