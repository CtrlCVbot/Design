# Field/State Mapping: 화물 운송정보

## 필드 그룹

| 그룹 | 필드 | 역할 |
| --- | --- | --- |
| 운송 조건 | `톤수`, `차종`, `대수`, `실중량` | 차량 조건과 실제 화물 중량을 결정하는 정보 |
| 금액 조건 | `결제방법`, `청구비용`, `운송비용`, `수수료`, `수익/차주운임` | 화주 청구와 차주 정산을 구분하는 정보 |
| 품목 row | `품목` | `운송+품목` 박스의 2번째 row로 표시되는 화물 설명 |
| 상태 조건 | `결제방법` | `인수증`, `선불`, `착불`, `선착불`에 따라 금액 계산과 수수료 표시 제어 |
| 파생 정보 | `중량`, 운송구간 `거리/기준 금액` | 다른 섹션에서 표시되지만 이 섹션 값에 의존 |

| `차종` | `축카고` | select | 운송 조건 | 운송구간 기준 금액 |

| `결제방법` | `인수증` 또는 `선착불` | select/segmented | 금액 조건 | `수수료`, `수익/차주운임` 계산식 |

| `수익/차주운임` | `40,000` 또는 `120,000` | calculated/display | 금액 조건 | 인수증은 `수익`, 선착불은 `차주운임` 라벨 |

## 제외 또는 이동 필드

| 필드 | 처리 | 이유 |
| --- | --- | --- |
| `광폭` | 기준 필드에서 제외 | 이번 섹션의 핵심 차량/화물 조건에서 제외. 필요 시 차량 세부 옵션에서 별도 검토 |
| `도착` | 기준 필드에서 제외 | 운송구간의 상하차 일정/조건과 중복 |
| 기존 `합계` | `수익/차주운임`으로 명칭 변경 | 인수증은 `수익`, 선착불은 `차주운임`으로 의미를 분리하기 위함 |

## 상태 mapping

| 상태 | 조건 | 화면 표시 |
| --- | --- | --- |
| `empty` | 값이 아직 없음 | placeholder 또는 `선택` 상태 |
| `activation-ready` | 값 없음 상태에서 row 활성화 전 | `운송+품목 입력`, `금액 조건 선택` CTA 표시 |
| `activating` | 사용자가 입력 CTA를 클릭함 | CTA가 대상 라벨 방향으로 이동 |
| `activation-finished` | CTA animation 또는 fallback 중 하나가 완료됨 | 다이얼로그 open은 1회만 실행 |
| `dialog-open` | CTA가 사라지고 입력 다이얼로그가 열림 | 입력값 선택 또는 취소 |
| `dialog-closed` | 적용 또는 취소로 다이얼로그가 닫힘 | `<dialog open>` 제거 및 display hidden |
| `activated-empty` | 다이얼로그 적용 완료 | `운송`/`품목` 또는 `금액` row 항상 노출 |
| `selected` | 사용자가 값을 선택함 | 현재 값 표시 |
| `changed` | 기존 오더에서 값이 변경됨 | 변경 필드 강조 |
| `auto-filled` | `청구비용` 입력으로 `운송비용`이 자동 입력됨 | 자동 입력 상태 표시 가능 |
| `weight-auto-filled` | `톤수` 선택으로 `실중량`이 톤수의 110%로 자동 입력됨 | `0.00` 단위 입력값 표시 |
| `weight-overridden` | 사용자가 `실중량`을 직접 수정함 | 자동값 대신 수동 입력값 사용 |
| `manual-overridden` | 사용자가 `운송비용`을 직접 수정함 | 청구비용은 유지, 운송비용만 변경 |
| `conditional-visible` | `선불/착불/선착불` 조건 | `수수료` 표시 |
| `conditional-hidden` | `인수증` 조건 | `수수료` 숨김 또는 비활성 |
| `locked` | 선착불 전환 후 `청구비용=0원` | 직접 수정 제한 |
| `calculated` | `차주운임` 계산 완료 | 계산값 표시 |

## 필드별 상태 규칙

| 필드 | 값 없음 | 값 있음 | 변경 시 |
| --- | --- | --- | --- |
| `톤수` | `톤수 선택` | `5톤` | 실중량 110% 자동 입력, 운송구간 기준 금액 재계산 |
| `차종` | `차종 선택` | `축카고` | 운송구간 기준 금액 재계산 |
| `대수` | `대수 입력` | `1` | 금액 조건 확인 필요 가능 |
| `실중량` | `0.00` | `5.50` | 사용자가 직접 수정 가능 |
| `결제방법` | `결제방법 선택` | `인수증`, `선불`, `착불`, `선착불` | 수수료 표시와 결과 라벨/계산식 변경 |
| `청구비용` | `청구비용 입력` | `140,000` | `운송비용`에 같은 금액 자동 입력 |
| `운송비용` | 자동 입력 대기 | `100,000` | 청구비용은 유지, 차주운임 재계산 |
| `수수료` | 인수증에서는 숨김/비활성 | `20,000` | 자동 선착불 전환, 청구비용 0원 고정, 운송비용 포커스 |
| `수익/차주운임` | 계산 대기 | `40,000` 또는 `120,000` | 인수증은 `수익`, 선착불은 `차주운임`으로 표시 |
| `품목` | `품목 입력` | `장비 운송` | `운송+품목` 박스의 2번째 row에서 `4. 화물정보 요약` 갱신 |

## 결제방법별 계산

| 결제방법 | 수수료 표시 | 청구비용 | 결과 라벨/계산 | 의미 |
| --- | --- | --- | --- | --- |
| `인수증` | 숨김/비활성 | 사용자가 입력 | `수익 = 청구비용 - 운송비용` | 운영 수익 |
| `선불` | 표시 | 조건에 따라 `0원` 가능 | `운송비용 + 수수료` | 차주에게 청구되는 운임 |
| `착불` | 표시 | 조건에 따라 `0원` 가능 | `운송비용 + 수수료` | 차주에게 청구되는 운임 |
| `선착불` | 표시 | `0원` 고정 | `운송비용 + 수수료` | 차주에게 청구되는 운임 |

## 입력 이벤트 규칙

| 이벤트 | 시스템 처리 |
| --- | --- |
| `톤수` 선택 | `실중량`을 선택 톤수의 110%로 자동 입력하고 `0.00` 단위로 표시 |
| `실중량` 직접 수정 | 사용자가 입력한 값을 유지하고 이후 자동값과 구분 |
| `결제방법` 선택 | `인수증`, `선불`, `착불`, `선착불`에 따라 수수료 표시와 결과 라벨/계산식을 변경 |
| `청구비용` 입력 | `운송비용`을 같은 금액으로 자동 입력 |
| `운송비용` 직접 수정 | `청구비용`은 변경하지 않음 |
| `수수료` 입력 | `결제방법`을 자동으로 `선착불`로 변경 |
| `수수료` 입력 후 | `청구비용=0원` 고정, `운송비용`으로 포커스 이동 |
| `결제방법` 변경 | 필드 위치는 유지하고 수수료 표시/숨김/비활성만 변경 |
| `운송+품목 입력` 클릭 | CTA를 `운송+품목` 라벨 방향으로 이동시킨 뒤 다이얼로그를 열고, 적용 후 `운송`/`품목` row로 전환 |
| `금액 조건 선택` 클릭 | CTA를 `금액` 라벨 방향으로 이동시킨 뒤 다이얼로그를 열고, 적용 후 금액 row로 전환 |
| CTA animation 완료 | animationend와 fallback timer 중 먼저 도착한 1회만 다이얼로그를 열고 나머지는 무시 |
| 다이얼로그 적용 | row를 `activated` 상태로 바꾸고 dialog는 `open=false`, `display=none` 상태로 닫음 |

## 값 없음 활성화 전환

값 없음 상태는 `08-empty-state-activation-transition.md`를 기준으로 다음 상태를 거칩니다.

| 상태 | 표시 | 다음 상태 |
| --- | --- | --- |
| `activation-ready` | `[운송+품목] placeholder [운송+품목 입력] [톤수/차종/품목]` | `activating` |
| `activating` | 입력 버튼이 대상 라벨로 이동 | `activation-finished` |
| `activation-finished` | CTA 완료 이벤트 1회 처리 | `dialog-open` |
| `dialog-open` | 입력 다이얼로그 표시 | `activated-empty` 또는 `activation-ready` |
| `dialog-closed` | 적용/취소 후 다이얼로그 숨김 | `activated-empty` 또는 `activation-ready` |
| `activated-empty` | `운송` row와 `품목` row 또는 `금액` row 표시 | `selected`, `weight-auto-filled`, `calculated` |

## 운송구간과의 연결

운송구간의 보조 메타 중 거리와 기준 금액은 다음 값에 의존합니다.

| 운송구간 메타 | 필요한 값 | 표시 원칙 |
| --- | --- | --- |
| `거리` | 상차지 주소, 하차지 주소 | 주소 확정 후 계산 가능 |
| `기준 금액` | 상차지/하차지 주소, `톤수`, `차종` | `톤수`, `차종` 선택 전에는 확정처럼 보이지 않게 표시 |
| `톤수·차종 변경 시 재계산` | `톤수`, `차종` 변경 이벤트 | 화물 운송정보와 운송구간 사이의 연결 상태로 표시 |

## 다음 섹션과의 연결

| 현재 필드 | 연결 섹션 | 연결 방식 |
| --- | --- | --- |
| `톤수` | `실중량`, `4. 화물정보 요약` | 실중량 `톤수 110%` 자동 입력. 실중량이 비어 있으면 요약 문장의 중량 대체값 |
| `실중량` | `4. 화물정보 요약` | 화물중량 110% 이하 안내와 실제 중량 표시. 요약에서는 `5.5톤`처럼 문장화 |
| `결제방법`, `청구비용`, `운송비용`, `수수료`, `차주운임` | 목록, 정산 흐름 | 하단 화물 목록의 결제/금액 컬럼과 연결 |
| `품목` | `4. 화물정보 요약` | `운송+품목` 박스의 2번째 row에서 입력 후 화물정보 요약 문구에 반영 |

요약 문장 연결의 상세 기준은 `11-summary-data-connection.md`와 `sections/cargo-summary-docs/06-cargo-transport-summary-data-contract.md`를 따릅니다. 금액 조건은 정산 정보이므로 요약 문장에는 기본 포함하지 않습니다.
