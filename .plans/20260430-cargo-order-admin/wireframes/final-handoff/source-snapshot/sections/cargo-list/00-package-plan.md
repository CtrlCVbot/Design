# Cargo List Package Plan

## 1. 목표

B 통합안 하단의 `화물 목록` 섹션을 독립적으로 기획, 검토, 와이어프레임화할 수 있는 패키지로 분리합니다.

차주 정보 섹션처럼 문서와 HTML이 서로 연결되는 구조를 사용합니다. 다만 이번 단계에서는 실제 HTML 구현을 시작하지 않고, 문서 패키지 구조와 작업 순서를 먼저 확정합니다.

## 3. 현재 화물 목록 섹션 분석

| 영역 | 현재 확인 내용 | 개선 전제 |
| --- | --- | --- |
| 목록 위치 | B 통합안 하단의 `list-area` | 상단 입력/수정 섹션과 분리된 운영 목록 |
| 상태 탭 | 전체목록, 접수목록, 완료목록, 취소 | 탭별 count, 상태 정의, 기본 선택 기준 필요 |
| 검색 안내 | 날짜와 검색 기준이 텍스트로 표시 | 실제 input, 검색 범위, 날짜 필터로 분리 검토 |
| 페이지 컨트롤 | 이전, 현재 페이지, 다음 | 페이지 크기, 총 건수, 고정 위치 검토 |
| 보기 옵션 | 처리시간순, 항목 너비 저장, 글자 크게 | 사용자 개인화 옵션과 목록 액션 분리 필요 |
| 테이블 컬럼 | 선택, 화물ID, 처리시간, 상태, 상하차지, 화물정보, 금액, 차주/차량 등 대량 컬럼 | 핵심 컬럼과 상세/보조 컬럼의 정보 구조 재정리 필요 |
| row 상태 | 선택 row, 취소 row 색상 구분 | 접수/완료/취소 외 배차중, 공유중 등 확장 상태 검토 |
| 상단 연결 | 선택 row가 상단 입력 결과와 연결된다는 안내만 존재 | row 클릭 시 상단 폼 로드, 수정 완료 후 목록 갱신 흐름 정의 필요 |

## 5. HTML 와이어프레임 방향

| 화면 상태 | 포함 이유 | 표현 방식 |
| --- | --- | --- |
| 기본 목록 | 현재 B 통합안의 기준 상태 | 상태 탭 + 검색/필터 + 목록 row |
| 필터 적용 | 운영자가 가장 자주 쓸 흐름 | 선택된 탭, 검색어, 정렬 상태를 명확히 표시 |
| 검색 결과 없음 | 빈 결과 대응 | 빈 테이블 대신 안내 row와 필터 초기화 CTA |
| 선택 row | 상단 입력 폼과 연결되는 핵심 상태 | 선택 row 강조, 상단 폼 로드 예정 표시 |
| 수정 후 갱신 | 상단에서 수정한 값이 목록에 반영되는 흐름 | 변경된 row highlight, 갱신 timestamp |
| 취소/완료 row | 상태별 시각 구분 | badge, row tone, 가능한 액션 제한 |
| 컬럼 밀도 비교 | 현재 컬럼이 많아 가로 스크롤이 큼 | 핵심 컬럼형, 전체 컬럼형, 상세 확장형 비교 |

## 6. 우선 검토할 정보 구조

| 구분 | 포함 후보 | 검토 기준 |
| --- | --- | --- |
| 핵심 식별 | 선택, 화물ID, 상태, 처리시간 | row를 빠르게 찾고 선택할 수 있는가 |
| 운송 요약 | 상차지, 하차지, 화물정보, 상하차일시 | 상단 운송구간/화물정보와 같은 언어를 쓰는가 |
| 금액 | 청구비용, 운송비용, 수수료, 수익/차주운임 | 최근 정리한 금액 조건과 충돌하지 않는가 |
| 차량/차주 | 차주명, 연락처, 차량번호, 톤수, 차종, 출처 | 차주 정보 Phase 2와 연결 가능한가 |
| 정산/증빙 | 사업자번호, 계산서 발행일, 계산서 금액 | 목록 기본 노출 또는 상세 확장 중 어디가 맞는가 |
| 운영 메타 | 처리자, 수정시간, 공유 상태, 배차 출처 | 운영 판단에 필요한가, 컬럼 과밀을 만들지는 않는가 |

## 8. B 통합안 반영 원칙 초안

| 원칙 | 내용 |
| --- | --- |
| 기존 tone 유지 | 색감, 선, 폰트, 밀도는 B Original Tone을 따른다. |
| 상단 섹션과 용어 통일 | 화주 정보, 운송구간, 화물 운송정보, 차주 정보에서 확정한 용어를 목록에도 사용한다. |
| 핵심 컬럼 우선 | 운영자가 반복 확인하는 컬럼은 고정 또는 전면 배치한다. |
| 보조 컬럼 정리 | 정산/증빙/사업자 정보는 기본 컬럼, 확장 row, 상세 패널 중 적절한 위치를 다시 판단한다. |
| 선택 row 연결 | 목록 row 선택 시 상단 오더 입력/수정 폼에 로드되는 흐름을 명확히 한다. |
| 가로 스크롤 완화 | 전체 컬럼 보존이 필요하더라도 핵심 컬럼 view와 전체 컬럼 view를 분리 검토한다. |

## 9. 다음 액션

1. `01-prd-cargo-list.md`를 작성해 화물 목록 섹션의 목표와 범위를 확정합니다.
2. 현재 테이블 컬럼을 핵심/보조/상세 후보로 분류합니다.
3. 기본 목록, 선택 row, 빈 결과, 필터 적용 상태를 wireframe 문서로 정리합니다.
4. 이후 독립 HTML을 만들고, B 통합안의 `list-area` 교체 기준을 작성합니다.
