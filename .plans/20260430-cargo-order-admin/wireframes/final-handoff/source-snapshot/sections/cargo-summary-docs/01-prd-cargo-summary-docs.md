# PRD - 화물정보 요약

## 1. 배경

`화물정보 요약`은 화물 운송정보에 입력된 톤수, 차종, 품목, 상하차 조건을 운영자가 빠르게 읽을 수 있는 한 문장으로 정리하는 파트입니다.

이번 단계에서는 기존에 함께 묶였던 중량, 계산 기준, 이하 여부, 안내문, 증빙 옵션을 제외하고 요약 문장만 남깁니다.

## 2. 목표

| 목표 | 설명 |
| --- | --- |
| 정보 밀도 유지 | 운송구간/화주 정보처럼 한 줄 row로 보여준다 |
| 디자인 통일 | 같은 종이 배경, 얇은 선, 라벨 chip, 손글씨형 값 표현을 사용한다 |
| 요약 집중 | 화물정보 요약 외 항목은 모두 제거한다 |
| 수정 진입 단순화 | `수정` 버튼 또는 요약 값 선택으로 같은 위치에서 수정한다 |

## 3. 범위

### 포함

| 항목 | 처리 |
| --- | --- |
| 화물정보 요약 | 화물 운송정보에서 파생된 문장형 요약 표시 |
| 값 없음 상태 | 화물 운송정보 입력 전 placeholder 표시 |
| 수정 중 상태 | 같은 row 안에서 input으로 전환 |

### 제외

| 제외 항목 | 이유 |
| --- | --- |
| 중량 | `3. 화물 운송정보`의 실중량에서 관리 |
| 계산 기준 | 톤수 110% 기준은 화물 운송정보 쪽에서 확인 |
| 이하 여부 | 요약 파트의 핵심 정보가 아니므로 제거 |
| 안내문 | 별도 안내/검증 정책에서 다룸 |
| 증빙 옵션 | 정산/출력 흐름에서 별도 정리 |

## 4. 요구사항

| ID | 요구사항 | 우선순위 |
| --- | --- | --- |
| CS-01 | 요약은 운송구간/화주 정보와 같은 한 줄 row로 보여준다 | High |
| CS-02 | row 첫 칸에는 `요약` 라벨을 둔다 | High |
| CS-03 | 값 칸에는 `화물정보 요약` 라벨과 요약 문장을 표시한다 | High |
| CS-04 | 값이 없을 때는 placeholder 문구를 표시한다 | Medium |
| CS-05 | 수정 버튼 클릭 시 같은 위치에서 input으로 전환한다 | Medium |
| CS-06 | 긴 문장은 row 안에서 말줄임 처리한다 | High |
| CS-07 | 요약 문장의 원본 데이터는 `3. 화물 운송정보`의 `품목`, `실중량`, `톤수`, `차종`, `대수`로 정의한다 | High |
| CS-08 | 상하차 일시와 상하차 방법은 `2. 운송 구간`에서 받아 요약 조건으로 사용한다 | Medium |
| CS-09 | 금액 조건은 정산 정보이므로 요약 문장에 기본 포함하지 않는다 | High |
| CS-10 | 자동 생성 요약과 사용자가 직접 수정한 override 문장을 분리한다 | High |

## 5. Acceptance Criteria

| 기준 | 통과 조건 |
| --- | --- |
| 디자인 | 운송구간/화주 정보 row와 같은 톤으로 보인다 |
| 범위 | 화면에 화물정보 요약 외 항목이 없다 |
| 상태 | 값 있음, 값 없음, 수정 중 상태를 확인할 수 있다 |
| 상호작용 | 수정 버튼으로 input 전환이 가능하다 |
| 데이터 연결 | `3. 화물 운송정보`의 핵심 필드가 요약 문장 원본으로 정의되어 있다 |
| 저장 정책 | 자동 생성값과 사용자 override 값을 구분할 수 있다 |
| 레이아웃 | desktop에서 가로 overflow 없이 보인다 |

## 6. 데이터 연결 결정

| 항목 | 결정 |
| --- | --- |
| 자동 생성 규칙 | `품목` -> `실중량` -> `톤수` -> `차종` -> `대수` 순으로 반영하고, 운송구간의 상하차 일시/방법을 보조 조건으로 붙인다 |
| 저장 방식 | 원본 필드는 수정하지 않고, 요약 문장 수정값은 별도 override로 관리한다 |
| stale 처리 | 원본 필드가 바뀐 뒤 override와 불일치할 수 있으면 `stale` 후보로 보고 `요약 재생성` 또는 `수정 유지`를 검토한다 |
| 세부 계약 | `06-cargo-transport-summary-data-contract.md`를 기준 문서로 사용한다 |

## 7. 남은 결정

| 항목 | 결정 필요 |
| --- | --- |
| 상태 chip | `자동`, `수정됨`, `재생성 필요` 같은 chip을 실제 row에 노출할지 결정 필요 |
| 운영 표기 | `축차/축카고`, `지금상/당착` 같은 요약 토큰 표기 확정 필요 |
