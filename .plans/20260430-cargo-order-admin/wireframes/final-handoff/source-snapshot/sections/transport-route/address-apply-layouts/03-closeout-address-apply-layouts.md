# Closeout: 운송구간 주소 검색/적용 레이아웃 기준

## 목적

이 문서는 `transport-route-address-apply-layouts.html` 기준의 주소 검색/적용 레이아웃 정리 상태를 고정한다.

기존 `transport-route-d-workflow.html`은 운송구간 D안의 원본 흐름으로 유지하고, 이 closeout은 별도 확장 HTML의 기준과 다음 인계 조건만 정리한다.

## 기준 상태

| 항목 | 상태 | 기준 |
| --- | --- | --- |
| 기준 HTML | 완료 | `../transport-route-address-apply-layouts.html` |
| 기존 D workflow | 유지 | `../transport-route-d-workflow.html` 미수정 |
| 1열 단일 행 | 완료 | 상차/하차 각각 한 줄 표시 |
| 2열 정리안 | 삭제 | 하위 패키지 범위에서 제거 |
| 오른쪽 메타 | 삭제 | 배차 유형, 계산 메타 제외 |
| 펼침 구조 | 삭제 | 상세주소, 담당자, 연락처를 기본 행에 포함 |
| 변경 버튼 | 완료 | `상차지 변경`, `하차지 변경` |
| 주소 검색/적용 | 완료 | 상차/하차 공용 다이얼로그 |
| 통합 검색 | 완료 | 화주 등록 주소록 + Kakao API 결과를 같은 리스트에 표시 |
| 출처 배지 | 완료 | `화주 등록`, `Kakao` 배지로 구분 |
| Kakao 결과 저장 | 완료 | 선택 입력 후 화주 주소록 저장 옵션 제공 |
| 저장 활성 규칙 | 완료 | Kakao는 지명 입력 시, 화주 등록은 변경 발생 시 활성 |
| B 통합본 반영 | 보류 | HTML 확인 후 반영 |

## HTML 반영 요약

| 화면 | 결과 |
| --- | --- |
| 1열 정리안 | 상차/하차를 각각 한 줄 명세표로 표시 |
| 주소 검색/적용 | 상차/하차 버튼에서 공용 다이얼로그 오픈 |
| 검색 결과 | 화주 등록 주소록과 Kakao 검색 결과를 출처 배지로 구분 |
| 선택 미리보기 | 주소, 상세주소, 지명, 담당자, 연락처 순서로 표시 |
| 주소 | 읽기 전용으로 고정 |
| Kakao 결과 | 지명 또는 장소명이 있어야 주소록 저장 활성 |
| 화주 등록 결과 | 상세주소, 지명, 담당자, 연락처 중 변경이 있을 때 수정 저장 활성 |
| 적용 | 선택값을 1열 단일 행에 반영 |
| 상세 수정 | 상세주소, 담당자, 연락처 input 전환 예시 |

## 검증 상태

| 검증 | 결과 | 근거 |
| --- | --- | --- |
| HTML 서버 응답 | 통과 | `HTTP=200`, content length `41264` |
| 스크립트 문법 | 통과 | `scriptSyntax=ok` |
| 2열 제거 | 통과 | 하위 HTML에서 `2열 정리안`, `two-col` 미검출 |
| 메타 제거 | 통과 | 하위 HTML에서 배차 유형, 계산 메타 미검출 |
| 변경 버튼 | 통과 | `상차지 변경`, `하차지 변경` 확인 |
| 다이얼로그 오픈 | 통과 | 상차/하차 변경 버튼 클릭 시 각각 `상차지 주소 검색`, `하차지 주소 검색` 표시 |
| 적용 결과 반영 | 통과 | 선택한 지명, 주소, 상세주소, 담당자, 연락처가 1열 행에 반영됨 |
| 통합 출처 표시 | 통과 | 브라우저에서 `화주 등록`, `Kakao` 결과 4개 확인 |
| 저장 옵션 기본 상태 | 통과 | 화주 등록 결과는 변경 전 수정 저장 비활성, Kakao 결과는 지명 있을 때 저장 활성 |
| Kakao 적용 | 통과 | Kakao 결과 선택 후 주소록 저장 체크 여부와 무관하게 1열 행 반영 |
| Kakao 지명 필수 | 통과 | 지명/장소명 비움 시 저장 비활성 확인 |
| 화주 등록 수정 저장 | 통과 | 변경 전 비활성, 상세주소 변경 후 활성 확인 |
| 주소 읽기 전용 | 통과 | 주소 input `readonly` 확인 |

## 남은 보류 항목

| 항목 | 보류 사유 | 다음 처리 |
| --- | --- | --- |
| 실제 Kakao API | wireframe 샘플 데이터 단계 | 개발 전 API contract 정의 |
| 좌표 저장/검증 | 백엔드 정책 필요 | 주소 선택 결과 모델 정의 |
| 주소록 저장 시점 | 백엔드 정책 필요 | 다이얼로그 적용 시 즉시 저장 또는 오더 저장 시 함께 저장 중 결정 |
| B 통합본 반영 | 현재는 하위 HTML 기준 | 사용자 확인 후 반영 |
| 모바일 실제 렌더링 | 후속 검토 | 이번 검증은 데스크톱 브라우저와 가로 overflow 확인 중심 |

## 마무리 판정

주소 검색/적용 레이아웃은 1열 단일 행과 통합 검색 기준으로 재정리되었다.

다음 작업은 HTML을 확인한 뒤, 이 구조를 B 통합본에 반영할지 결정하는 것이다.
# 4차 Closeout: 다이얼로그 레이아웃과 미리보기 모드

| 항목 | 상태 | 기준 |
| --- | --- | --- |
| 다이얼로그 레이아웃 | 완료 | 화주 정보 섹션의 `화주/담당자 검색` 다이얼로그처럼 검색/필터/조회 결과/선택 미리보기/하단 적용 구조로 정리 |
| 조회 결과 리스트 | 완료 | `선택`, `출처`, `지명`, `주소`, `상태` 컬럼으로 구성 |
| `화주 등록` 미리보기 | 완료 | 선택 직후 라벨 모드로 표시하고, `수정` 버튼으로 입력폼 전환 |
| `Kakao` 미리보기 | 완료 | 선택 직후 입력폼 모드로 표시 |
| 주소 수정 제한 | 완료 | 주소 원문은 읽기 전용으로 유지하고, 상세주소/지명/담당자/연락처만 수정 가능 |
| 주소록 저장 옵션 | 완료 | 같은 위치에 노출하되 조건 충족 전에는 비활성화 |

## 4차 검증 상태

| 검증 | 결과 | 근거 |
| --- | --- | --- |
| HTML 서버 응답 | 통과 | `HTTP=200`, content length `41264` |
| 스크립트 문법 | 통과 | `scriptSyntax=ok` |
| 화주 등록 라벨 모드 | 통과 | `preview-card readonly-mode`, 라벨 영역 표시, 입력폼 숨김 |
| 화주 등록 수정 전환 | 통과 | `수정` 클릭 후 `preview-card editing-mode`, 입력폼 표시 |
| Kakao 입력폼 모드 | 통과 | Kakao 결과 선택 후 `preview-card form-mode`, 입력폼 표시 |
| 가로 overflow | 통과 | 데스크톱 뷰포트 기준 `overflowX=false` |
