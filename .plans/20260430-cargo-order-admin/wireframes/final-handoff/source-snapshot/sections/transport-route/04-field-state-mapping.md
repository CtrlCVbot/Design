# Field & State Mapping: 운송구간

## 데이터 모델 제안

1차 표시 모드는 D안 `compact-hybrid`이다. A/B/C는 비교와 회귀 확인용 표시 모드로 유지한다.

```ts
type RoutePoint = {
  type: "pickup" | "dropoff";
  placeName?: string;
  address: string;
  detailAddress?: string;
  timeLabel?: "지금" | "당일" | "내일" | string;
  method?: string;
  managerName?: string;
  managerPhone?: string;
  noteCount?: number;
  noteSummary?: string;
};

type RouteCalculationReadiness = {
  hasPickupAddress: boolean;
  hasDropoffAddress: boolean;
  hasTruckTonnage: boolean;
  hasTruckType: boolean;
  distanceLabel?: string;
  baseAmountLabel?: string;
};

type DispatchTypeSummary = {
  primary: "독차" | "혼적";
  urgent?: boolean;
  roundTrip?: boolean;
  reserved?: boolean;
  waypointCount?: number;
};
```

## 필드 목록

| 필드 | 의미 | 기본 요약 | 펼침 | 빈 상태 |
| --- | --- | --- | --- | --- |
| `type` | 상차/하차 구분 | `[상차지]`, `[하차지]` | 동일 | `상차`, `하차` |
| `placeName` | 상차지명/하차지명 | 값이 있으면 표시 | 동일 | 표시 안 함 |
| `address` | 상세주소 제외 주소 | 표시 | 표시 | placeholder |
| `detailAddress` | 상세주소 | 숨김 | 표시 | placeholder |
| `timeLabel` | 상차/하차 일시 요약 | 방법과 한 줄 표시 | 표시 가능 | placeholder |
| `method` | 상차/하차 방법 | 일시와 한 줄 표시 | 표시 가능 | placeholder |
| `managerName` | 담당자명 | 숨김 | 표시 | placeholder |
| `managerPhone` | 연락처 | 숨김 | 표시 | placeholder |
| `noteCount` | 특이사항 수 | 배지 표시 | 표시 | 숨김 |
| `noteSummary` | 특이사항 요약 | 숨김 | 표시 | placeholder |
| `distanceLabel` | 거리 보조 메타 | 조건 충족 시 표시 | 표시 가능 | `주소 입력 후 계산` |
| `baseAmountLabel` | 기준 금액 보조 메타 | 조건 충족 시 표시 | 표시 가능 | `톤수·차종 선택 후 계산` |
| `dispatchType` | 배차 유형 대표값 | 운송구간 전체 칩으로 표시 | 표시 가능 | `독차` 기본값 |
| `dispatchOptions` | 긴급/왕복/예약 옵션 | 선택된 값만 칩으로 표시 | 표시 가능 | 숨김 또는 `옵션 없음` |
| `waypointCount` | 경유 수 | 값이 있으면 운송구간 전체 칩으로 표시 | 표시 가능 | 숨김 |

## 표시 조건

| 조건 | UI 처리 |
| --- | --- |
| 상차지와 하차지가 모두 있음 | 데이터 있음 요약 상태 |
| 둘 중 하나만 있음 | 부분 입력 상태. 있는 행만 표시하고 부족한 행은 입력 안내 |
| 둘 다 없음 | 빈 상태 |
| `placeName` 없음 | 지명 줄 렌더링 생략 |
| `detailAddress` 있음 | 기본 요약에서는 숨기고 펼침에만 표시 |
| `noteCount > 0` | `특이사항 N` 배지 표시 |
| `noteCount` 없음 또는 0 | 배지 숨김 |
| 상차/하차 주소가 모두 없음 | 거리와 기준 금액은 `주소 입력 후 계산` 표시 |
| 상차/하차 주소는 있으나 `톤수` 또는 `차종` 없음 | 기준 금액은 `톤수·차종 선택 후 계산` 표시 |
| 상차/하차 주소 + `톤수` + `차종` 있음 | 거리와 기준 금액을 정밀 표시 |
| `톤수` 또는 `차종` 변경 | 거리/기준 금액을 `재계산 필요` 상태로 전환 |
| 배차 유형 대표값 없음 | `독차`를 기본값으로 초기화하되, 입력 전 안내가 필요하면 `배차 유형 선택 전` 문구를 보조로 표시 |
| 배차 유형 대표값이 `독차` 또는 `혼적` | 운송구간 전체 보조 칩으로 표시. 두 값은 동시에 표시하지 않음 |
| `urgent`, `roundTrip`, `reserved` 중 true | 대표 배차 유형 옆에 선택된 옵션 칩만 추가 |
| `urgent`, `roundTrip`, `reserved`가 모두 false | 옵션 칩을 숨기거나 `옵션 없음`을 약하게 표시 |
| `waypointCount > 0` | `경유 {count}` 칩 표시 |

## 상태별 UI 매핑

| 상태 | 상차지 표시 | 하차지 표시 | 상세 정보 | 안내 |
| --- | --- | --- | --- | --- |
| 데이터 있음 | D안 접힘 요약 행 표시 | D안 접힘 요약 행 표시 | 접힘 | 없음 |
| 빈 상태 | placeholder | placeholder | 없음 | 주소 입력 안내 |
| 펼침 상태 | D안 요약 + 상세 | D안 요약 + 상세 | 라벨과 함께 표시 | 접기 동작 |
| 부분 입력 | 입력된 행만 D안 요약 | 미입력 행 placeholder | 입력된 행만 표시 | 부족한 주소 입력 안내 |

## 계산 보조 메타 매핑

| 의존 값 | 소유 섹션 | 운송구간 표시 영향 |
| --- | --- | --- |
| 상차 주소 | 운송구간 | 거리/기준 금액 계산의 필수 조건 |
| 하차 주소 | 운송구간 | 거리/기준 금액 계산의 필수 조건 |
| 톤수 | 화물 운송정보 | 기준 금액과 운송료 정확도에 영향 |
| 차종 | 화물 운송정보 | 기준 금액과 운송료 정확도에 영향 |
| 거리 | 계산 결과 | 주소와 차량 조건이 충분할 때 보조 메타로 표시 |
| 기준 금액 | 계산 결과 | `톤수`와 `차종` 선택 후 보조 메타로 표시 |
| 배차 유형 | 운송구간 내부 보조 메타 | 대표 선택 `독차/혼적`과 멀티옵션 `긴급/왕복/예약`을 운송구간 전체 보조 메타로 표시 |
| 상차 조건 | 운송구간 상차 행 상세 | 상차방법, 상차일시, 30분 옵션, 담당자/연락처를 상차 행 summary/detail로 표시 |
| 하차 조건 | 운송구간 하차 행 상세 | 하차방법, 하차일시, 당착/내착, 30분 옵션, 보험료 요약을 하차 행 summary/detail로 표시 |
| 경유 수 | 배차 유형/경유지 입력 | 상차/하차 행이 아니라 운송구간 전체 칩으로 표시 |

## 표시 모드 매핑

```ts
type TransportRouteDisplayMode = "sheet-row" | "sheet-card" | "stacked-card" | "compact-hybrid";
```

| 표시 모드 | 용도 | 필드 배치 |
| --- | --- | --- |
| `sheet-row` | A안. 비교 근거 | `구분 / 지명 / 주소 / 일시·방법 / 특이사항 / 상세`를 한 표 행에 가로 배치 |
| `sheet-card` | B안. 비교 근거 | 상차/하차를 카드로 나누되 카드 내부는 `지명 | 주소 | 일시·방법 | 특이사항` 가로 배치 |
| `stacked-card` | C안. 비교 근거 | 카드 내부에서 `지명`, `주소`, `일시·방법`을 세로로 쌓음 |
| `compact-hybrid` | D안. 1차 기준안 | 접힘 상태에서는 `상차/하차`, 지명, 주소, 일시·방법, 특이사항 값을 한 줄로 배치하고 라벨은 숨김 |

## B 통합본 병합 매핑

| 기존 섹션 | 이동 위치 | 표시 방식 |
| --- | --- | --- |
| 5 `배차 유형` | 1 `운송 구간` 하단 보조 메타 | `독차(기본)`, `혼적 전환 가능`, `예약`, `긴급·왕복 미선택` 칩 |
| 6 `상차 조건` | 상차 행 summary | summary에는 `지금 / 지게차`만 표시. 선택 화면은 별도 구현 |
| 7 `하차 조건` | 하차 행 summary | summary에는 `당일 / 지게차`만 표시. 선택 화면은 별도 구현 |
| 섹션 번호 | 전체 화면 번호 | `화주 정보`는 5번, `수수료율 정보`는 6번으로 재정렬 |

## 표시 모드별 펼침 매핑

| 표시 모드 | 상세주소 | 담당자/연락처 | 특이사항 요약 |
| --- | --- | --- | --- |
| `sheet-row` | 주소 컬럼 아래 상세 행 | 일시/방법 컬럼 아래 상세 행 | 특이사항 또는 우측 상세 영역 |
| `sheet-card` | 카드의 주소 영역 아래 | 카드의 일시/방법 영역 아래 | 카드의 우측/요약 영역 |
| `stacked-card` | 카드 하단 상세 영역 | 카드 하단 상세 영역 | 카드 하단 상세 영역 |
| `compact-hybrid` | 펼침 영역에서만 라벨과 함께 표시 | 펼침 영역에서만 라벨과 함께 표시 | 펼침 영역에서만 라벨과 함께 표시 |

## 포맷 규칙

| 항목 | 포맷 |
| --- | --- |
| 행 제목 | `[상차지]`, `[하차지]` |
| 주소 | `시/도 시/군/구 읍/면/동 리` |
| 일시/방법 | `{일시} · {방법}` |
| 특이사항 | `특이사항 {count}` |
| 담당자 | `{managerName} / {managerPhone}` |
| 배차 유형 | `{primary} · {selectedOption1} · {selectedOption2} · 경유 {count}`. 기본 `primary`는 `독차` |

## QA 체크

| 체크 | 기준 |
| --- | --- |
| 2행 구조 | 상차지 행 다음에 하차지 행이 나온다. |
| 기본 요약 | 상세주소, 담당자, 연락처가 기본 요약에 노출되지 않는다. |
| 빈 값 | 값 없는 지명은 줄 자체가 렌더링되지 않는다. |
| 특이사항 | 긴 설명은 배지로 축약되고 펼침에 상세가 나온다. |
| 빈 상태 | 사용자가 다음에 주소를 입력해야 한다는 안내가 보인다. |
| D안 기준 | `compact-hybrid` 닫힘 상태에서는 상세 라벨이 보이지 않고, 펼침 상태에서만 상세 라벨이 보인다. |
