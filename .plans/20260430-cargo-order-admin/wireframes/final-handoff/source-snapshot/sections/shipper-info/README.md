# 화주 정보 섹션 기획 패키지

## 목적

이 패키지는 화물 오더 접수/수정 화면의 `화주 정보` 섹션을 독립적으로 정리하기 위한 1차 산출물입니다. 현재 B 통합본에서는 화면 상단 `1. 화주 정보`로 배치합니다.

운송구간과 화물 운송정보 섹션처럼 전체 B 통합본에서 분리해, 먼저 필수 항목과 화면 상태를 작은 단위로 확인합니다.

## 패키지 상태

| 항목 | 상태 | 메모 |
| --- | --- | --- |
| 1차 정리 | 완료 | 필수 5개 항목, 조회 다이얼로그, 담당자 추가, inline edit 기준 고정 |
| B 통합본 반영 | 완료 | `Cargo Order Wireframe B Original Tone.html` 1번 섹션 반영 |
| 다음 섹션 | 준비됨 | `sections/driver-info`에서 `5. 차주 정보` 정리 시작 |

| 항목 | 역할 | 1차 처리 |
| --- | --- | --- |

| 담당자 이메일 | 증빙, 정산, 안내 발송 기준 | 필수 표시 |

## 1차 결정

| 결정 | 내용 |
| --- | --- |
| 표시 방식 | 값이 있으면 한 줄 요약 row로 보여준다 |
| 입력 진입 | 값이 없으면 `화주 정보 입력` CTA가 `화주` 라벨 쪽으로 흡수되는 전환을 사용한다 |
| 조회 진입 | 슬라이딩 전환 완료 후 `화주/담당자 통합 조회` 다이얼로그를 연다 |
| 선택 반영 | 결과 행 클릭은 선택만 하고, `화주 정보에 적용` 클릭 시 5개 항목을 한꺼번에 반영한다 |
| 취소 정책 | 선택 없이 닫기/취소 시 빈 row도 닫고 다시 `화주 정보 입력` CTA 상태로 돌아간다 |
| 수정 방식 | 조회 다이얼로그가 닫힌 뒤 현재 화주 정보 row에 표시된 `담당자 연락처`, `담당자 이메일`만 같은 위치의 input으로 임시 수정한다 |
| 보조 기능 | 조회, 사업자 번호 검증, 담당자 자동완성은 1차 화면에서는 제외하고 후속 기능으로 둔다 |
| 상태 보기 | 독립 HTML에서는 `적용 후 row`, `입력 전/조회`, `수정 기준`, `변경 애니메이션`을 탭으로 전환해 중복 섹션처럼 보이지 않게 한다 |
| 편집 가능 표시 | 연락처/이메일 값의 `수정` 배지는 기본 숨김으로 두고, hover/focus 때만 노출한다 |
| 담당자 추가 | 문서에서는 3개 대안을 비교하고, HTML에서는 행 선택 이후 우측 미리보기 하단에 담당자 추가를 노출한다 |
| 담당자 역할 | `배차`, `정산`, `관리`를 멀티 선택할 수 있고 기본값은 `배차`다 |
| 추가 후 상태 | 담당자 추가 시 조회 리스트에 `신규` 행을 추가하고, 해당 행을 자동 선택한다 |
| 화주 변경 | 기존 `수정` 버튼 문구는 `화주/담당자 변경`으로 고정하고, `화주 정보 입력`과 동일한 통합 조회 다이얼로그를 연다 |
| 변경 애니메이션 | A안 `버튼 흡수형`을 적용안으로 확정하고, 독립 HTML에는 A~E 후보 비교 탭을 참고용으로 유지한다 |
| 임시 수정 범위 | 연락처/이메일 인라인 수정은 현재 오더 적용용이며 다이얼로그 조회 리스트와 화주/담당자 마스터 데이터는 수정하지 않는다 |
| B 통합본 반영 | `Cargo Order Wireframe B Original Tone.html`의 1번 섹션에 A안 흡수 전환, 조회 다이얼로그, 담당자 추가 기능을 반영했다 |
| B 통합본 정렬 | 화주 정보, 운송구간, 화물 운송정보의 row 높이를 32px 리듬으로 맞추고 가로 overflow를 제거했다 |

## 사용 방법

1. `shipper-info-section-wireframe.html`을 브라우저에서 열어 값 없음, 값 있음, 수정 흐름을 확인합니다.
2. `02-wireframe-shipper-info.md`에서 B 통합본에 들어갈 row 구조를 확인합니다.
3. B 통합본의 `1. 화주 정보` 섹션에서 `화주/담당자 변경` 버튼의 A안 흡수 전환, 조회 적용, 담당자 추가 흐름을 확인합니다.
