# 01. 화면 섹션 맵

## 전체 구조

현재 화면은 한 화면 안에 `오더 입력 폼`, `배차/상하차 조건`, `의뢰자/수수료`, `업무 액션`, `목록 조회`, `화물 목록 테이블`이 모두 들어간 고밀도 업무 화면입니다.

최신 통합 HiFi 기준인 `results/html/cargo-order-admin-hifi-master.html`에서는 기존 우측 `예약 영역`을 `보조 정보` 패널로 정리합니다. 이 패널은 접수/수정 입력값 자체를 다시 반복하지 않고, 선택된 화물 운송건을 운영자가 판단할 때 필요한 메모, 금액 조정 내역, 운송 구간 확인 정보를 탭으로 제공합니다.

같은 master 기준에서 `운송 구간`과 `화물 정보`의 주요 다이얼로그에는 최근 사용 후보가 보조 상태로 포함됩니다. 상차지/하차지 주소 검색은 기존 검색 결과 영역의 초기 상태로 최근 장소를 보여주고, `운송+품목 입력`은 최근 조합을 입력폼에만 반영합니다. 상세 기준은 `sections/transport-dialog-recent-lists/`를 따릅니다.

2026-06-18 최신 master 기준에서는 header 상태 영역도 정리했습니다. 라벨 표시/숨김은 `Aa` icon button으로 축소하고, 배차 담당자는 독립 섹션을 추가하지 않고 상태 chip 옆 `배차 김민지` chip으로 표시합니다.

├──────────────────────────────────────────────────────────────────────────────┤

├──────────────────────────────────────────────────────────────────────────────┤

├──────────────────────────────────────────────────────────────────────────────┤
│ D. 표시 옵션 / 거리·금액 / 배차 담당자 / 문의 전화 / 광고 / 오더 현황         │
├──────────────────────────────────────────────┬───────────────────────────────┤
│ E. 오더 기본 입력 폼                         │ F. 배차·상하차·의뢰자 조건     │
│ - 상차지, 하차지                              │ - 독차/혼적, 긴급/왕복/예약    │
│ - 톤수, 차종, 대수                            │ - 상차방법, 하차방법           │
│ - 운송료, 수수료, 합계                        │ - 상차일시, 하차일시           │
│ - 추가정보, 화물정보                          │ - 의뢰자, 도착 연락처          │
│ - 증빙 옵션, 차주 정보                        │ - 주선약관 수수료율             │
├──────────────────────────────────────────────┴───────────────────────────────┤
│ K. 보조 정보 패널: 메모 / 금액 로그 / 운송 구간 지도                           │

│ G. 오더 처리 액션: 신규, 등록, 수정, 복사, 화물취소, 배차취소 등              │
├──────────────────────────────────────────────────────────────────────────────┤
│ H. 목록 필터 / 검색 / 정렬 / 표시 설정 / 출력·정산 액션                       │
├──────────────────────────────────────────────────────────────────────────────┤

├──────────────────────────────────────────────────────────────────────────────┤
│ J. 하단 상태 바: 회사명, 로그인 ID, 사업자 유형                               │
└──────────────────────────────────────────────────────────────────────────────┘

### 최신 HiFi header 상태 영역

| 항목 | 현재 master 표현 | 역할 |
| --- | --- | --- |
| 상태 chip | `신규`, `접수` 등 | 오더의 현재 처리 상태 |

## K. 보조 정보 패널

`보조 정보`는 입력 폼을 보완하는 우측 패널입니다. 입력해야 하는 필드를 늘리는 영역이 아니라, 이미 선택된 오더에 대해 운영자가 빠르게 확인할 보조 판단 정보를 모아 보여주는 영역입니다.

| 탭 | 목적 | 핵심 표시 |
| --- | --- | --- |
| 메모 | 운영자 메모, 고객 요청, 배차 특이사항을 확인 | 리스트 우선 표시, 추가는 다이얼로그 |
| 금액 로그 | 기준 청구금/배차금에서 증감된 `조정금`과 수익을 확인 | 명세서형 요약, 상세 항목 접기/펼치기 |
| 운송 구간 지도 | 상차지와 하차지의 이동 관계를 확인 | 지도, 거리, 예상 시간, 경유 여부, 주소 미입력 상태 |

### 표시 상태

| 상태 | 화면 기준 |
| --- | --- |
| 데이터 있음 | 기존 오더 조회/수정 기본 상태. `메모`, `금액 로그`, `운송 구간 지도` 탭 순서로 표시 |
| 신규 접수 초기화 | `신규(F3)` 클릭 후 전체 데이터가 초기화되면 세 탭 모두 빈 상태로 표시 |
| 부분 데이터 있음 | 화주 선택 시 메모 mock 표시, 상차지+하차지 적용 시 지도 mock 표시, 정산 정보 적용 시 금액 로그 mock 표시 |
| 숨김/축소 | 보조 판단 정보가 업무 흐름을 방해하는 상태에서는 패널을 숨기지 않고 탭별 빈 상태로 대체 |

## G. 오더 처리 액션

| 버튼 | 역할 |
| --- | --- |
| `신규(F3)` | 신규 접수 플로우 시작. 전체 입력 상태를 초기화하고 `new-reset`으로 진입 |
| `화물등록` | 메인 화면에 적용된 오더 데이터를 실제 API로 등록하는 최종 CTA |
| `화물수정` | 선택된 오더 수정 |
| `화물복사` | 기존 오더를 복제해 새 오더 작성 |
| `화물취소` | 오더 취소 |
| `배차취소` | 배차된 차주/차량 취소 |
| `취소자료 감추기` | 취소 오더를 목록에서 숨김 |
| `엑셀` | 목록 내보내기 |
| `인쇄` | 출력 |
| `정보변경` | 선택 오더의 부가 정보 변경 |
| `화물위치` | 차주 또는 화물 위치 확인 |
| `매체오더 일괄변경` | 외부 매체 오더의 일괄 변경 |
| `위탁증` | 위탁증 출력 또는 확인 |

### 신규 접수 상태 흐름

`신규(F3)`는 단순 빈 폼 전환이 아니라 신규 접수 전용 흐름의 진입점이다. 상세 기획은 `sections/new-order-registration-flow/`를 기준으로 한다.

| 단계 | 상태 | 화면 정책 |
| --- | --- | --- |
| 신규 접수 클릭 직후 | `new-reset` | 모든 섹션 초기화, 신규 접수 안내형 섹션 헤더 표시, 화주 정보 focus |
| 최초 입력 진행 | `new-wizard-active` | wizard 다이얼로그와 왼쪽 프로세스 패널 표시 |
| 금액까지 필수 입력 완료 | `new-required-complete` | 마지막 금액 다이얼로그에서 `화물 등록 완료` 또는 `차주 정보로 이동` 제공 |
| 차주 정보 선택 | `new-driver-optional` | 차주 정보는 선택 사항이며 건너뛰기 가능 |
| 메인 화면 적용 | `new-submitted` | API 통신 없이 wizard 데이터를 메인 화면에 적용, 섹션 헤더 숨김, 메인 `화물등록` 표시 |

`화물등록`은 `new-submitted` 이후 실제 API 통신을 시작하는 버튼이다. 실제 API endpoint, payload schema, 서버 validation code는 후속 개발 착수 전까지 보류한다.
