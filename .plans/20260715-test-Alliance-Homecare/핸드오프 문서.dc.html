03 · 모션 구현

motion.js와 데이터 속성

스크롤 모션은 마크업의 데이터 속성 + 공용 모듈 하나로 동작합니다. 라이브 스펙·데모는 모션 v1.1 문서 참조.

data-reveal="1"
뷰포트 진입 시 1회, 24px 상승 + 페이드 (500ms · cubic-bezier(0.22, 1, 0.36, 1))
data-reveal-group="1"
컨테이너의 직계 자식들을 70ms 간격 스태거로 등장 (최대 5개까지 지연, 이후 동시)
data-count="1200" data-count-suffix="+"
숫자 카운트업 900ms ease-out. 초기 텍스트는 최종값으로 두기 (JS 실패 시 그대로 노출) · tabular-nums 필수
Fail-safe 원칙
콘텐츠를 미리 숨기지 않고 WAAPI(element.animate)로 등장만 재생 — 옵저버·JS가 죽어도 콘텐츠는 항상 보임. 이 원칙을 유지할 것
히어로 진입
CSS @keyframes heroIn + 70ms 지연 스태거(랜딩은 아이브로우→제목→본문→CTA 4단)
내비 축소
scroll 24px 초과 시 78→60px + 그림자, 로고 23→19px (250ms) — 사이트 내비 컴포넌트 내장
페이지 전환 · 접근성
@view-transition { navigation: auto; } 크로스페이드 · prefers-reduced-motion 시 전 모션 비활성 · 앵커 scroll-margin-top: 98px
04 · 구현 노트 & 주의사항

이관 시 반드시 알아야 할 것

스타일 규칙
· 모든 스타일은 인라인. 전역 CSS는 리셋·키프레임·::selection·호버 파생 규칙만
· 호버/포커스/프레스 상태값에는 !important가 붙어 있음 — 인라인 기본값을 이기기 위한 의도적 처리로, 일반 CSS 재구현 시에는 클래스 기반으로 정리 가능
· 상태 피드백 표준: 호버 리프트 -2px + 딥 톤, 프레스 scale(0.98), 포커스 링 3px (테라코타 35%)
· 카드 이미지 줌·화살표 슬라이드는 [data-card] · [data-arrow] 셀렉터로 동작
콘텐츠 · 데이터
· 이미지는 전부 image-slot 플레이스홀더 — 실서비스에선 실제 사진(img)으로 교체. 슬롯 id: hero-portrait · promise-portrait · about-story · svc-nursing/aide/chronic/membership/senior/pediatric
· 브랜드명·전화번호(1588-2470)·후기·통계(20년/1,200+/97%)·연혁·언론사명은 모두 가상 샘플 — 실데이터로 교체 필수
· 상담 폼 제출은 프런트 시뮬레이션(1.1s 후 완료 화면) — 실제 API 연동 지점은 상담 신청 페이지 submit 핸들러
