# 차주용 앱 신규 기능 아이디어 Backlog

## 1. 분류 기준

| 우선순위 | 의미 |
| --- | --- |
| MVP | 배차 확인, 상차완료/하차완료 직접 상태 변경, 정산 조회에 직접 필요한 기능 |
| Phase 2 | 하차 담당자 확인 링크, 모바일 서명, 확인 상태, 화주 확인 보강에 필요한 기능 |
| Phase 3 | 알림, 사진 증빙, 대기료/추가비, 오프라인 저장 등 운영 편의와 현장 사용성을 높이는 기능 |
| Phase 4 | 화주용 서비스, 외부 플랫폼, 서류함, 복수 계정 등 주변 시스템과 연결되는 확장 기능 |
| Later | 정책, 비용, 신뢰 리스크가 커서 현재 Epic 이후로 검토할 기능 |

## 2. 기능 아이디어 목록

| 기능 아이디어 | 사용자 가치 | 필요한 화면/기능 | 우선순위 | 비고 |
| --- | --- | --- | --- | --- |
| 오늘 운행 홈 | 앱을 열자마자 오늘 해야 할 운행을 확인 | 오늘 배차 카드, 진행 상태, 다음 액션 버튼 | MVP | 검색형 홈보다 업무형 홈 권장 |
| 내 배차 목록 | 배차받은 화물을 놓치지 않음 | 상태 탭, 기간 필터, 배차 카드 | MVP | 현재 `results` 화면 구조를 재활용 가능 |
| 배차 상세 | 운행에 필요한 정보를 한 화면에서 확인 | 상하차지, 담당자, 화물, 운임, 특이사항 | MVP | 현재 `ticket` 화면 확장 |
| 상차완료 버튼 | 주선사에게 진행상태를 빠르게 공유 | 상태 버튼, 확인 modal, 시각 기록 | MVP | 위치/사진은 옵션 |
| 하차완료 버튼 | 하차 도착과 작업 완료를 빠르게 공유 | 상태 버튼, 확인 modal, 시각 기록 | MVP | MVP에서는 링크 없이 하차완료 상태를 직접 공유 |
| 하차 담당자 확인 링크 | 인수증 대체/보조 증빙 확보 | 모바일 확인 페이지, 서명, 확인 기록 | Phase 2 | MVP 이후 별도 기획 |
| 특이사항 보고 | 파손, 수량 차이, 대기 등 이슈를 즉시 공유 | 사진, 메모, 유형 선택 | MVP | 운행완료/정산 보류와 연결 |
| 월별 운행 내역 | 차주가 수입과 운행량을 확인 | 월 캘린더, 기간 필터, 상태 필터 | MVP | 정산 확인과 연결 |
| 송금 상태 조회 | 송금 여부 문의를 줄임 | 송금일, 금액, 정산 상태 | MVP | 차주는 조회 중심 |
| 정산 문의 | 금액 이견을 빠르게 접수 | 문의 버튼, 배차 담당자 연결, 메모 | MVP | 담당자 연락처 필요 |
| 길찾기 연동 | 상하차지 이동 편의 | 지도 앱 열기, 주소 복사 | MVP | Kakao/Naver/Tmap 선택 가능 |
| 담당자 빠른 연락 | 현장 지연/주소 문의 감소 | 전화, 문자, 복사 버튼 | MVP | 개인정보 마스킹 정책 필요 |
| 상태 타임라인 | 현재 어디까지 처리됐는지 명확 | 배차됨 -> 상차완료 -> 하차완료 -> 운행완료 | MVP | 상세 화면에 고정 표시 |
| 확인 링크 재발송 | 담당자가 링크를 못 받은 경우 해결 | 재발송 버튼, 발송 이력 | Phase 2 | 하차 담당자 확인 링크와 함께 기획 |
| 배차 알림 | 새 배차를 놓치지 않음 | push/SMS/Kakao 알림 | Phase 3 | MVP는 목록 조회 중심 |
| 상하차 예정 알림 | 시간 지연을 줄임 | 출발 전/상차 전 reminder | Phase 3 | 설정형 알림 필요 |
| 운행 전 체크리스트 | 준비물/차량 조건 누락 방지 | 품목, 차량, 상차방법 확인 checklist | Phase 3 | 위험 화물/냉장 등 확장 가능 |
| 사진 증빙 묶음 | 인수증 외 증빙 강화 | 상차 사진, 하차 사진, 파손 사진 | Phase 3 | 저장 용량/보존 정책 필요 |
| 전자 인수증 PDF | 정산/분쟁 대응 자료 확보 | 확인 기록 PDF 다운로드 | Phase 2 | 실무 정산 처리 기준 확인 필요 |
| 차량/프로필 관리 | 배차 조건 매칭 정확도 향상 | 차량번호, 톤수, 차종, 연락처 관리 | Phase 3 | 관리자 승인 정책 필요 |
| 계좌 정보 확인 | 송금 오류 방지 | 계좌 별칭, 예금주 확인 | Phase 3 | 보안/개인정보 민감 |
| 운행 가능 지역 설정 | 맞지 않는 배차 감소 | 선호 지역, 회차 가능 지역 | Phase 3 | 배차 추천과 연결 |
| 즐겨찾는 상하차지 | 반복 운행 편의 | 최근 장소, 즐겨찾기 | Phase 3 | 주소록 scope 필요 |
| 대기료/추가비 요청 | 현장 대기 비용 반영 | 추가비 요청, 사진/메모, 승인 상태 | Phase 3 | `PricingAdjustment`와 연결 |
| 운행 중 위치 공유 | 담당자 문의 감소 | 위치 공유 toggle, ETA 표시 | Phase 3 | 개인정보/배터리 이슈 |
| 하차 담당자 직접 수정 요청 | 연락처 오류 대응 | 담당자 연락처 수정 요청 | Phase 2 | 확인 링크 발송/재발송과 함께 기획 |
| 오프라인 임시 저장 | 통신 불량 현장에서 상태 누락 방지 | 상태 변경 임시 저장, 재전송 | Phase 3 | 중복 처리 방지 필요 |
| 복수 차량/기사 계정 | 법인/팀 차주의 운영 지원 | 차량 선택, 기사 전환 | Phase 4 | 권한 모델 복잡 |
| 세금계산서/서류함 | 정산 서류 확인 | 계산서, 위탁증, 전자 인수증 목록 | Phase 4 | 관리자 문서 시스템과 연동 |
| 매출 리포트 | 월별 운행/수입 분석 | 차트, CSV, 기간 비교 | Phase 3 | 데이터 정합성 필요 |
| 주선사 평가/메모 | 차주 경험 관리 | 주선사별 메모, 비공개 평가 | Later | 운영 정책 민감 |
| 자동 배차 추천 | 운행 조건에 맞는 신규 화물 추천 | 추천 목록, 수락/거절 | Phase 4 | 현재 목표는 배차받은 내역 관리 |

## 3. MVP 추천 묶음

| 묶음 | 포함 기능 | 이유 |
| --- | --- | --- |
| 운행 수행 묶음 | 내 배차 목록, 배차 상세, 상차완료, 하차완료 | 차주가 매일 사용하는 핵심 흐름 |
| 상태/예외 묶음 | 상태 타임라인, 특이사항 보고 | 주선사가 운행완료와 정산 가능 여부를 판단할 수 있는 최소 근거 |
| 정산 조회 묶음 | 운행 내역, 송금 상태, 정산 문의 | 송금 문의를 줄이고 신뢰를 높임 |

## 4. 후속 Phase 추천 묶음

| Phase | 묶음 | 포함 기능 | 이유 |
| --- | --- | --- | --- |
| Phase 2 | 하차 확인 고도화 | 하차 담당자 확인 링크, 서명, 확인 링크 재발송, 전자 인수증 PDF, 하차 담당자 직접 수정 요청 | 인수증 대체/보조 증빙과 송금 판단 속도를 강화 |
| Phase 3 | 운영 편의 | 배차/상하차 알림, 체크리스트, 즐겨찾기, 오프라인 저장 | 현장 사용성을 빠르게 개선 |
| Phase 3 | 증빙/비용 처리 | 사진 묶음, 대기료/추가비 요청, 매출 리포트 | 분쟁/정산 대응력과 차주 수익 확인성을 높임 |
| Phase 4 | 확장 연동 | 화주용 서비스 확인, 세금계산서/서류함, 화물맨 연동, 복수 기사/법인 계정, 자동 배차 추천 | 주변 시스템과 권한 모델이 필요하므로 장기 확장으로 분리 |

## 5. Later 후보 판단

## 6. 디자인 초안 반영 메모

| 현재 초안 요소 | 유지 여부 | 조정 방향 |
| --- | --- | --- |
| 파란색 브랜드 헤더 | 부분 유지 | 업무 앱 첫 화면에서는 높이를 줄이고 오늘 운행 중심으로 변경 |
| 운임순/거리순/상차순 필터 | 유지 | `내 배차`에서는 상태/기간 필터를 더 우선 |
| 배차 카드 | 유지 | 상태 badge, 다음 액션, 하차완료/정산 상태 표시 추가 |
| 배차 상세 티켓형 카드 | 유지 | 바코드 영역은 상태 타임라인/운행 기록 영역으로 재해석 |
| 배차받기 버튼 | 조정 | 이미 배차된 내역에서는 `상세보기` 또는 `배차 확인`으로 변경 |
