# 3차 퍼블리싱 기능/IA 리뷰

## 1. 결론

3차 퍼블리싱의 디자인 시스템은 유지한다. `배차 확인증`과 티켓형 카드 모티프는 차주용 화물 앱의 정체성을 잘 만든다.

최종 Phase 1 MVP 퍼블리싱은 3차 원본 디자인을 복제한 뒤, 하단 `정산` 탭의 도착지만 특정 건의 `정산/송금 상세`에서 월별/기간별 정산 요약을 보여주는 `정산 홈`으로 변경한다.

채택 IA는 A안으로 고정했다. 하단 `정산` 탭은 유지하되, 특정 건의 상세 화면이 아니라 집계형 `정산 홈`으로 진입한다.

| 안 | 하단 탭 | 판단 |
| --- | --- | --- |
| A안 채택 | `내 배차` / `운행 내역` / `정산` | `정산` 탭을 집계형 정산 홈으로 승격한다. |
| B안 대안 | `내 배차` / `운행 내역` / `문의` 또는 `더보기` | 정산은 `운행 내역`에서 건별 상세로만 진입한다. |

## 2. 3차 퍼블리싱 현재 구조

| 화면 | 현재 역할 | 기능 기준 판단 |
| --- | --- | --- |
| 내 배차 목록 | 오늘/진행/보류 배차 확인, 다음 액션 진입 | 유지 |
| 배차 상세 | 배차 확인증, 상태 타임라인, 상하차 정보, 상태 CTA | 유지 |
| 상차완료/하차완료 modal | 상태 변경 전 확인 | 유지 |
| 운행 내역 | 월별 운행 목록, 상태 필터, 건별 정산 상세 진입 | 유지하되 정산 홈과 역할 분리 완료 |
| 정산상세 | 특정 운행 건의 송금상태/송금일/송금금액 확인 | 하단 탭 직접 진입 대상이 아니라 child screen으로 유지 |
| 특이사항 보고 | 유형, 메모, 사진 optional, 보류 후보 | 유지 |

## 3. 불필요하거나 재정의할 부분

| 항목 | 현재 문제 | 최종 반영 |
| --- | --- | --- |
| 하단 `정산` 탭의 직접 상세 진입 | 탭을 눌렀는데 특정 운행 건 상세가 열려 맥락이 약하다. | `SCR-008 정산 홈`을 하단 탭 도착 화면으로 추가했다. |
| `onNavSettlement` 기본값 | 현재 첫 번째 송금완료 건을 열기 쉬운 구조다. | `settlementHome`으로 이동하도록 변경했다. |
| `정산/송금 상세`의 탭 소속 | 건별 상세는 `운행 내역`과 `배차 상세`에서 들어갈 때 자연스럽다. | 하단 탭의 기본 화면이 아니라 `운행 내역` 또는 `정산 홈`의 child screen으로 둔다. |
| `정산 확인` CTA 중복 | 상세 화면 CTA와 하단 정산 탭이 같은 상세 화면을 열면 기능이 겹친다. | 상세 CTA는 건별 상세 진입, 하단 탭은 집계 화면 진입으로 분리했다. |
| 공식 문서처럼 보이는 `배차 확인증` 표현 | 디자인 정체성은 좋지만 실제 법적/운영상 공식 확인증으로 오해될 수 있다. | 내부 정책에 따라 `배차 요약`, `운행 확인`, `배차 확인` 중 선택한다. |
| 과도한 티켓 카드 장식 가능성 | 3차 디자인의 장점이지만 반복 목록에서 그림자/반경이 강하면 피로도가 생길 수 있다. | 현재 퍼블리싱에서는 3차 디자인을 유지하고, 후속 구현에서만 조정 검토한다. |

## 4. 퍼블리싱 반영된 추가 부분

### 4.1 정산 탭 고정 기준

`정산` 탭은 아래 정보를 한 화면에서 요약한다.

| 정보 | 설명 | 예시 문구 |
| --- | --- | --- |
| 이번 달 송금 요약 | 해당 기간의 송금완료, 미정산, 보류 합계 | `이번 달 송금 요약 1,190,000원` |
| 송금완료 금액 | 송금일과 송금금액이 확정된 합계 | `송금완료 470,000원` |
| 미정산 금액 | 운행완료 또는 정산대기 상태의 합계 | `미정산 410,000원` |
| 보류 금액 | 특이사항/확인 필요로 보류된 합계 | `보류 310,000원` |
| 다음 송금 예정 | 가장 가까운 송금 예정일과 금액 | `다음 송금 06.28 · 470,000원` |
| 상태별 내역 | 정산대기, 송금완료, 보류 탭 또는 섹션 | `정산대기 2건`, `보류 1건` |
| 정산 문의 CTA | 보류/미정산 이슈를 문의로 연결 | `정산 문의` |

### 4.2 정산 홈 화면 고정안

최종 화면 ID는 `SCR-008 정산 홈`으로 둔다.

```text
┌────────────────────────────┐
│ 정산                       │
│ 2026년 6월                 │
├────────────────────────────┤
│ 이번 달 송금 요약          │
│ 총액     1,190,000원       │
│ 송금완료 470,000원         │
│ 미정산   410,000원         │
│ 보류     310,000원         │
├────────────────────────────┤
│ 다음 송금 예정             │
│ 06.28 · 470,000원 · 1건    │
├────────────────────────────┤
│ [전체] [정산대기] [송금완료] [보류] │
├────────────────────────────┤
│ 정산 묶음 카드             │
│ 06.28 송금예정 · 470,000원 │
│ 포함 운행 1건              │
├────────────────────────────┤
│ 미정산/보류 카드           │
│ 수량 차이 확인 중          │
└────────────────────────────┘
```

### 4.3 운행 내역과 정산 홈의 역할 분리

| 화면 | 핵심 질문 | 보여줄 정보 |
| --- | --- | --- |
| 운행 내역 | 내가 언제 어떤 운행을 했나? | 운행 건, 구간, 상태, 배차금, 건별 정산 상세 링크 |
| 정산 홈 | 이번 달 얼마가 송금됐고 얼마가 남았나? | 집계 금액, 송금 예정, 미정산/보류 요약, 정산 묶음 |
| 정산/송금 상세 | 이 운행 건의 정산 상태는 무엇인가? | 배차금, 조정 요약, 송금금액, 송금일, 송금상태 |

## 5. 기능 아이디어

| 아이디어 | MVP 적합도 | 설명 |
| --- | --- | --- |
| 월별 정산 요약 카드 | 높음 | `송금완료`, `미정산`, `보류` 금액을 한 번에 보여준다. |
| 다음 송금 예정 카드 | 높음 | 차주가 가장 궁금해할 `언제 얼마 들어오나`를 바로 보여준다. |
| 상태별 정산 필터 | 높음 | `전체`, `정산대기`, `송금완료`, `보류`로 묶어 조회한다. |
| 정산 묶음 카드 | 중간 | 같은 송금일에 묶인 운행들을 한 카드로 보여준다. |
| 보류 사유 요약 | 높음 | 보류 금액과 사유를 함께 보여 정산 문의를 줄인다. |
| 정산 문의 deep link | 높음 | 특정 정산 건/보류 건 맥락을 담당자 문의로 넘긴다. |
| 송금 캘린더 | 낮음 | 보기 좋지만 Phase 1 MVP에서는 과할 수 있다. |
| 수입 추이 그래프 | 낮음 | 차주에게 유용할 수 있으나 MVP 이후가 적절하다. |
| 엑셀/PDF 다운로드 | 제외 | 백엔드/문서 생성 계약이 필요하므로 Phase 1 MVP 밖이다. |
| 송금 계좌 표시 | 제외 | 개인정보/금융정보 노출 리스크가 있어 Phase 1에서 제외한다. |

## 6. 정산 상태 정의 보강

정산 홈은 금액 집계 기준을 화면 문구로 명확히 해야 한다.

| 상태 | 집계 포함 | 차주 노출 문구 |
| --- | --- | --- |
| 운행완료 | 미정산 후보 | `정산 확인 중` |
| 정산대기 | 미정산 금액 | `송금 전 정산 처리 중` |
| 송금완료 | 송금완료 금액 | `송금완료` |
| 보류 | 보류 금액 | `확인 필요` |

주의: `하차완료`는 차주가 현장 완료를 등록한 상태일 뿐, 미정산 금액에 바로 포함하지 않는다. 주선사의 `운행완료` 또는 정산 가능 판단 이후 정산 집계 대상으로 본다.

## 7. 디자인 유지 기준

3차 디자인 시스템은 유지한다.

| 요소 | 유지/조정 |
| --- | --- |
| 티켓형 배차 카드 | 유지 |
| `배차 확인증` 모티프 | 유지하되 명칭 정책 확인 |
| 블루 primary CTA | 유지 |
| 상태별 색상 체계 | 유지 |
| 절취선/거리/트럭 아이콘 | 유지 |
| 큰 그림자와 18px 반경 | 현재 퍼블리싱에서는 원본 유지, 후속 구현에서만 절제 검토 |
| 하단 탭 | `정산` 탭을 정산 홈으로 재정의 완료 |

## 8. 퍼블리싱 반영 결과

1. `SCR-008 정산 홈`을 기준 화면으로 추가하고, 기존 `SCR-006 정산/송금 상세`는 child screen으로 유지했다.
2. `운행 내역`은 운행 중심, `정산 홈`은 금액 집계 중심으로 분리했다.
3. 3차 퍼블리싱의 `onNavSettlement`는 첫 번째 정산 상세가 아니라 정산 홈으로 이동하게 변경했다.
4. 공통 화면인 `내 배차`와 `운행 내역`은 Visual QA에서 원본 대비 픽셀 차이 `0%`로 확인했다.
5. `배차 확인증` 명칭은 현재 디자인 원본을 유지하되, 운영 문서 명칭 정책은 후속 확인 항목으로 남긴다.
