# Wireframes - Phase 1A 운행 수행

## 1. 공통 UI 정책

| 항목 | 정책 |
| --- | --- |
| 첫 화면 | `내 배차 목록`을 앱의 홈으로 둔다. |
| 하단 탭 | `내 배차`, `운행 내역`, `정산` 중심으로 구성한다. `정산` 탭은 `SCR-008 정산 홈`으로 진입한다. |
| 상태 badge | `배차됨`, `운행 준비`, `상차완료`, `하차완료`, `운행완료`, `정산대기`, `송금완료`, `보류`를 텍스트와 색상으로 함께 표시한다. |
| CTA | 화면별 1차 CTA는 하단 고정 영역 또는 카드 하단에 둔다. |

## 2. SCR-001 내 배차 목록

### 목적

차주가 앱을 열자마자 오늘 처리해야 할 배차와 다음 액션을 확인한다.

```text
┌────────────────────────────┐
│ 내 배차                    │
│ 오늘 2건 · 진행 1건        │
├────────────────────────────┤
│ [오늘] [예정] [진행] [완료] │
│ 기간 필터 / 상태 필터      │
├────────────────────────────┤
│ 배차 카드                  │
│ 서울 강서 -> 부산 사상     │
│ 상차 09:30 · 하차 15:00    │
│ 품목 철재 · 배차금 450,000 │
│ [운행 준비] [상차완료]     │
├────────────────────────────┤
│ 배차 카드                  │
│ [보류] 사유 요약           │
│ [담당자 문의]              │
├────────────────────────────┤
│ 하단 탭                    │
└────────────────────────────┘
```

| 구분 | 내용 |
| --- | --- |
| 주요 정보 | 오늘 운행 건수, 진행 중 건수, 보류 건수, 상차지, 하차지, 시간, 품목, 배차금, 상태 badge |
| CTA | `상세보기`, `상차완료`, `하차완료`, `담당자 문의` |
| Empty | "현재 배차된 운행이 없습니다." |
| Error | "배차 내역을 불러오지 못했습니다. 다시 시도해 주세요." |
| Loading | "배차 내역을 불러오는 중입니다." |

## 3. SCR-002 배차 상세

### 목적

차주가 운행에 필요한 정보와 상태 변경, 담당자 문의, 특이사항 보고를 한 화면에서 처리한다.

```text
┌────────────────────────────┐
│ < 배차 상세                │
│ [운행 준비] #DT-12344      │
├────────────────────────────┤
│ 상태 타임라인              │
│ 배차됨 -> 운행 준비        │
├────────────────────────────┤
│ 상차지                     │
│ 주소 / 시간 / 담당자       │
│ [전화] [길찾기] [주소복사] │
├────────────────────────────┤
│ 하차지                     │
│ 주소 / 시간 / 담당자       │
│ [전화] [길찾기] [주소복사] │
├────────────────────────────┤
│ 화물 정보                  │
│ 품목 / 중량 / 차량 조건    │
├────────────────────────────┤
│ 운임 정보                  │
│ 배차금 / 조정 요약         │
├────────────────────────────┤
│ 특이사항                   │
│ [특이사항 보고]            │
├────────────────────────────┤
│ [상차완료]                 │
└────────────────────────────┘
```

| 구분 | 내용 |
| --- | --- |
| 주요 정보 | 상태 badge, 배차번호, 상하차지, 담당자 연락처, 화물 정보, 배차금, 조정 요약, 상태 타임라인 |
| CTA | `상차완료`, `하차완료`, `정산 확인`, `특이사항 보고`, `담당자 문의` |
| 안내 문구 | "하차완료는 차주의 현장 완료 등록입니다." / "주선사 확인 후 운행완료/정산대기로 전환됩니다." |
| Empty | "배차 정보를 찾을 수 없습니다." |
| Error | "배차 상세를 불러오지 못했습니다." |
| Loading | "배차 상세를 불러오는 중입니다." |

## 4. SCR-003 상차완료 확인 modal

```text
┌────────────────────────────┐
│ 상차완료로 등록할까요?     │
├────────────────────────────┤
│ 상차지: 서울 강서구        │
│ 등록 시각: 현재 시각       │
│ 등록 후 주선사에 공유됩니다│
├────────────────────────────┤
│ [취소]       [상차완료 등록]│
└────────────────────────────┘
```

| 구분 | 내용 |
| --- | --- |
| 목적 | 실수 등록을 막기 위해 상차지, 등록 시각, 상태 의미를 확인한다. |
| CTA | `상차완료 등록`, `취소` |
| Error | "상차완료를 저장하지 못했습니다. 다시 시도해 주세요." |
| Loading | "상차완료를 저장하는 중입니다." |

## 5. SCR-004 하차완료 확인 modal

```text
┌────────────────────────────┐
│ 하차완료로 등록할까요?     │
├────────────────────────────┤
│ 하차지: 부산 사상구        │
│ 등록 시각: 현재 시각       │
│ 현장 작업 완료 사실을      │
│ 주선사에 공유합니다.       │
├────────────────────────────┤
│ 주선사 확인 후 운행완료와  │
│ 정산대기로 전환됩니다.     │
├────────────────────────────┤
│ [취소]       [하차완료 등록]│
└────────────────────────────┘
```

| 구분 | 내용 |
| --- | --- |
| 목적 | 차주가 `하차완료`를 직접 등록하되, 주선사의 `운행완료` 판단과 다름을 안내한다. |
| CTA | `하차완료 등록`, `취소` |
| Error | "하차완료를 저장하지 못했습니다. 다시 시도해 주세요." |
| Loading | "하차완료를 저장하는 중입니다." |
