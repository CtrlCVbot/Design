# Wireframes - Phase 1B/1C 조회와 예외 처리

## 1. SCR-005 운행 내역

### 목적

차주가 월별 또는 기간별로 운행 결과와 정산 상태를 조회한다.

```text
┌────────────────────────────┐
│ 운행 내역                  │
│ 2026년 6월                 │
├────────────────────────────┤
│ [월 선택] [기간 설정]      │
│ [전체] [운행완료] [정산대기]│
│ [송금완료] [보류]          │
├────────────────────────────┤
│ 요약                       │
│ 운행 12건 / 송금 8건       │
├────────────────────────────┤
│ 운행 카드                  │
│ 서울 -> 부산 / 450,000     │
│ [송금완료] 06.28           │
├────────────────────────────┤
│ 하단 탭                    │
└────────────────────────────┘
```

| 구분 | 내용 |
| --- | --- |
| 주요 정보 | 월/기간, 상태 필터, 운행 건수, 송금 완료 건수, 정산대기 건수, 운행 카드 |
| CTA | 운행 카드 선택, 기간 설정, 담당자 문의 |
| Empty | "선택한 기간의 운행 내역이 없습니다." |
| Error | "운행 내역을 불러오지 못했습니다." |
| Loading | "운행 내역을 불러오는 중입니다." |

## 2. SCR-008 정산 홈

### 목적

차주가 월별 또는 기간별로 송금완료 금액, 미정산 금액, 보류 금액, 다음 송금 예정 내역을 한 화면에서 확인한다. 하단 `정산` 탭의 기본 화면이다.

```text
┌────────────────────────────┐
│ 정산                       │
│ 2026년 6월                 │
├────────────────────────────┤
│ 이번 달 송금 요약          │
│ 총액     1,190,000원       │
│ 송금완료 470,000원         │
│ 미정산   410,000원         │
│ 보류     310,000원         │
├────────────────────────────┤
│ 다음 송금 예정             │
│ 06.28 · 470,000원 · 1건    │
├────────────────────────────┤
│ [전체] [정산대기] [송금완료]│
│ [보류]                     │
├────────────────────────────┤
│ 정산 묶음 카드             │
│ 06.28 송금예정 · 470,000원 │
│ 포함 운행 1건              │
├────────────────────────────┤
│ 보류 카드                  │
│ 수량 차이 확인 중          │
│ [정산 문의]                │
├────────────────────────────┤
│ 하단 탭                    │
└────────────────────────────┘
```

| 구분 | 내용 |
| --- | --- |
| 주요 정보 | 월/기간, 송금완료 금액, 미정산 금액, 보류 금액, 다음 송금 예정, 상태별 정산 묶음 |
| CTA | 정산 묶음 선택, 기간 설정, 정산 문의 |
| Empty | "선택한 기간의 정산 정보가 없습니다." |
| Error | "정산 요약을 불러오지 못했습니다." |
| Loading | "정산 요약을 불러오는 중입니다." |

## 3. SCR-006 정산/송금 상세

### 목적

차주가 `운행 내역` 카드 또는 `정산 홈`의 정산 묶음에서 진입해 특정 운행 건의 차주 노출 가능 정산 정보와 송금 상태를 확인한다.

```text
┌────────────────────────────┐
│ < 정산/송금 상세           │
│ [정산대기] 또는 [송금완료] │
├────────────────────────────┤
│ 운행 정보                  │
│ 서울 -> 부산 / 06.24       │
├────────────────────────────┤
│ 금액                       │
│ 배차금 450,000             │
│ 조정 요약 +20,000          │
│ 송금금액 470,000           │
├────────────────────────────┤
│ 송금 정보                  │
│ 송금상태 / 송금일          │
├────────────────────────────┤
│ [정산 문의]                │
└────────────────────────────┘
```

| 구분 | 내용 |
| --- | --- |
| 주요 정보 | 운행 구간, 운행일, 배차번호, 배차금, 조정 요약, 송금금액, 송금상태, 송금일 |
| 노출 제외 | 내부 청구금, 수수료, 내부 메모, 송금 계좌 정보 |
| CTA | `정산 문의`, `운행 상세 보기` |
| Empty | "정산 정보가 아직 준비되지 않았습니다." |
| Error | "정산 정보를 불러오지 못했습니다." |
| Loading | "정산 정보를 불러오는 중입니다." |

## 4. SCR-007 특이사항 보고

### 목적

차주가 운행 중 또는 하차 후 예외 상황을 메모와 optional 사진으로 보고하고, 필요한 경우 보류 후보로 전달한다.

```text
┌────────────────────────────┐
│ < 특이사항 보고            │
├────────────────────────────┤
│ 유형 선택                  │
│ [파손] [수량 차이] [대기]  │
│ [주소/담당자] [금액 이견]  │
├────────────────────────────┤
│ 메모                       │
│ 내용을 입력해 주세요.      │
├────────────────────────────┤
│ 사진 첨부 optional         │
│ [사진 추가]                │
├────────────────────────────┤
│ [제출]                     │
└────────────────────────────┘
```

| 구분 | 내용 |
| --- | --- |
| 주요 정보 | 유형, 메모, optional 사진, 제출 후 안내 |
| 유형 | 파손, 수량 차이, 대기, 주소/담당자 이슈, 금액 이견, 기타 |
| CTA | `제출`, `사진 추가`, `취소` |
| Validation | "특이사항 내용을 입력해 주세요." |
| Error | "특이사항을 제출하지 못했습니다. 다시 시도해 주세요." |
| Loading | "특이사항을 제출하는 중입니다." |
