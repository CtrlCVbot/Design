# nbbb1.vercel.app/dispatches — Userflow

분석일: 2026-07-08 · 근거: 실 조작 기반 (01-기능분석-이벤트정리.md 참조)

이 화면의 사용자는 **관제 담당자(배차 매니저)** 1인이며, 핵심 과업은 4개다:
1. 신규 운행 등록  2. 차주 배정  3. 운행 상태 관제  4. 운임/정산 정보 관리 (+ 거래처/차주 소통)

---

## Flow 1. 신규 운행 등록 (해피 패스)

```mermaid
flowchart TD
    S([배차 관리 진입<br>좌: 운행등록 / 우: 운행내역]) --> C1{거래처 입력 방식}
    C1 -->|최근 거래처 칩 클릭| C2[거래처명·전화·담당자 자동입력<br>+ 자주 쓰는 구간 칩 갱신]
    C1 -->|직접 입력 후 검색| C3[우측 패널 → 주요 상하차지<br>추천 목록으로 전환]
    C3 -->|구간/상차지/하차지 선택| C4[주소 자동입력 + 패널 닫기]
    C2 --> R1{노선 입력 방식}
    R1 -->|자주 쓰는 구간 칩| R2[상차지+하차지 동시 입력<br>+ 운임 추천 버튼 노출 최근/최빈]
    R1 -->|주소 검색란 클릭| R3[카카오 우편번호 위젯<br>임베디드 레이어] --> R4[주소 선택]
    R1 -->|지역 칩 클릭| R4
    C4 --> R1
    R2 --> T1[상·하차일시: 퀵버튼<br>지금/오늘/내일/월요일 또는 피커]
    R4 --> T1
    T1 -.->|선택| W[+ 경유지 추가<br>최대 3개, 개별 삭제]
    T1 --> V1{차량 스펙}
    V1 -->|스펙 칩| V2[톤급+차종 동시 설정]
    V1 -->|셀렉트 직접| V2
    V2 --> P1[정산방법 선택<br>인수증이면 수수료 잠금]
    P1 --> P2[정산 예정일 퀵버튼<br>미지정/바로/15일/당월말/45일/익월말]
    P2 --> F1{운임 입력}
    F1 -->|추천 버튼| F2[최근/최빈 운임 자동입력]
    F1 -->|직접 입력| F3[콤마 자동 포맷]
    F2 --> G[화물품목·메모 입력]
    F3 --> G
    G --> SUB[+ 배차 등록]
    SUB --> VAL{필수값 검증<br>상차지·하차지·톤급·차종·운임}
    VAL -->|누락| ERR[alert: 누락 항목 나열] --> G
    VAL -->|통과| OK[리스트 최상단 '배차중' 행 추가<br>카운트 갱신 + 폼 초기화]
```

**설계 의도**: 모든 필드에 "직접 입력"과 "원클릭 추천(칩/퀵버튼)"의 이중 경로를 제공 — 반복 업무인 배차 등록의 입력 비용을 최소화하는 흐름.

## Flow 2. 차주 배정 (AI 추천)

```mermaid
flowchart TD
    S([운행 내역에서 '배차중' 행 확인<br>신호등 점 = 배차 난이도]) --> E[행 클릭 → 아코디언 확장]
    E --> AI[AI 배차 분석 어드바이저 확인<br>🟢원활 / 🟡주의 / 🔴우려<br>대기차주·경쟁화물·평균운임]
    AI -->|운임 보강 필요 판단| FARE[운임 스테퍼로 증액 → 수정<br>토스트 확인]
    AI --> A1{배정 방식}
    FARE --> A1
    A1 -->|수동| M1[차량번호·차주명·연락처 직접 입력]
    A1 -->|차량배정 버튼| P1[좌측 패널 → 차량/차주 배정 모드]
    P1 --> P2[선택된 배차 정보 확인]
    P2 --> P3{차주 선택}
    P3 -->|AI 최적 추천 3명| P4[추천 사유·거리·이력·계좌 검토]
    P3 -->|전체 가용 목록 ~20명| P4
    P4 --> SEL[선택 배정]
    SEL --> DONE[상태: 배차중 → 배차완료 자동 전환<br>차주 정보 행에 반영 / 좌측 패널 복귀]
    P1 -->|닫기| BACK[운행 등록 복귀]
```

**설계 의도**: 난이도 신호등(리스트) → AI 어드바이저(확장) → AI 추천 차주(배정 패널)로 이어지는 **3단계 AI 보조 깔때기**. 배정 실패 위험이 높은 건을 리스트 단계에서부터 시각적으로 선별하게 한다.

## Flow 3. 운행 상태 관제 (라이프사이클 관리)

```mermaid
flowchart LR
    subgraph 필터링
    F1[날짜 필터 7종] --> F3[교차 카운트 실시간 표시]
    F2[상태 필터 7종] --> F3
    F4[검색어+검색 버튼] --> F3
    end
    F3 --> ROW[행 확장]
    ROW --> ST{상태 버튼 클릭}
    ST --> S1[배차완료] --> S2[상차완료] --> S3[하차완료] --> S4[운행완료]
    ST -.-> S5[배차대기 되돌림]
    ST -.-> S6[배차취소]
    S1 & S2 & S3 & S4 & S5 & S6 --> UPD[뱃지·필터 카운트 즉시 갱신<br>현재 필터 밖이면 행 이탈]
```

## Flow 4. 운임/수수료 수정

```
행 확장 → [운임 및 수수료 정보 수정] 카드
  → 대상 토글(운/수) → 스테퍼(±1만/±5천/±1천) 또는 직접 입력
  → [수정] → 토스트 "수정 완료" → 리스트 운임 컬럼 즉시 반영
```

## Flow 5. 거래처/차주 소통 (대화방)

```
행 확장 → [대화방] (상세 헤더 또는 배정 차주 연락처 옆)
  → 좌측 패널 = 대화방 모드 (상대 정보 + 메시지 스레드)
  → 입력 → [전송] → 버블 즉시 추가
  → [전체 화면] → /chats 페이지 이동   |   [운행등록으로 복귀] → 기본 모드
```

## 패널 모드 전환 맵 (전체 상태도)

```mermaid
stateDiagram-v2
    state "좌측 패널" as L {
        [*] --> 운행등록
        운행등록 --> 차량차주배정 : 행 확장 내 [차량배정]
        차량차주배정 --> 운행등록 : [닫기] / [선택 배정] 완료
        운행등록 --> 대화방 : 행 확장 내 [대화방]
        대화방 --> 운행등록 : [운행등록으로 복귀]
        대화방 --> chats페이지 : [전체 화면]
    }
    state "우측 패널" as R {
        [*] --> 운행내역
        운행내역 --> 추천목록 : 거래처 [검색] / 구간 [...]
        추천목록 --> 운행내역 : [닫기] / [선택]
        운행내역 --> 접힘 : [접기]
        접힘 --> 운행내역 : [운행내역 펼치기]
    }
```
