# 화물 관리 화면 명세

## 1. 레이아웃 원칙

관리자 웹 서비스이므로 정보 밀도를 높게 유지한다. 다만 밀도는 무질서하게 많은 요소를 놓는 것이 아니라, 업무 판단에 필요한 정보를 위치와 계층으로 정리하는 방식으로 만든다.

### 데스크톱 기준

| 영역 | 기본 너비 | 역할 |
|---|---:|---|
| 전역 내비게이션 | 64px | 메뉴 이동, 현재 페이지 표시 |
| 필터 모드 사이드바 | 300px | 기존 전역 내비게이션이 오른쪽으로 확장되며 상세 필터 표시 |
| 1열: 화물 목록 | 60% | 검색, 필터, 테이블, 빠른 액션 |
| 2열: 상세/등록 | 40% | 화물 상세 조회, 수정, 신규 등록 |
| 3열: 조회/선택 | 30% | 차주/차량 또는 화주 조회, 등록, 선택 |

### 모바일 기준

| 영역 | 형태 | 역할 |
|---|---|---|
| 상단 바 | 고정 | 검색, 필터, 등록 진입 |
| 목록 | 카드형 | 필수 정보 5개 중심 |
| 상세 | 단계형 | 기본 정보, 운송 정보, 정산 정보, 배차 정보 순서 |
| 선택 | 전체 화면 단계 | 차주/화주 검색, 등록, 선택 |

## 2. 화물 목록

### 상단 도구 영역

| 컴포넌트 | 목적 | 비고 |
|---|---|---|
| 상세 필터 아이콘 버튼 | 왼쪽 필터 열기 | 검색 input 앞에 배치, 텍스트 대신 필터 아이콘 사용 |
| 검색 input | 화주, 상하차지, 차량번호, 차주명 검색 | 클릭 시점 검색 또는 Enter 검색 |
| 하단 뱃지 필터 1열 | 현재 상태 quick filter | 배차중, 배차완료, 운행중, 완료, 취소와 카운트 표시 |
| 하단 뱃지 필터 2열 | 기본 또는 적용된 필터 값 | 기본값은 일시, 톤수 전체, 차종 전체, 화주(선택안함), 차주(선택안함) |
| 화물 등록 버튼 | 2열 등록 화면 열기 | 목록 상단 우측 고정 |

검색 툴바 하단의 뱃지 필터는 데스크톱에서 1행 2열로 분리한다. 왼쪽은 현재 상태 파트, 오른쪽은 기본값 또는 사용자가 적용한 상세 필터 값 파트다. 적용 필터 파트는 화면 오른쪽 끝으로 밀지 않고 현재 상태의 마지막 뱃지 뒤에 적당한 간격을 두고 이어 배치한다.

1열 목록에서 화물 행을 선택하면 `2열 상세` 화면으로 전환한다. 이때 선택한 행은 2열 상세의 왼쪽 목록에서도 선택 상태를 유지한다.

1열 목록의 `화물 등록` 버튼을 선택하면 `2열 등록` 화면으로 전환한다. 2열 등록 화면의 왼쪽 목록은 2열 상세 화면의 왼쪽 목록과 같은 축약 구조를 사용한다. 즉 필터 아이콘이 있는 검색 툴바, 현재 상태/적용 필터 뱃지 요약을 유지하고, 테이블은 등록 맥락에 맞춘 컴팩트 컬럼 버전을 유지한다.

필터, 2열 상세, 2열 등록, 3열 차주 조회, 차주 등록, 3열 화주 조회처럼 왼쪽에 화물 목록을 유지하는 화면은 모두 1열 목록의 구조를 최대한 따른다. 화면 폭이 좁은 2열/3열에서는 `축약 검색 툴바 + 현재 상태/적용 필터 뱃지 요약 + 컴팩트 목록 테이블` 조합으로 줄여 표시한다.

### 데스크톱 테이블 컬럼

| 컬럼 | 표시 내용 | 우선도 | 모바일 표시 |
|---|---|---:|---|
| 상태 | 배차/운행 상태 뱃지 | 1 | 표시 |
| 화주 | 거래처명, 담당자 | 1 | 표시 |
| 상차지 | 지역 + 상세 주소 축약 | 1 | 노선으로 통합 |
| 하차지 | 지역 + 상세 주소 축약 | 1 | 노선으로 통합 |
| 상차일시 | 날짜/시간 | 1 | 표시 |
| 하차일시 | 날짜/시간 | 2 | 숨김 |
| 차량 스펙 | 톤급, 차종 | 1 | 필요 시 표시 |
| 차주/차량 | 차주명, 차량번호 | 1 | 상태 문구로 축약 |
| 운임 | 청구 운임 | 1 | 표시 |
| 정산 | 정산 방법, 예정일 | 2 | 숨김 |
| 배차 신호 | 난이도, 추천 가능 여부 | 2 | 아이콘 또는 짧은 라벨 |
| 최근 수정 | 담당자, 시간 | 3 | 숨김 |

2열 상세 화면의 왼쪽 목록은 1열 목록의 전체 구조를 축약해서 가져간다. 즉 화물 목록 헤더, 필터 아이콘이 있는 검색 툴바, 현재 상태/적용 필터 뱃지 요약을 유지하되, 실제 테이블은 현재처럼 컴팩트 컬럼 버전을 유지한다.

## 3. 상세 필터

상세 필터는 별도 컬럼이 새로 추가되는 패널이 아니다. 기존 왼쪽 전역 메뉴 사이드바(`DB/화/차/정/톡`)가 필터 모드로 전환되면서 오른쪽으로 넓어지는 형태를 기본으로 한다.

필터 모드 규칙:

- 필터를 열면 전역 메뉴 아이콘 목록은 숨기고 같은 사이드바 자리에 상세 필터를 표시한다.
- 최상단에는 뒤로가기 버튼을 둔다.
- 뒤로가기를 누르면 필터 모드만 종료되고, 다시 `DB/화/차/정/톡` 전역 메뉴 사이드바로 돌아간다.
- 화물 목록, 선택 행, 검색 결과 맥락은 유지한다.
- 필터는 `drawer`나 `dialog`가 아니며, 목록 위를 덮지 않는다.

필터 그룹:

1. 기간
2. 배차 상태
3. 운행 상태
4. 화주
5. 차주
6. 차량 스펙
7. 상차 지역
8. 하차 지역
9. 운임 범위
10. 정산 상태
11. 미수/주의 여부

필터 하단에는 `적용`, `초기화`, `저장된 필터` 액션을 둔다.

## 4. 화물 상세/수정 패널

2열 상세은 우선 읽기 중심의 업무 명세서처럼 보이도록 설계한다. 기본 상태에서는 입력 박스보다 `라벨 + 값` 텍스트를 우선하고, 수정이 필요할 때만 `수정 시작` 또는 섹션 액션을 통해 편집 흐름으로 들어간다.

2열 상세/등록 화면에서 목록과 상세/등록 패널 비율은 기본 데스크톱 기준 6:4로 둔다. 상세 패널에서는 최근 등록 정보 뱃지를 노출하지 않고 현재 선택된 화물의 확정 값을 문서처럼 보여준다. 등록 패널에서는 정보 밀도를 높이기 위해 섹션 상단 액션 행에 최신 등록 정보 뱃지를 보여주고, 뱃지를 선택하면 해당 섹션의 필드 값이 적용되는 패턴을 사용한다.

기본 데스크톱 폭에서는 다음 배치를 우선한다.

- 화주 정보: 업체명, 사업자번호, 담당자, 연락처, 이메일을 한 행에 배치
- 운송 정보: 상차행과 하차행을 같은 컬럼 구조로 배치하고, 4열에는 일시와 상/하차방법을 한 묶음으로 표시
- 화물 정보, 정산 정보, 배차 정보: 관련 필드는 가능한 한 한 행으로 묶어 표시
- 2열 상세/등록의 각 섹션은 공통 7컬럼 기준선을 사용해 같은 순서의 필드 X 좌표를 맞춘다. 화주, 운송, 화물, 정산, 배차 섹션은 같은 컬럼 토큰을 공유한다.
- 4번째 열의 기준 필드는 화주/배차 연락처, 운송 일시+방법, 화물 품목, 정산 이익으로 둔다.
- 화주 이메일은 상세 화면에서는 5~6열을 사용하고, 등록 화면에서는 2행 입력 행의 마지막 칸에 배치해 긴 값이 아래로 늘어지지 않게 한다.
- 일시는 `지금`, `오늘 HH:mm`, `내일 HH:mm`, 그 외 `MM/DD HH:mm` 형식으로 짧게 표기한다.
- 화면이 좁아지는 경우에만 줄바꿈 또는 단계형 배치로 전환

### 섹션

| 섹션 | 주요 필드 |
|---|---|
| 기본 정보 | 화물번호, 상태, 담당자, 생성일, 최근 수정일 |
| 화주 정보 | 업체명, 사업자번호, 담당자, 연락처, 이메일 |
| 운송 정보 | 상차지명, 주소, 상세주소, 일시+상/하차방법, 담당자, 연락처 / 하차행 동일 |
| 화물 정보 | 톤수, 차종, 실중량(최대 110), 품목 |
| 정산 정보 | 결제방법, 청구금, 배차금(운송비), 이익 |
| 배차 정보 | 차주명, 차량번호, 톤수, 연락처, 차종, 배차콜센터 |
| 이력 | 수정 이력, 상태 변경 이력, 배차 이력 |

운송 정보의 상차/하차 행은 공통 7컬럼 기준선 위에 6개 필드를 배치한다. `주소`와 `상세주소` 컬럼은 다른 보조 필드보다 넓게 잡아 2열 상세/등록 화면에서도 주소가 과도하게 좁아지지 않게 한다.

## 5. 화물 등록 패널

등록 화면은 상세 화면과 같은 섹션 구조를 사용한다. 등록 후 상세 화면으로 전환되기 때문에 사용자는 같은 정보 구조를 유지한 채 결과를 확인한다.

등록 화면의 각 섹션은 가능한 한 `상단 액션 행 + 입력 행`의 2행 구조를 기본으로 한다.

상단 액션 행의 구성:

- 왼쪽: 섹션명
- 가운데~오른쪽: 최근 선택/최근 등록 뱃지 리스트
- 오른쪽 끝: 조회가 필요한 섹션은 검색 아이콘 버튼, 조회가 없는 섹션은 32px 고정 액션 슬롯

입력 행의 구성:

- 화주 정보: 업체명, 사업자번호, 담당자, 연락처, 이메일 5개 입력창을 한 줄에 배치
- 화물 정보: 톤수, 차종, 실중량, 품목을 한 줄에 배치
- 정산 정보: 결제방법, 청구금, 배차금(운송비), 이익을 한 줄에 배치
- 배차 정보: 차주명, 차량번호, 톤수, 연락처, 차종, 배차콜센터를 한 줄에 배치
- 운송 정보: 상차/하차 데이터가 많기 때문에 상단 액션 행은 동일하게 맞추되, 입력은 상차행과 하차행을 각각 유지한다.

검색 아이콘 버튼 동작:

- 화주 정보: 오른쪽 3열의 화주/담당자 조회 화면으로 전환
- 운송 정보: 자주 쓰는 구간 또는 최근 구간 조회 흐름으로 진입
- 배차 정보: 오른쪽 3열의 차주/차량 조회 화면으로 전환

필수값:

- 화주
- 사업자번호
- 담당자
- 연락처
- 상차지
- 하차지
- 상차일시
- 톤수
- 차종
- 실중량
- 품목
- 결제방법
- 청구금
- 배차금

선택값:

- 경유지
- 하차일시
- 상하차 담당자/연락처
- 이익
- 차주/차량 사전 선택

## 6. 차주/차량 조회 및 등록 패널

3열의 차주/차량 패널은 조회, 등록, 선택, 배차 입력을 한 영역 안에서 처리한다.

3열 화면에서도 가운데 2열은 별도 요약 카드가 아니라 진입 전의 작업 패널을 유지한다. 차주 조회, 차주 등록, 화주 조회는 모두 진입 출처에 따라 `2열 등록` 패널 또는 `2열 상세` 패널을 유지한다. `2열 등록` 화면에서 조회 버튼을 누르면 가운데 열은 등록 패널을 유지하고, `2열 상세` 화면에서 변경 버튼을 누르면 가운데 열은 상세 명세서 패널을 유지한다. 목록 선택 맥락과 현재 작업 정보가 같은 위치에 남아 있어야 한다.

3열 상태의 컬럼 비율은 `3:4:3`으로 둔다. 왼쪽 목록은 30%, 가운데 작업 패널은 40%, 오른쪽 조회/등록/선택 패널은 30%다.

차주 등록 화면은 오른쪽 3열만 등록 폼으로 교체한다. 왼쪽 화물 목록과 가운데 작업 패널은 3열 차주 조회 화면과 동일하게 유지해, 사용자가 차주를 새로 등록하더라도 현재 화물 맥락을 잃지 않게 한다.

### 조회 목록 상단

- 검색 input
- 상태 필터: 전체, 대기중, 운행중, 휴무, 배차불가
- 차량 스펙 필터
- 거리/최근이력/운임순 정렬
- 차주 등록 버튼

### 차주 카드

| 정보 | 설명 |
|---|---|
| 차주명 | 주요 식별값 |
| 차량번호 | 배차 반영 대상 |
| 차량 스펙 | 톤급, 차종 |
| 기타 정보 | 추천 순위, 소속 구분, 현위치, 상태, 최근 이력, 계좌 상태 |

차주 카드는 화주 카드와 같은 4칸 요약 그리드를 사용한다. 차주명, 차량번호, 차량 스펙, 기타 정보를 한 줄에 배치해 3열의 좁은 폭에서도 빠르게 비교할 수 있게 한다.

### 배차 정보 입력

차주를 선택하면 3열 하단에 고정된 배차 입력 영역이 열린다. 선택 전에는 입력 영역을 숨기고, 선택 후에는 배차 콜센터만 입력 또는 선택할 수 있게 한다.

- 배차 콜센터
- 배차 버튼

## 7. 화주 조회 및 등록 패널

화주도 차주/차량과 동일한 3열 구조를 사용한다.

화주 조회 화면의 가운데 2열은 진입 출처에 따라 다르게 배치한다.

- 2열 등록 화면의 `화주/담당자 조회`에서 진입한 경우: 가운데 열은 `2열 등록`의 등록 패널을 그대로 유지한다.
- 2열 상세 화면의 `화주 변경`에서 진입한 경우: 가운데 열은 `2열 상세`의 명세서 패널을 그대로 유지한다.
- 이 규칙은 사용자가 화주를 선택한 뒤 어느 화면에 값이 적용되는지 즉시 이해하도록 돕기 위한 맥락 유지 장치다.

### 화주 카드

화주 조회 목록은 담당자 단위 카드가 아니라 화주 단위 카드로 묶는다. 화주 카드는 업체명, 사업자번호, 총 담당자 수, 기타 정보를 한 줄의 요약 그리드로 표시한다. 기타 정보에는 최근 사용 여부, 정산 조건, 주요 구간, 최근 운송 요약처럼 빠른 판단에 필요한 정보를 압축한다. 화주 카드를 클릭하면 해당 카드 바로 아래에 담당자 리스트가 들여쓰기되어 펼쳐지고, 사용자는 그 안에서 실제 적용할 담당자를 선택한다.

| 정보 | 설명 |
|---|---|
| 화주명 | 거래처 식별값 |
| 사업자번호 | 정산 및 세금계산서 기준 |
| 총 담당자 수 | 카드 하단에 펼칠 담당자 리스트 규모 |
| 기타 정보 | 최근 사용 여부, 정산 조건, 주요 구간, 최근 운송 요약, 위험 신호 |

### 담당자 리스트

| 정보 | 설명 |
|---|---|
| 담당자명 | 실제 적용 대상 |
| 담당 역할 | 배차 담당, 정산 담당, 야간 담당 등 |
| 연락처 | 전화번호 |
| 이메일 | 담당자 이메일 |

담당자 카드는 1행으로 표시한다. 담당자명, 담당 역할, 연락처, 이메일을 한 줄에 배치하고, 화주 카드보다 안쪽으로 들여써 회사-담당자 계층이 눈에 띄게 한다.

## 8. 상태와 색상

기존 디자인 시스템의 토스 계열 톤을 유지한다.

| 의미 | 색상 방향 | 사용처 |
|---|---|---|
| 주요 액션 | 파랑 | 등록, 저장, 배차, 선택 |
| 진행/대기 | 주황 | 배차중, 대기중 |
| 완료 | 청록 | 배차완료, 운행완료, 정산완료 |
| 위험 | 빨강 | 취소, 미수, 계좌 확인 필요 |
| 일반 배경 | 밝은 회색 | 페이지 배경, 입력 필드 |

## 9. 반응형 규칙

| 구간 | 구조 |
|---|---|
| 1440px 이상 | 전역 내비게이션 + 최대 3열 |
| 1024px 이상 | 1~2열 우선, 3열은 우측 오버레이가 아니라 페이지 내 열로 표시 |
| 768px 이상 | 목록 + 상세를 상하 또는 2열 혼합으로 조정 |
| 767px 이하 | 카드 목록 + 단계형 상세 화면 |

모바일에서 숨기는 정보는 삭제가 아니라 상세 화면으로 이동한다. 목록은 빠른 선택을 위한 요약만 담당한다.

## 10. 남은 결정 사항

| 항목 | 결정 필요 내용 |
|---|---|
| 배차 상태 전이 | 모든 상태를 자유 변경할지, 순차 전이만 허용할지 결정 필요 |
| 화주/차주 선택 후 3열 유지 여부 | 선택 후 바로 닫을지, 추가 편집을 위해 유지할지 결정 필요 |
| 정산 상태 범위 | 오더 단계에서 얼마나 보여줄지, 별도 정산 페이지와 어디서 나눌지 결정 필요 |
| 저장 방식 | 필드 단위 자동 저장인지, 섹션별 저장인지 결정 필요 |
| 권한 | 관리자, 배차 담당자, 정산 담당자별 수정 권한 구분 필요 |
