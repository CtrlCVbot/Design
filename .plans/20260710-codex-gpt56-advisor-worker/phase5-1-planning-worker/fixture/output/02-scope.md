# Scope boundaries

## In scope

- One planning package is the place to confirm both the decision and its scope. `[PROD-01]`
- The Pilot deliverable consists of exactly five documents. `[PROD-02]`
- Every material claim must be traceable by an evidence ID. `[REQ-01]`
- Links between the documents must be valid Markdown links. `[REQ-02]`
- The phased-rollout decision is documented without a production write, global-configuration change, or source-code modification. `[CON-01]`

## Non-goals

- Code implementation is a Non-Goal for this Pilot. `[PROD-02]`
- Production writes, global-configuration changes, and source-code changes are outside the permitted scope. `[CON-01]`

The evidence-to-decision trace is in [03-evidence-matrix.md](./03-evidence-matrix.md).
