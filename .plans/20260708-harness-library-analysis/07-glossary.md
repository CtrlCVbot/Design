# 07. 용어집 — 비개발자를 위한 쉬운 풀이

| 용어 | 쉽게 말하면 |
|---|---|
| **하네스 (harness)** | AI가 멋대로 하지 못하게 채우는 "안전벨트 세트" — 자동 안내 + 자동 검사 + 차단 장치의 묶음. 이 라이브러리에선 `.claude/`·`.codex/` 폴더 내용물 전체 |
| **canonical (원본)** | 진짜 원본이 되는 문서. 여기선 `guidelines/`·`playbooks/`의 평범한 마크다운. 도구용 사본은 전부 여기서 자동 생성 |
| **어댑터 (adapter)** | 같은 내용을 특정 도구(Claude, Codex)가 읽는 형식으로 바꿔 끼우는 변환 부품 |
| **번들 (bundle)** | 한 프로젝트의 하네스를 통째로 찍은 스냅샷 (`bundles/mm-broker/`). 설치 시 그대로 복사돼 같은 환경을 재현 |
| **훅 (hook)** | 특정 순간(편집 직전, 커밋 직전 등)에 자동으로 끼어들어 실행되는 검사원 프로그램 |
| **게이트 (gate)** | 통과 못 하면 다음 단계(커밋·머지)로 못 가게 막는 관문. "안내"와 달리 물리적으로 막는다 |
| **rules.json** | "어느 파일은 무엇을 하면 안 된다"를 적어 둔 규칙 장부. 검사원 3명(Claude·Codex·git)이 모두 이 장부 하나를 본다 |
| **단일 규칙 소스** | 규칙 장부가 한 곳뿐이라, 한 번 고치면 모든 검사원에게 동시에 반영되는 구조 |
| **add-only** | 이미 있는 파일은 건너뛰고 새 파일만 추가하는 복사 방식. 이 라이브러리에선 `.claude`·`.codex` 번들과 입구 파일에 적용된다 (반대로 지침 사본 `docs/dev-guidelines`는 재실행 때마다 최신본으로 갱신됨) |
| **멱등 (idempotent)** | 같은 명령을 몇 번을 다시 실행해도 결과가 같음 → 겁 없이 재실행 가능 |
| **스크럽 (scrub)** | 공유본을 만들 때 개인 계정·비밀번호를 `<QA_ACCOUNT>` 같은 자리표시자로 가리는 것 |
| **파리티 (parity)** | "내 환경과 남의 환경이 얼마나 똑같은가"의 정도. 이 라이브러리의 목표는 조건부 100% |
| **AGENTS.md** | 모든 AI 도구와 사람이 공통으로 읽는 입구 안내판. "어떤 일엔 어떤 문서를 보라"는 표가 있다 |
| **CLAUDE.md 셔틀** | Claude Code는 AGENTS.md를 직접 안 읽어서, "저기 안내판 보세요"라고 알려주는 한 줄짜리 연결 파일 |
| **pre-commit** | git이 커밋을 만들기 직전에 자동 실행하는 검사 지점. 여기 게이트를 걸면 도구와 무관하게 누구나 검사받는다 |
| **CI (gates.yml)** | GitHub 서버에서 코드가 올라올 때마다 자동으로 도는 검사. 로컬 검사를 우회해도 여기서 잡힌다 — 진짜 방어선 |
| **브랜치 보호** | main(본 줄기)에 "PR 필수 + 검사 통과 + 승인 1명" 규칙을 걸어 직접 반영을 서버 차원에서 금지하는 GitHub 설정 |
| **apply_patch** | Codex가 파일을 고칠 때 쓰는 도구. 파일 경로를 따로 안 알려줘서 전용 검사원(codex-rules-guard)이 필요했다 |
| **skill-rules.json** | "이 파일을 만지면 이 스킬을, 이 키워드가 나오면 저 스킬을 읽어라"는 자동 발동 트리거 장부 |
| **knowledge/** | 제품의 페이지별 엣지케이스·회귀 지도 같은 도메인 지식 창고. QA 때 자동 주입된다 |
| **프로브 (probe) / 지표** | 검사원들이 실제로 일하는지 몰래 세는 계수기. `.harness-metrics.jsonl`에 기록되고 harness-metrics.mjs로 집계 |
| **드리프트 (drift)** | 원본과 사본이 조금씩 어긋나는 현상. sync.mjs 한 방 동기화와 단일 규칙 소스가 이를 막는다 |
