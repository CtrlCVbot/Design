## 관찰 메모

1. 섹션 0·7은 nbbb1의 "그릇"에 해당 — Phase 3 자산은 그릇의 내용물(섹션·store·guard·리액티비티 골격)은 풍부하나 그릇 자체(콕핏 2패널 배치, 사이드바, 칩 카운트, 6단계 상태 표기)는 전부 재배치/신규가 필요하다. 그대로이식 0건인 이유.
2. NB-003 콕핏 전환의 최대 리스크는 store 축 확장 — 현 panelMode(new/copy/edit)와 wizard 상태 모델(IMPL-D6-006)이 top drawer 전제로 짜여 있어, 좌패널 상시 노출로 바꾸면 drawer open/close에 걸린 초기화·dirty·focus 계약(IMPL-D7-016/018/019) 재검증이 필수다.
3. 상태 모델(NB-063)은 UI 재배치가 아니라 도메인 정합 작업 — legacy 상세 시트의 상태 변경 자산이 Phase 3 콘솔로 아직 이식되지 않은 지점(배차취소/운행마감 disabled)이 nbbb1 구현의 실질 출발점이다.
