## 사용자 결정 필요 항목 (userDecisionItems)

1. **NB-048** — AI 최적 추천 차주는 Phase 3가 의도적으로 보류한 고급 배차 추천(IMPL-D5-025, Epic 차원 별도 Feature 결정 필요)·후보 자동 매칭(IMPL-D5-045, Phase 3 이후 별도 기획)의 재개를 전제로 함. 추천 패키지를 신규 기획으로 착수할지, 1차 범위는 추천 없이 전체 차주 목록만으로 갈지 결정 필요.
2. **NB-050** — 차주 카드의 계좌(IMPL-D5-004 계획만)·현위치 거리·월 총운임·현재 운행 상태 필드는 데이터 소스가 없음. 카드를 가용 데이터(이름/차량번호/스펙/최근 배차/소속 chip)로 축소해 시작할지, 차주 데이터 모델 확장을 선행할지 결정 필요.
3. **NB-051** — 운행중 차주 연계지 거리는 차주 실시간 위치·운행 상태 수집 인프라(차주 앱 연동 등)가 전제. 거리 계산 chain(IMPL-D2-050)은 재사용 가능하나 입력 좌표 소스가 없음 — 1차 범위 포함 여부 결정 필요.
4. **NB-055** — 운행데이터 기반 주요구간 랭킹은 신규 집계 백엔드가 필요하며, 서버측 후보 병합/랭킹 API는 Phase 3에서 보류됨(IMPL-D2-034). 보류를 해제해 서버 집계로 구현할지, client 측 근사 집계로 시작할지 결정 필요.

## 관찰 메모

- 배정 패널(§4)의 실행 계층(assignDriver·flow_status 동기화·수배 경로 enum)은 Phase 3 완료 자산이 그대로 받쳐주나, **탐색/추천 계층(가용 상태·위치·추천·월 총운임)은 자산 공백**이 뚜렷함 — nbbb1 §4의 차별 가치가 대부분 신규 데이터 모델에 걸려 있음.
- 추천 목록 패널(§5)은 주소 후보 파이프라인(D2-010~018)이 거의 그대로 이식 가능하고, 공백은 "빈도 기반 순위"라는 정렬 소스 하나로 수렴함.
- Phase 3 기각 자산과의 정면 충돌은 없음. 신규 계열 즉시 배차 기각(IMPL-D5-016)·autoAccept 실행 기각(IMPL-D5-017)은 nbbb1 배정 패널이 기등록 오더 대상 수동 배정이라는 점에서 경계 밖.
