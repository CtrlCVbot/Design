# M1 — nbbb1 운행 등록 폼 (NB-005~NB-026) × Phase 3 구현 자산 매핑

- 작성일: 2026-07-09
- NB 소스: `_working/nb-inventory-draft.md` §1 운행 등록 폼 (22항목)
- IMPL 소스: `_working/impl/D1~D8` 8개 파일 전수 확인 (주 대응: D1 화주, D2 운송구간·주소, D3 화물, D4 운임·정산, D6 등록·제출)
- 전제: nbbb1은 콕핏 좌측 패널 상시 폼, Phase 3는 목록→top drawer + wizard 구조. 전 항목 공통으로 "drawer/dialog → 좌측 상시 패널" UI 재배치가 필요하며, 아래 분류는 그 재배치를 제외한 **기능 자산 재사용 가능성** 기준이다.
- 구조적 차이 (전 항목 공통 유의): nbbb1 폼은 거래처명/전화/담당자를 **자유 텍스트**로 받지만, Phase 3 오더 모델은 화주 company 엔티티 + 담당자(selectedManagerId, 서버 필수 uuid — IMPL-D1-046/D8-048 계획만·불일치 잔존)를 요구한다. nbbb1 재구현 시 화주 엔티티 선택 체계(D1 자산)를 유지하는 방향을 전제로 매핑했다.
