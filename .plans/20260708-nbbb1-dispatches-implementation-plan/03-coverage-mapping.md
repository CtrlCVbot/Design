## 1. 개요

### 1.1 매핑 방향 정의

**NB→IMPL (섹션 2)**: nbbb1 기능 65항목 각각에 대해 Phase 3 자산 대응 여부를 4분류로 판정했다.

| 분류 | 정의 |
|---|---|
| 그대로이식 | 완료된 Phase 3 자산이 기능적으로 동일 — UI 재배치 수준으로 재사용 |
| 수정후이식 | 데이터·서비스·draft 자산은 재사용 가능하나 UI 컨테이너·규칙·wiring 수정 필요 |
| 신규기획 | 대응 자산이 없거나(데이터 소스 부재 포함) Phase 3 보류 자산이라 신규 기획 필요 |
| 제외후보 | nbbb1 분석 범위 밖 등으로 이번 범위에서 제외를 검토할 항목 |

**IMPL→NB (섹션 3)**: IMPL 429항목 중 M1~M5에 인용되지 않은 287항목(차집합)을 3분류로 대조했다 — 미배치-완료자산(nbbb1 기획 추가 여부 검토 대상), 미배치-보류기각(배치 불필요), 미배치-인프라(화면 매핑 비대상 기반 자산).

1. 섹션 0·7은 nbbb1의 "그릇"에 해당 — Phase 3 자산은 그릇의 내용물(섹션·store·guard·리액티비티 골격)은 풍부하나 그릇 자체(콕핏 2패널 배치, 사이드바, 칩 카운트, 6단계 상태 표기)는 전부 재배치/신규가 필요하다. 그대로이식 0건인 이유.

#### 관찰 메모

1. §3 행 확장의 운임 편집 체인(NB-040~043)은 Phase 3 정산 자산의 성숙도가 가장 높은 영역(저장 provider·목록 즉시 갱신·partial retry까지 완료)이라 이식 리스크가 낮다. 반면 상태 전환(NB-046)은 mutation이 의도적으로 끊겨 있어 legacy 자산 연결 + 전이 규칙 결정이 선행돼야 한다.
2. 목록(§2)은 "데이터 소스 재사용 + UI 전면 재설계" 패턴 — D7-043 보류의 재개 조건(신규 목록 기획)이 nbbb1로 충족되므로, 이 그룹 착수 전 D7-043 gate 해제 결정과 목록 API 500 이슈(D7-066/D5-026) 런타임 확인을 먼저 처리하는 것이 안전하다.
3. AI 계열(NB-035/038)은 Phase 3에 대응 자산이 전무하고 nbbb1 원본에도 산출 로직이 미기록이라, 데이터 소스 정의(대기 차주·경쟁 화물·구간 운임 통계)가 별도 기획 선행 과제다.

## 4. 매핑에서 드러난 구조적 발견

각 그룹의 관찰 메모와 판정 근거를 횡단 종합한 것이다. 모든 서술은 위 매핑 표의 ID를 근거로 한다.

### 4.1 지배적 이식 패턴 — "dialog 후보 목록의 표면화"

nbbb1 재구현의 본질은 신규 기능 개발이 아니라 **Phase 3 dialog/wizard 내부 자산을 상시 노출 표면으로 끌어올리는 재배치**다. 수정후이식 36건의 다수가 이 단일 패턴으로 수렴한다.

- 등록 폼(§1): dialog 내부에 있던 최근 화주(IMPL-D1-012)·주소 후보(IMPL-D2-011/013/014)·최근 화물(IMPL-D3-008) 목록을 폼 표면의 **인라인 칩**으로 표면화 — NB-005/009/015/023. M1 관찰 메모 기준 등록 폼 22항목 중 17항목(77%)이 완료 자산 기반 재사용.
- 패널(§4·§5): 차주 조회 dialog(IMPL-D5-002)·주소 검색 dialog(IMPL-D2-010)를 좌/우 **상시 패널 모드**로 재배치 — NB-049/054. 데이터·적용 로직은 무수정 재사용.
- 목록(§2·§3): 상세 top drawer(IMPL-D7-025)를 **행 하단 인라인 아코디언**으로, 탭 필터(IMPL-D7-022)를 **건수 칩**으로 재구성 — NB-034/027/029. "데이터 소스 재사용 + 표시 컨테이너 교체"의 동일 구도.

이 패턴의 공통 세부 수정은 (a) preview→apply 2단계의 즉시 적용 축약(NB-005/009), (b) 칩 노출 개수·순위 규칙 신설, (c) dirty/confirm guard(IMPL-D7-031/018)의 패널 모드 전환용 재지정(NB-053)이다.

### 4.2 신규기획 17건의 공통 근본 원인 — 집계 데이터 소스 부재

신규기획 판정의 다수는 UI가 아니라 **데이터 소스가 없어서**다. 세 덩어리로 묶인다.

1. **구간(상차→하차 pair) 빈도 집계 부재**: NB-007(구간 칩)·NB-022(운임 추천)·NB-055(추천 패널 주요구간)·NB-065(입력 추천 체인의 구간 단계)가 전부 동일 원인. Phase 3 주소 후보는 전부 side별 단일 주소 기준(IMPL-D2-013/014)이고 pair 랭킹 소스가 없다. **구간 집계 쿼리 하나가 NB 4개 항목을 동시에 해결한다**(M1 관찰 2, M3 관찰 메모 일치). 단, 집계 scope(선택 화주 한정 vs 자사 오더 전체)는 cross-company recent 기각 금지선(IMPL-D8-018, IMPL-D1-024 — RSMP-BLOCK-001/004)과의 경계 확인이 선행돼야 한다(NB-065 결정 항목).
2. **차주 실시간 상태·위치·이력 지표 부재**: NB-035(난이도 신호등)·NB-038(AI 어드바이저)·NB-048(AI 추천 차주)·NB-050(카드 확장 필드)·NB-051(연계지 거리)가 대기 차주/경쟁 화물/현위치/월 총운임 데이터 모델 부재라는 동일 공백 위에 있다. nbbb1 원본에도 산출 로직이 미기록(⚠)이라 지표 정의부터 별도 기획 선행 과제다(M2·M3 관찰 메모).
3. **정산 예정일 필드 부재**: NB-020(퀵버튼)·NB-039(확장 카드 표시)가 요구하는 필드가 Phase 3 정산 draft/payload 어디에도 없다(M1의 D4 전 항목 확인 근거) — 오더 스키마 확인부터 필요.

### 4.4 대화방(§6) — 유일한 전면 자산 공백 도메인

M4 전수 검토 결론대로 Phase 3에 채팅 도메인 자산이 존재하지 않는다(기각 기록조차 없음 — 애초 범위 밖). 재사용 가능한 것은 UI가 아니라 주변 계약뿐이다: 콘솔 store 패턴(IMPL-D7-013), 화주/차주 연락처 snapshot(IMPL-D1-001, IMPL-D5-001), legacy LMS 문자(IMPL-D5-067). 메시지 저장·실시간 전달·읽음 처리 백엔드는 Phase 3의 "backend 무변경" 원칙 밖의 **첫 신규 서버 도메인**이 될 가능성이 높아 별도 기획 패키지 규모로 산정해야 한다(M4 관찰 3). 채널 결정(실시간 채팅 신규 vs LMS 활용/병행)이 선행 결정이다(NB-060).

### 4.5 콕핏 구조 전환의 최대 리스크 — store 축 확장과 wizard 종속 자산

Phase 3의 "페이지 이동 없는 한 화면" 골격(IMPL-D7-002/013/014)은 nbbb1 콕핏과 방향이 같지만, **top drawer 오버레이 전제**로 짜여 있다. 좌측 상시 패널로 바꾸면 (a) panelMode 단일 축(new/copy/edit)을 좌 3모드 × 우 2모드+접힘의 2축 상태 머신으로 확장해야 하고, (b) drawer open/close에 걸린 초기화·dirty·focus 계약(IMPL-D7-016/018/019)과 wizard 상태 모델(IMPL-D6-006)의 재검증이 필수다(M5 관찰 2). 미배치-완료자산 115건 중 상당수가 "wizard 구조 종속 — nbbb1 상시 폼에서는 재검토 필요"로 표기된 것(IMPL-D6-007/009/010/011, IMPL-D5-062, IMPL-D8-011 등)도 같은 이유 — 이 자산들은 자동 승계가 아니라 콕핏 구조 확정 후 재판정 대상이다.

### 4.6 착수 전 선결 기술 이슈 (매핑 잔존 리스크)

매핑 과정에서 반복 인용된 미해소 이슈 — 화면 기획과 별개로 구현 착수 전 확인이 필요하다.

- **목록 조회 API 안정성**: smoke 중 500/0건 이슈 미확인 잔존 (IMPL-D7-066, IMPL-D5-026, IMPL-D8-029) — §2 목록·필터 리액티비티 품질의 전제 조건 (M2 관찰 2, M5 NB-064 비고).
- **서버/클라 validation 불일치**: 담당자 미선택 시 서버 400 (IMPL-D1-046, IMPL-D8-048 계획만, IMPL-D6-031 부분) — nbbb1 필수값 규칙 조정 시 함께 해소 (M1 NB-025).
- **저장 후 recent cache invalidate 미연결**: IMPL-D6-026, IMPL-D3-009 계획만 — nbbb1 입력 가속 체인(등록→최근 후보 갱신)의 신선도 확보에 필수 (M1 NB-025, M5 NB-065).
