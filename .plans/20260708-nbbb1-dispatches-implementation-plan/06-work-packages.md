# 06. Work Packages — nbbb1 /dispatches 구현 작업 분해

- 작성일: 2026-07-09
- 목적: 매핑 결과(M1~M5)의 NB 65항목을 한 세션 내 완료 가능한 work package(WP) 단위로 분해하고, 각 WP의 범위·분류 구성·선행 조건·완료 기준·난이도를 확정한다. 07 문서(Phase 로드맵)의 입력이 된다.
- 입력 소스:
  - `_working/mapping/M1.md` ~ `M5.md` (NB 65항목 분류·대응 IMPL 매핑)
  - `_working/nb-inventory-draft.md` (NB 화면 구조·기능 정의)
  - `_working/impl/D1-화주-거래처.md` ~ `D8-공통-횡단-기타.md` (IMPL 근거 원본, 필요 시 참조)
- ID 체계:
  - **NB-###**: nbbb1 목표 기능 항목 (`nb-inventory-draft.md`)
  - **IMPL-D#-###**: Phase 3 구현 자산 항목 (`_working/impl/D1~D8`)
  - **WP-###**: 본 문서가 신설하는 work package ID
  - **DEC-{주제}**: 사용자 결정 대기 주제 그룹. M1~M5의 userDecisionItems를 주제별로 묶은 명칭이며, 04 문서(사용자 결정 목록)의 주제 그룹명과 일치시켜 지칭한다. 본 문서 §2에 그룹 정의를 둔다.
  - **GAP-{주제}**: Phase 3 잔존 미해소 이슈(부분/계획만/미확인 상태 IMPL). 본 문서 §3에 정의를 둔다.
- WP 크기 규칙: 화면 영역 1개 또는 연관 기능 3~5개, 한 세션 내 완료 가능한 규모. 초과 시 분할했다(예: 등록 폼 22항목 → WP-003~010의 6개+2개 패키지로 분할).

## 1. 분해 원칙

1. **NB 전수 배치**: NB-001~NB-065 전 항목이 정확히 1개 WP에 속한다(§6 커버리지 표로 검증). 제외후보 NB-058은 결정 대기 WP(WP-024)에 별도 표기로 배치한다.
2. **화면 영역 응집**: nbbb1 화면 구조(§0 전역 / §1 등록 폼 / §2 목록 / §3 행 확장 / §4 배정 패널 / §5 추천 패널 / §6 대화방 / §7 횡단)를 기본 축으로 하되, 횡단 항목(NB-063~065)은 실제 구현이 일어나는 화면 WP에 합류시켰다(NB-064→목록 필터, NB-065→운임 추천 체인, NB-063→상태 전환).
3. **결정 격리**: 사용자 결정(DEC)에 막히는 신규기획 항목은 결정 불요 이식 항목과 같은 WP에 섞지 않는다. 예: 정산 예정일(NB-020, 스키마 결정 필요)은 정산 토글 WP(WP-007)가 아니라 오더 필드 확장 WP(WP-009)로 분리.
4. **데이터 소스 공유 묶음**: M1 관찰 메모 2("구간(pair) 집계 쿼리 하나가 NB-007·NB-022·NB-055를 동시에 해결")에 따라 집계 기반 항목을 WP-019/020으로 묶되, 백엔드 집계 신설(WP-019)과 그 소비(WP-020)를 분리해 세션 크기를 지켰다.
