# 미결 질문과 가정

## 1. Figma 주석 기반 미결 질문

| ID | 질문 | 영향 | 추천 처리 |
| --- | --- | --- | --- |
| `OQ-MM-001` | 수입 flow에서 `B/L 입력` 후 적하목록 발생 전까지 추가 작업이 있는가? | 수입 화면/상태 모델의 누락 가능성 | 업무 담당자 확인 후 수입 상세 flow 보완 |
| `OQ-MM-002` | `EDI 전송 / 확인 / 정정`을 한 번에 처리하는 화면이 필요한가? | EDI 오류 대응 UX와 화면 구조 결정 | EDI 작업 큐를 별도 화면 후보로 채택 |
| `OQ-MM-003` | 유니패스, 트래킹 조회는 어느 단계에서 필수인가? | 외부 조회 링크/상태 저장 범위 결정 | `B/L`, 적하목록, 운송완료 전후 사용 시점 확인 |
| `OQ-MM-004` | `Original B/L`, `Check B/L`, `Surrender B/L`, `Sea way bill`, `AWB`를 같은 모델로 다룰 수 있는가? | `B/L` 유형별 필수 필드와 상태가 달라질 수 있음 | 유형별 필드/상태 매트릭스 작성 |
| `OQ-MM-005` | `TOMS`와 `PMS`의 최종 책임 경계는 어디인가? | 중복 입력과 마이그레이션 전략에 영향 | 신규 기능은 `PMS`, 기존 접점은 연동/참조로 분리 |
| `OQ-MM-006` | 다우 송금/입금 확인을 `PMS`에서 실시간으로 볼 필요가 있는가? | 정산 자동화 범위 결정 | MVP는 상태 수동 입력, 후속은 연동 검토 |
| `OQ-MM-007` | 적하목록 선사 제출은 메일/FAX 제출 이력만 저장하면 되는가? | 증빙 보관과 감사 추적 범위 결정 | 제출 채널, 제출 파일, 제출 시간 저장 |
| `OQ-MM-008` | 수출 `FCL/LCL`과 수입 모두 같은 `Order` 객체를 써도 되는가? | 데이터 모델 단순화와 예외 처리 균형 | 공통 `Order` + 분기별 task 구조 추천 |

## 2. 현재 가정

| ID | 가정 | 영향 |
| --- | --- | --- |
| `ASM-MM-001` | `수출입 통합 (TO - BE)` 섹션이 현재 목표 flow의 기준이다. | 문서 전체가 해당 섹션 중심으로 작성됨 |
| `ASM-MM-002` | 보라색 `PMS` 박스는 내부 제품화/화면화 후보이다. | 화면 후보와 요구사항을 `PMS` 기준으로 구성 |
| `ASM-MM-003` | 회색 박스는 당장 자동화되지 않는 외부/수동/기존 시스템 업무이다. | MVP에서는 상태/증빙 등록 중심으로 다룸 |
| `ASM-MM-004` | `B/L 입력`은 수출/수입 공통 게이트이다. | 상태 모델에서 핵심 분기점으로 설정 |
| `ASM-MM-005` | 실적정산은 정산/청구/입금 완료 후 최종 상태이다. | 정산 완료 기준을 후속 상세화 필요 |

## 3. 결정 후보

| 결정 항목 | 추천안 | 이유 | Trade-off |
| --- | --- | --- | --- |
| 첫 상세화 범위 | `B/L 입력`부터 EDI까지 | 수출/수입 공통이면서 Figma 개선 주석이 집중됨 | 견적/정산 자동화는 후순위가 됨 |
| 화면 구성 | 오더 상세 안에 `서류`, `B/L`, `EDI`, `정산` 탭 구성 | 하나의 오더에서 전체 상태를 추적하기 쉬움 | 화면이 커질 수 있어 작업 큐 보완 필요 |
| EDI UX | 별도 `EDI 작업 큐` + 오더 상세 내 상태 요약 | 오류/정정 대상만 모아보기 가능 | 오더 상세과 큐 간 상태 동기화 필요 |
| 외부 시스템 연동 | 초기에는 링크/증빙/상태 수동 등록 | 빠르게 운영 적용 가능 | 자동화 수준은 낮음 |
| 데이터 구조 | 공통 `Order` + 분기별 `Task` | 수출/수입 공통과 예외를 함께 관리 | task 설계가 필요 |

## 4. 후속 확인 체크리스트

| 순서 | 확인할 것 | 확인 대상 |
| --- | --- | --- |
| 1 | `수출 1차/2차`, `수입 1차/2차`와 TO-BE 통합본의 차이 | Figma 보드 |
| 2 | 수입 `B/L 입력` 후 적하목록 전 단계 | 수입 운영 담당자 |
| 3 | `B/L` 유형별 필수 필드와 상태 | 수출입 운영 담당자 |
| 4 | EDI 정정신고의 실제 입력 항목과 승인/재전송 조건 | EDI 담당자 |
| 5 | 실적정산 완료 조건 | 정산 담당자 |
| 6 | 다우, `Ulogis`, 선사 사이트 연동 가능성 | 시스템 담당자 |
