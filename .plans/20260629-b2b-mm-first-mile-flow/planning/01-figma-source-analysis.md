## 4. 시각 범례 해석

`수출입 통합 (TO - BE)` 화면에서 색상은 시스템/행위 주체를 구분하는 용도로 보인다.

| 색상/형태 | 의미 |
| --- | --- |
| 파란 타원 | 시작점 또는 로그인 |
| 보라색 박스 | `PMS` 중심 내부 업무/화면 |
| 회색 박스 | 메일, 엑셀, 유선, 다우, TOMS, 외부 전달 등 내부 시스템 밖 또는 반자동 업무 |
| 초록색 박스 | 선사 사이트, `Ulogis` 등 외부 시스템 직접 작업 |
| 보라색 다이아몬드 | 핵심 분기 또는 게이트 |
| 노란 라벨 | 조건, 주석, 미결 질문, 강조 영역 |
| 점선 박스 | 외부/반자동 협업 구간 또는 묶음 업무 |

## 5. TO-BE 통합 flow 요약

### 5.1 공통 진입

1. `로그인 (PMS)`
2. `기초 데이터 등록 (PMS)`
3. `신규 업체 / 담당자 등록 (PMS)`
4. `견적 문의 (TOMS, 메일)`
5. 화주와 스케줄 조율 구간으로 진입

### 5.2 견적 및 오더 등록

| 단계 | 시스템/채널 | 산출물 |
| --- | --- | --- |
| 선사, 항공사, 콘솔 비용 확인 | 메일, 유선 | 운임/스케줄 확인 |
| 견적서 작성 | 다우, 엑셀 | `견적서` |
| 견적서 공유 | TOMS, 메일, 카톡 | 공유된 견적 |
| 오더/서류 등록 | PMS | 오더 및 서류 초안 |
| 오더 등록 | TOMS | 외부/기존 오더 |
| 서류 등록 | PMS | `CI / PL` |
| 오더 디테일 확인/수정 | PMS | 정리된 오더 상세 |

### 5.3 `B/L` 입력 이후 분기

`B/L 입력 (PMS)`이 수출/수입 flow의 공통 게이트로 보인다.

| 분기 | 대표 흐름 |
| --- | --- |
| `FCL 수출` | 선복 부킹 요청 -> 선사 승인 -> 부킹 번호 부여 -> `S/R` 제출 -> `VGM` 전송 -> Master copy `B/L` 수신 -> 비용 송금/이체증 제출/Master way bill 발행 |
| `LCL 수출` | 콘솔사 선적 요청 -> `S/R` 출력 후 콘솔사 제출 -> 콘솔사 Master copy `B/L` 수신 -> Master way bill 발행 |
| `수입` | `B/L 입력 (양재 IT)` 이후 `M/BL`, `H/BL` 또는 적하목록/EDI 흐름으로 연결 |

### 5.4 적하목록, EDI, 운송/선적 완료

| 단계 | 시스템/채널 | 메모 |
| --- | --- | --- |
| `M/BL`, `H/BL` 발행 | PMS | 양식: 적하목록 |
| 적하목록 발행/출력 | PMS | EDI 전송 전 산출물 |
| EDI 전송 | PMS | 오류 여부 분기 |
| 오류 `N` | 메일/FAX로 선사 제출 | 정상 제출 flow |
| 오류 `Y` | EDI 정정신고 | 정정 후 선사 제출 |
| 선적완료 | 수출 | 이후 정산/청구 flow |
| 운송완료 | 수입 | 이후 정산/청구/`D/O` flow |

### 5.5 정산/청구/입금

| 분기 | 대표 흐름 |
| --- | --- |
| 수출 | 선적완료 -> 매입확인/해외 정산서 -> 지출결의서 발행 -> 송금 -> 화주 청구서 -> 세금계산서 발행 -> 입금 확인 -> 실적정산 |
| 수입 | 운송완료 -> 화주 청구서 -> 현금 또는 신용거래 -> 지출결의서/해외 정산서 -> 송금/이체증 -> `M/D/O`, `H/D/O` 발급/출력 -> 화주/운송사 전달 -> 세금계산서 -> 입금 확인 -> 실적정산 |

## 6. 확인 제한

Figma MCP 호출 제한으로 `수출 1차`, `수출 2차`, `수입 1차`, `수입 2차` 섹션의 확대 스크린샷과 내부 텍스트 전체 추출은 진행하지 못했다.

따라서 이 문서의 직접 근거는 다음으로 제한한다.

| 근거 | 수준 |
| --- | --- |
| 최상위 섹션 목록 | 직접 확인 |
| `수출입 통합 (TO - BE)` 전체 스크린샷 | 직접 확인 |
| `수출입 통합 (TO - BE)` 좌/중/우 crop | 직접 확인 |
| 1차/2차 섹션의 세부 비교 | 미확인, 후속 보완 필요 |
