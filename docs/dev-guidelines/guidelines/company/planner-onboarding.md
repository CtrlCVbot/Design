# 개발이 처음이어도 괜찮습니다 — 시작 안내

이 프로젝트는 안전장치(하네스)가 켜져 있어서, 큰 실수는 자동으로 막아 줍니다. 아래 흐름만 따르면 됩니다.

## 어떻게 일하나 (5단계)

1. **원하는 걸 한국어로 설명하세요.** 예: "주문 목록에 검색창 추가해줘". 코드 용어를 몰라도 됩니다.
2. **AI가 계획을 먼저 보여 줍니다.** 읽어 보고 맞으면 "응, 진행해", 아니면 "그게 아니라 ~"라고 고쳐 말하세요.
3. **AI가 코드를 작성합니다.** 이 프로젝트 규칙(테스트 먼저 짜기 등)은 AI가 알아서 따릅니다.
4. **저장(커밋)할 때 자동 검사가 돕니다.** 테스트·빌드·비밀값 검사를 통과해야만 저장됩니다.
   - 통과하면 그대로 진행됩니다.
   - **빨간 메시지가 나오면** 멈추고, 그 메시지를 AI에게 그대로 보여주며 "이거 고쳐줘"라고 하세요.
5. **작업이 끝나면 개발자에게 리뷰를 요청하세요.** ("PR 올려줘"라고 하면 됩니다. 개발자가 확인 후 합칩니다.)

## 꼭 지킬 것 3가지

- **새 브랜치에서 작업하세요.** main(본 줄기)에 바로 올리지 않습니다. "브랜치 만들어줘"라고 하면 됩니다.
- **빨간 에러를 우회하지 마세요.** 검사를 끄거나 건너뛰지 말고, 메시지대로 고치거나 개발자에게 보여주세요.
- **위험한 작업은 직접 하지 마세요.** 삭제·강제 푸시·운영 배포는 개발자에게 부탁하세요.

## 막혔을 때

- 같은 에러가 **2번 이상 반복**되면 → 거기서 멈추고 개발자에게 화면을 보여주세요. 계속 시도하면 더 꼬일 수 있어요.
- 무슨 말인지 모르겠는 영어 에러가 나오면 → AI에게 "이 에러 한국어로 쉽게 설명해줘"라고 하세요.

## 안심해도 되는 이유

- 저장 전 검사와 (설정돼 있으면) 서버 검사가 있어서, 통과 못 한 코드는 본 줄기로 못 갑니다.
- 모든 작업은 git에 기록으로 남아서 되돌릴 수 있습니다.
- 그래도 **로직이 진짜 맞는지는 사람이 봅니다.** 그래서 마지막에 개발자 리뷰가 있는 거예요. 검사를 통과했다고 "완벽"은 아니니, 리뷰 단계를 건너뛰지 마세요.
