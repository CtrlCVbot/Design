# 세션 마무리 (/wrap)

세션 종료 시 작업 내용을 분석하고 문서화, 자동화 기회를 제안합니다.

## 수행 단계

### Phase 1: 병렬 분석 (4개 에이전트)

다음 4개 분석을 **Task 도구를 사용하여 병렬로** 실행합니다:

#### 1. doc-updater (문서 업데이트 제안)
- 이번 세션에서 발견한 패턴, 규칙, 주의사항 분석
- CLAUDE.md에 추가할 내용 제안
- dev/active/*/context.md 업데이트 필요 여부 확인

#### 2. automation-scout (자동화 기회 탐지)
- 세션에서 반복된 패턴 식별
- 새로운 skill/command/agent로 자동화할 수 있는 작업 제안
- 기존 skill 개선점 발견

#### 3. learning-extractor (학습 내용 추출)
- 이번 세션에서 배운 것
- 실수했던 것과 해결 방법
- 새로 발견한 코드베이스 패턴

#### 4. followup-suggester (후속 작업 정리)
- 미완성 작업 목록
- 다음 세션 우선순위 제안
- 관련 이슈/TODO 정리

### Phase 2: 중복 검증

Phase 1 결과를 검토하여:
- CLAUDE.md에 이미 있는 내용 필터링
- 기존 skill과 중복되는 제안 제거
- 실제로 가치 있는 제안만 선별

### Phase 3: 사용자 선택

AskUserQuestion 도구로 다음 옵션 제공:
- 커밋 생성 여부
- CLAUDE.md 업데이트 여부
- 새 skill/command 생성 여부
- context.md 업데이트 여부

## 출력 형식

```markdown
## 세션 요약

### 작업 완료
- [x] 완료된 작업 1
- [x] 완료된 작업 2

### 문서 업데이트 제안
> CLAUDE.md에 추가 권장:
```
[추가할 내용]
```

### 자동화 기회
- **제안 1**: [설명]
  - 구현 방법: skill/command/agent
  - 예상 효과: ...

### 학습 내용
- **발견**: [내용]
- **실수 → 해결**: [내용]

### 다음 세션 TODO
1. [ ] 우선순위 높음: ...
2. [ ] 우선순위 중간: ...
```

## 실행 방법

1. 대화 컨텍스트 전체를 분석합니다
2. git status로 변경된 파일 확인합니다
3. 4개 분석을 병렬로 실행합니다
4. 중복 검증 후 결과를 통합합니다
5. 사용자에게 액션 선택지를 제공합니다
