# 플랜 공유용 마크다운 생성 (/share-plan)

현재 플랜을 팀 공유용 마크다운 파일로 변환합니다.

## 플랜 소스 탐색 순서

1. `dev/active/*/plan.md` (활성 작업 플랜)
2. `.claude/plans/*.md` (Claude 플랜 파일)
3. 현재 대화 컨텍스트에서 논의된 플랜

소스를 찾지 못하면 사용자에게 확인합니다.

## 출력 포맷

아래 구조로 변환하여 `dev/active/[task-name]/plan-shared.md`에 저장합니다.
`dev/active/[task-name]/` 디렉토리가 없으면 프로젝트 루트에 `plan-shared.md`로 저장합니다.

```markdown
# [작업명] 실행 계획

> 작성일: YYYY-MM-DD

---

## 배경

이 작업이 필요한 이유와 현재 상태를 설명합니다.

## 목표

- 목표 1
- 목표 2

## 구현 범위

### Phase 1: [제목]
- 내용

### Phase 2: [제목] (있는 경우)
- 내용

## 단계별 작업

| Step | 작업 | 상태 | 비고 |
|------|------|------|------|
| 1 | 작업 설명 | ⬜ | |
| 2 | 작업 설명 | ⬜ | |

> 상태: ⬜ 대기 / 🔄 진행중 / ✅ 완료 / ⏭️ 스킵

## 파일 변경 맵

### 수정 파일
| 파일 | 변경 내용 |
|------|----------|

### 신규 파일
| 파일 | 용도 |
|------|------|

### 삭제 파일 (있는 경우)
| 파일 | 이유 |
|------|------|

## 검증 방법

- **단위 테스트**: 검증 대상
- **통합 테스트**: 검증 대상
- **QA**: 검증 시나리오
```

## 변환 규칙

### 1. 원본 플랜에서 추출할 내용
- 코드 블록의 구현 디테일 → **제거** (공유 문서에 코드 불필요)
- 비즈니스 모델/문제 분석 → **배경** 섹션으로
- 설계 결정/원칙 → **배경** 섹션 하단에 간결하게
- 구현 단계 → **단계별 작업** 테이블로 변환
- 파일 변경 → **파일 변경 맵** 테이블로 변환
- 검증 방법 → **검증 방법** 섹션으로

### 2. 진행 상태 반영
- `dev/active/[task]/tasks.md` 가 있으면 체크리스트 상태를 단계별 작업 테이블에 반영
- MEMORY.md의 진행 상황도 참고

### 3. 톤 & 스타일
- 코드/기술 용어는 유지하되, 구현 세부사항은 생략
- 각 섹션은 비개발자도 맥락을 이해할 수 있도록 작성
- 간결하게, 불필요한 반복 없이

## 수행 단계

1. 플랜 소스 탐색 (위 순서대로)
2. `dev/active/[task]/tasks.md` 있으면 함께 읽기
3. 위 포맷으로 변환하여 파일 저장
4. 저장 경로를 사용자에게 안내
