# Dev Docs 생성

현재 승인된 계획 또는 논의된 작업을 기반으로 dev docs를 생성합니다.

## 수행 단계

1. **작업명 확인**: 사용자에게 작업명을 확인하거나 대화에서 추론
2. **디렉토리 생성**: `dev/active/[task-name]/` 생성
3. **plan.md 생성**:
   - 작업 목표
   - 주요 변경 사항
   - 예상 영향 범위
4. **context.md 생성**:
   - 관련 핵심 파일 경로
   - 주요 결정 사항
   - 참조해야 할 기존 코드
5. **tasks.md 생성**:
   - 체크리스트 형태 (- [ ] task)
   - 작은 단위로 분해
   - 우선순위 순서로 정렬

## 출력 형식

```
dev/active/[task-name]/
├── plan.md      # 계획 문서
├── context.md   # 컨텍스트 문서
└── tasks.md     # 작업 체크리스트
```

## 주의사항
- 컨텍스트가 부족하면 사용자에게 질문
- 기존 dev/active/에 같은 이름이 있으면 확인
- 작업 완료 시 dev/completed/로 이동 안내
