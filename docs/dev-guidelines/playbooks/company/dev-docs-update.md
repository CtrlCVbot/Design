# Dev Docs 업데이트

현재 진행 중인 작업의 dev docs를 업데이트합니다.
컨텍스트 부족이나 세션 전환 전에 사용합니다.

## 수행 단계

1. **활성 작업 확인**: `dev/active/` 디렉토리에서 현재 작업 찾기
2. **context.md 업데이트**:
   - 새로 발견된 관련 파일 추가
   - 주요 결정 사항 기록
   - 다음 세션을 위한 힌트 추가
   - "Last Updated" 타임스탬프 갱신
3. **tasks.md 업데이트**:
   - 완료된 작업 체크 (- [x])
   - 새로 발견된 작업 추가
   - 현재 진행 중인 작업 표시

## context.md 업데이트 형식

```markdown
## Last Updated
YYYY-MM-DD HH:MM

## 다음 세션 힌트
- 현재 진행 중: [작업 내용]
- 다음 단계: [다음 할 일]
- 주의사항: [알아야 할 것]
```

## 사용 시점
- 컨텍스트가 부족해지기 전
- 긴 작업 중간에 주기적으로
- 세션 종료 전
